# 任务清单模板

这份文档用于控制 Agent 每次执行什么、属于哪个执行层、允许改什么、禁止改什么、如何验收。

## 1. 状态说明

| 状态 | 含义 |
|---|---|
| TODO | 未开始 |
| DOING | 正在做 |
| REVIEW | 等待审查 |
| FIXING | 修复中 |
| DONE | 已完成 |

## 2. 任务类型

| 任务类型 | 说明 | 常见产物 |
|---|---|---|
| DOC | 文档任务 | PRD、页面清单、数据规范 |
| DESIGN | 设计 / 原型任务 | 低保真原型、高保真说明、页面状态图 |
| FRONTEND | 前端任务 | 页面、组件、样式、交互、状态展示 |
| BACKEND | 后端任务 | 数据模型、API、业务规则、存储 |
| API | 接口 / 联调任务 | 接口规范、Mock 映射、错误状态 |
| REVIEW | 审查 / 验收任务 | 审查报告、验收报告、修复任务 |

## 3. 任务清单

| 任务编号 | 任务类型 | 任务名称 | 依赖文档 | 允许修改范围 | 禁止修改范围 | 验收标准 | 状态 |
|---|---|---|---|---|---|---|---|
| TASK-001 | DOC |  |  |  |  |  | TODO |

## 4. Agent 输入模板

```text
当前任务：TASK-xxx
任务类型：DOC / DESIGN / FRONTEND / BACKEND / API / REVIEW

目标：

必须阅读：

允许修改：

禁止修改：

验收标准：

完成后输出：
1. 修改文件列表
2. 实现说明
3. 对应文档
4. 验证方式
5. 风险点
```

## 5. 任务控制检查

- [ ] 一个任务只做一件事。
- [ ] 每个任务都有任务类型。
- [ ] 每个任务都有依赖文档。
- [ ] 每个任务都有允许修改范围。
- [ ] 每个任务都有禁止修改范围。
- [ ] 每个任务都有验收标准。
- [ ] 前端任务不能自行新增后端接口。
- [ ] 后端任务不能自行新增 PRD 之外的业务能力。
- [ ] 设计任务不能自行新增页面范围。
- [ ] Agent 不得自行合并多个任务。
