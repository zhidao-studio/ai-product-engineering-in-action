# 文档到代码映射模板

这份文档用于回答：代码为什么这样写，它来自哪份文档。

## 1. 总体映射

| 文档 | 约束代码 | 说明 |
|---|---|---|
| 项目总说明 | README / 页面文案 | 控制产品边界 |
| PRD | 功能实现 | 控制功能范围 |
| 页面清单 | 页面结构 | 控制页面范围 |
| 用户流程 | 交互逻辑 | 控制动作和结果 |
| 数据规范 | 数据结构 | 控制字段和存储 |
| 技术结构 | 文件职责 | 控制实现边界 |
| 任务清单 | 每次变更 | 控制 Agent 执行范围 |

## 2. 页面到代码映射

| 页面 | 来源文档 | 代码位置 | 验收方式 |
|---|---|---|---|

## 3. 功能到代码映射

| 功能编号 | 功能名称 | 来源文档 | 代码位置 | 验收标准 |
|---|---|---|---|---|

## 4. 数据字段到代码映射

| 字段 | 来源文档 | 使用位置 | 说明 |
|---|---|---|---|

## 5. 审查问题

- [ ] 代码是否对应某个任务？
- [ ] 页面是否来自页面清单？
- [ ] 字段是否来自数据规范？
- [ ] 交互是否来自用户流程？
- [ ] 功能是否满足 PRD 验收标准？
- [ ] 是否存在无法回溯到文档的代码？

## 6. 一句话原则

```text
代码不能回溯到文档，就不能通过验收。
```
