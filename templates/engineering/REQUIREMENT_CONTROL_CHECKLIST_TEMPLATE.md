# 需求实现控制检查清单模板

这份清单用于判断：需求是否已经被文档控制住，是否可以交给 Agent 实现，以及实现后是否可以验收通过。

## 1. 实现前检查

### 产品边界

- [ ] 需求已经进入项目总说明。
- [ ] 目标用户明确。
- [ ] 第一版做什么明确。
- [ ] 第一版不做什么明确。

### 功能范围

- [ ] 功能已经进入 PRD。
- [ ] 功能有编号。
- [ ] 功能有优先级。
- [ ] 功能有验收标准。

### 页面和流程

- [ ] 页面已经进入页面清单。
- [ ] 页面入口明确。
- [ ] 用户流程闭环。
- [ ] 用户动作后有结果承接。

### 数据和技术结构

- [ ] 数据字段已经进入数据规范。
- [ ] 文件职责已经进入技术结构。
- [ ] 禁止事项已经写清楚。

### 任务边界

- [ ] 当前任务已经进入任务清单。
- [ ] 当前任务只做一件事。
- [ ] 允许修改范围明确。
- [ ] 禁止修改范围明确。

## 2. Agent 输入检查

- [ ] 当前任务编号。
- [ ] 当前任务目标。
- [ ] 必须阅读的文档。
- [ ] 允许修改的文件。
- [ ] 禁止修改的文件和能力。
- [ ] 验收标准。
- [ ] 完成后输出要求。

## 3. 实现后检查

- [ ] Agent 只完成当前任务。
- [ ] Agent 只修改允许文件。
- [ ] Agent 没有新增未经文档批准的能力。
- [ ] 实现符合 PRD。
- [ ] 实现符合页面清单。
- [ ] 实现符合用户流程。
- [ ] 实现符合数据规范。
- [ ] 代码能回溯到文档。
- [ ] 用户能完成核心流程。

## 4. 最终结论

```text
是否通过：通过 / 有条件通过 / 不通过
原因：
必须修复：
可选优化：
是否可以进入下一任务：是 / 否
```

## 5. 一句话原则

```text
需求没有进文档，就不能进代码。
代码不能回溯到文档，就不能通过验收。
```
