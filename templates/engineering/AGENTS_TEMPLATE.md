# Agent 执行规则模板

## 1. 核心原则

```text
文档工程是控制层，Agent 是执行层。
```

Agent 必须在文档和任务边界内执行。

## 2. 执行规则

1. 每次只完成一个任务。
2. 修改前必须阅读相关文档。
3. 当前任务必须来自任务清单。
4. 不允许修改无关文件。
5. 不允许引入无关依赖。
6. 不允许写入密钥、token、账号、密码。
7. 不允许新增文档没有定义的页面、字段、功能或平台能力。
8. 不允许扩大任务范围。
9. 发现需求不清楚时，先返回问题，不要自行猜测。
10. 完成后必须说明修改内容、验证方式、风险点。

## 3. 完成后输出

```text
修改文件：
实现说明：
对应文档：
验证方式：
风险点：
后续建议：
```

## 4. 自检问题

- [ ] 是否只完成当前任务？
- [ ] 是否只修改允许文件？
- [ ] 是否符合 PRD？
- [ ] 是否符合页面清单？
- [ ] 是否符合用户流程？
- [ ] 是否符合数据规范？
- [ ] 是否满足验收标准？
