# API 契约模板

## 1. API 目标

本文件用于约束前端和后端如何通信。

AI 不得自行发明接口路径、请求参数、响应字段或错误码。

## 2. 基础约定

Base URL：

```text
/api
```

数据格式：JSON

认证方式：无 / Cookie / Bearer Token / 其他

分页方式：page + size / cursor / 不分页

## 3. 接口列表

| 编号 | 方法 | 路径 | 用途 | 调用页面 | 对应功能 |
|---|---|---|---|---|---|
| API-001 | GET | /api/places | 获取地点列表 | 首页 | F001 |

## 4. 请求定义

```text
API 编号：
方法：
路径：
Query 参数：
Path 参数：
Body：
```

## 5. 响应定义

```json
{
  "code": "OK",
  "message": "success",
  "data": {}
}
```

## 6. 字段定义

| 字段 | 类型 | 是否必填 | 来源数据规范 | 说明 |
|---|---|---|---|---|

## 7. 错误码

| 错误码 | HTTP 状态 | 说明 | 前端展示 |
|---|---:|---|---|

## 8. Mock 规则

第一版如果后端未完成，前端使用 Mock。

Mock 字段必须和 API 响应字段一致。

## 9. AI 禁止行为

AI 不得：

- 自行新增接口
- 自行改变字段类型
- 自行改变响应结构
- 自行新增前端不使用的接口
- 自行绕过统一错误响应

## 10. 验收标准

- [ ] API 来自 PRD 或任务包
- [ ] 字段来自数据规范
- [ ] 前端调用和后端响应一致
- [ ] 错误码可被前端展示
- [ ] Mock 和真实 API 字段一致
