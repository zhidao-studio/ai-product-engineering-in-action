## 3. 变更协议

任何变更必须先判断：

```text
这是 bug？
这是体验问题？
这是范围变更？
这是新需求？
```

不同类型走不同流程。

## 4. 反馈分类

| 反馈类型 | 处理方式 |
|---|---|
| bug | 进入修复任务 |
| 体验问题 | 进入体验优化任务 |
| 范围变更 | 先更新产品定义或 PRD |
| 新需求 | 进入下一版本评估 |
