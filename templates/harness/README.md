# Harness 模板

本目录提供可复用的文档模板。

模板目标不是写漂亮文档，而是让 AI 产品交付过程可约束、可验收、可追溯。

## 模板清单

| 模板 | 作用 | 填写示例 |
|---|---|---|
| `01_PRODUCT_DEFINITION_HARNESS.md` | 产品定义 Harness | `examples/01_minimal_harness/01_PRODUCT_DEFINITION.md` |
| `02_DESIGN_SPEC_HARNESS.md` | 设计规格 Harness | `examples/01_minimal_harness/02_DESIGN_SPEC.md` |
| `03_ENGINEERING_SPEC_HARNESS.md` | 工程规格 Harness | `examples/01_minimal_harness/03_ENGINEERING_SPEC.md` |
| `04_TASK_EXECUTION_HARNESS.md` | 任务执行 Harness | `examples/01_minimal_harness/04_TASK_EXECUTION.md` |
| `05_VALIDATION_HARNESS.md` | 验证 Harness | `examples/01_minimal_harness/05_VALIDATION.md` |
| `06_ACCEPTANCE_HARNESS.md` | 体验验收 Harness | `examples/01_minimal_harness/06_ACCEPTANCE.md` |
| `07_DELIVERY_ITERATION_HARNESS.md` | 交付迭代 Harness | `examples/01_minimal_harness/07_DELIVERY_ITERATION.md` |

## 使用方式

1. 先读 `examples/01_minimal_harness/`，看一套填好的最小示例。
2. 再复制本目录下的模板。
3. 替换为自己的项目内容。
4. 交给 AI / Agent 之前先跑 checklist。
5. AI 交付后用对应 checklist 验收。

## 模板和示例的关系

```text
模板 = 空白表单，告诉你要填什么。
示例 = 填好的参考，告诉你大概填到什么程度。
checklist = 判断是否可以进入下一步。
```

## 原则

```text
模板不是记录结果，
而是约束 AI 接下来能做什么。
```
