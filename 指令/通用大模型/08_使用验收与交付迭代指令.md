# 使用验收与交付迭代指令

## 使用场景

当验证约束已经确认，需要定义用户如何验收、后续如何交付和迭代时，使用本指令。

本指令只生成使用验收和交付迭代约束。

## 输入材料

```text
已确认的项目文档：
【粘贴产品定义、界面规格、工程规格、接口契约、数据库结构、代码任务包、验证约束】
```

## 指令正文

```text
请根据已确认的项目文档，起草使用验收和交付迭代约束。

必须输出两份内容：

A. 使用验收
1. 用户角色
2. 核心使用流程
3. 使用前置条件
4. 空状态和异常状态验收
5. 多设备检查
6. 用户确认项

B. 交付迭代
1. 交付条件
2. 上线前检查
3. 反馈分类规则
4. 变更协议
5. 迭代记录表

要求：
- 使用验收必须让非技术人能照着完成
- 不只检查页面能否打开，要检查核心流程是否闭环
- 新需求、缺陷、体验问题、范围变化必须分类处理
- 任何变更必须先判断是否需要更新项目文档

最后请输出：
“请用户确认使用验收和交付迭代约束。确认后，可以进入逐任务代码执行。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/08_使用验收.md
docs/09_交付迭代.md
```
