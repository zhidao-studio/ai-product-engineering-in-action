# 验证约束指令

## 使用场景

当代码任务包已经确认，需要定义每个任务如何判断完成时，使用本指令。

本指令只生成验证约束，不写代码。

## 输入材料

```text
已确认的项目文档：
【粘贴产品定义、界面规格、工程规格、接口契约、数据库结构】

已确认的代码任务包：
【粘贴 docs/06_代码任务包.md】
```

## 指令正文

```text
请根据已确认的项目文档和代码任务包，起草验证约束。

必须输出：

1. 功能检查表
2. 界面检查表
3. 数据检查表
4. 工程检查表
5. 接口检查表
6. 追溯检查表
7. 不通过时应该回到哪份文档修

要求：
- 每个检查项必须能对应到一份约束文档
- 不允许用“看起来完成了”作为通过标准
- 必须区分前端、后端、接口、数据库和基础设施检查
- 必须说明哪些检查由 AI 执行，哪些需要用户确认

最后请输出：
“请用户确认验证约束。确认后，再进入使用验收和交付迭代约束。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/07_验证.md
```
