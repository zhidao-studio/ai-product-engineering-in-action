# 工程规格约束指令

## 使用场景

当产品定义和界面规格都已经确认，需要约束工程实现边界时，使用本指令。

本指令只生成第 3 层：工程规格约束。

## 输入材料

```text
已确认的产品定义：
【粘贴 docs/01_产品定义.md】

已确认的界面规格：
【粘贴 docs/02_界面规格.md】

默认工程基线：
【粘贴或引用 文档/工程基线/01_工程基线总说明.md】
```

## 指令正文

```text
请只根据已确认的产品定义、界面规格和工程基线，起草“工程规格约束”。

必须输出以下内容：

1. 前端范围
2. 后端范围
3. 是否需要数据库
4. 核心数据字段
5. 页面和组件拆分
6. 接口或 Mock 方案
7. 技术边界
8. 第一版不做的工程能力
9. 需要用户确认的问题

要求：
- 不要自行更换技术栈
- 不要引入 Spring Cloud
- 不要新增产品定义外的功能
- 不要直接写代码
- 不要生成代码任务包

最后请输出：
“请用户确认工程规格。确认后，如果需要后端，再进入接口契约和数据库结构。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/03_工程规格.md
```
