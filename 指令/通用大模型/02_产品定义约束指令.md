# 产品定义约束指令

## 使用场景

当你只有一个产品想法，需要先锁定产品边界时，使用本指令。

本指令只生成第 1 层：产品定义约束。

不要生成界面规格、工程规格、代码任务包或验收内容。

## 输入材料

```text
产品想法：
【填写产品想法】

补充背景：
【填写已有背景，没有就写“暂无”】
```

## 指令正文

```text
请根据输入材料，只起草“产品定义约束”。

必须输出以下内容：

1. 产品一句话说明
2. 目标用户
3. 用户真实问题
4. 第一版做什么
5. 第一版不做什么
6. 成功标准
7. 需要用户确认的问题

要求：
- 不要写界面设计
- 不要写技术方案
- 不要写接口和数据库
- 不要写代码任务
- 不要自行扩大第一版范围

最后请输出：
“请用户确认产品定义。确认后，再进入界面规格约束。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/01_产品定义.md
```
