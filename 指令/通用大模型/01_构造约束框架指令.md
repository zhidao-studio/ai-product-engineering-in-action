# 逐层构造约束框架总指令

## 使用场景

当你只有一个产品想法，还没有任何项目约束文档时，先使用本指令。

本指令不要求 AI 一次性生成 7 层约束。

本指令只用于生成逐层执行计划，并明确下一步应该从“产品定义约束”开始。

## 核心规则

```text
不要一次性生成 7 层约束框架。
必须逐层推进。
每一层必须以上一层已确认产物作为输入。
没有确认上一层，不进入下一层。
```

## 指令正文

```text
我现在有一个产品想法：
【填写产品想法】

请不要直接生成完整 7 层约束框架。

请先输出一个逐层构造计划，告诉我接下来应该按什么顺序生成以下文档：

1. 产品定义约束
2. 界面规格约束
3. 工程规格约束
4. 接口契约和数据库结构
5. 任务执行约束 / 代码任务包
6. 验证约束
7. 使用验收和交付迭代约束

输出要求：
- 说明每一步的输入是什么
- 说明每一步的输出文件是什么
- 说明每一步需要我确认什么
- 只开始第 1 步，不生成后续层正文

最后请提醒我：下一步应使用“02_产品定义约束指令.md”。
```

## 使用原则

```text
逐层生成，逐层确认，逐层进入下一步。
```
