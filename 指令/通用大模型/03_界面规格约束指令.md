# 界面规格约束指令

## 使用场景

当产品定义已经确认，需要约束页面和体验表达时，使用本指令。

本指令只生成第 2 层：界面规格约束。

## 输入材料

```text
已确认的产品定义：
【粘贴 docs/01_产品定义.md】

参考截图或参考网站：
【可选。粘贴截图说明、图片链接、参考网站 URL 或写“暂无”】

不喜欢的设计：
【可选。说明需要避免的风格】
```

## 指令正文

```text
请只根据已确认的产品定义，起草“界面规格约束”。

必须输出以下内容：

1. 页面清单
2. 每个页面的模块
3. 页面状态：加载中、空状态、成功状态、错误状态
4. 视觉风格
5. 配色和布局说明
6. 组件样式说明
7. 参考截图或参考网站如何约束设计
8. 需要用户确认的问题

要求：
- 不要写前端技术实现
- 不要写后端接口
- 不要写数据库
- 不要新增产品定义中没有的功能
- 如果没有参考截图或参考网站，必须提示用户补充参考输入

最后请输出：
“请用户确认界面规格。确认后，再进入工程规格约束。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/02_界面规格.md
```
