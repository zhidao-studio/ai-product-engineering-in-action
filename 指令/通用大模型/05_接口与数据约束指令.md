# 接口与数据约束指令

## 使用场景

当工程规格已经确认，并且项目需要后端或持久化数据时，使用本指令。

本指令只生成接口契约和数据库结构。

## 输入材料

```text
已确认的产品定义：
【粘贴 docs/01_产品定义.md】

已确认的界面规格：
【粘贴 docs/02_界面规格.md】

已确认的工程规格：
【粘贴 docs/03_工程规格.md】
```

## 指令正文

```text
请根据已确认的产品定义、界面规格和工程规格，起草接口契约和数据库结构。

必须输出两份内容：

A. 接口契约
1. 接口列表
2. 请求方法和路径
3. 请求参数
4. 响应结构
5. 错误码
6. 前端调用页面

B. 数据库结构
1. 数据模型列表
2. 表结构
3. 字段类型
4. 必填规则
5. 唯一约束
6. 创建时间和更新时间
7. 是否需要软删除

要求：
- 接口字段必须来自工程规格中的数据字段
- 数据库字段必须能支撑接口契约
- 不要新增产品定义外的功能
- 不要设计复杂权限系统，除非产品定义明确要求
- 不要写代码

最后请输出：
“请用户确认接口契约和数据库结构。确认后，再进入代码任务包。”
```

## 保存位置

将结果保存为目标项目中的：

```text
docs/04_接口契约.md
docs/05_数据库结构.md
```
