# 检查清单

本目录存放可执行 checklist。

## 目录说明

```text
checklists/harness/        harness 完整性检查
checklists/code_delivery/  工程基线、前端、后端、API、基础设施检查
```

## 使用方式

1. 写完 harness 后，先跑 `checklists/harness/`。
2. 准备进入代码交付前，再跑 `checklists/code_delivery/`。
3. 不通过时先补模板或缩小任务范围。

## 原则

```text
先 checklist，后行动。
先判断，后修改。
```
