# API 契约检查清单

## 1. API 定义

- [ ] 每个接口有编号
- [ ] 每个接口有方法
- [ ] 每个接口有路径
- [ ] 每个接口有用途
- [ ] 每个接口有调用页面
- [ ] 每个接口有关联功能

## 2. 请求和响应

- [ ] 请求参数明确
- [ ] 响应结构明确
- [ ] 响应字段来自数据规范
- [ ] 字段类型一致
- [ ] 错误码明确
- [ ] 前端错误展示明确

## 3. Mock 和真实接口

- [ ] Mock 字段和真实响应字段一致
- [ ] 前端可以从 Mock 平滑切换到真实 API
- [ ] 没有前端不用的接口
- [ ] 没有 PRD 不需要的接口

## 4. 验证

- [ ] 前端调用路径正确
- [ ] 后端返回结构正确
- [ ] 错误响应可被前端识别
- [ ] API 文档和代码一致
