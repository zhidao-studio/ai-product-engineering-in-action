# 数据库规范说明

本工程必须遵守：

```text
文档/工程基线/07_数据库设计规范.md
样板/代码交付/05_数据库结构样板.md
```

## 1. 默认技术选择

```text
MySQL
MyBatis-Plus
Flyway
```

如果项目使用其他数据库，必须在工程规格中说明差异。

## 2. 表命名

表名使用 snake_case，默认使用业务单数名。

推荐：

```text
video_task
wechat_user
user_role
role_permission
```

不推荐：

```text
VideoTask
videoTasks
t_video_task
tbl_video_task
```

## 3. 字段命名

数据库字段使用 snake_case。

Java、API、前端字段使用 lowerCamelCase。

示例：

```text
数据库字段：script_summary
Java 字段：scriptSummary
API 字段：scriptSummary
前端字段：scriptSummary
```

## 4. 主键和时间字段

业务主表默认包含：

```text
id
created_at
updated_at
```

Java Entity 对应：

```text
Long id
LocalDateTime createdAt
LocalDateTime updatedAt
```

## 5. 状态字段

状态字段使用：

```text
status
```

状态值必须在接口契约或数据库结构文档中说明。

推荐值：

```text
draft
pending
generating
completed
failed
```

## 6. 索引和唯一约束

普通索引：

```text
idx_表名_字段名
```

唯一索引：

```text
uk_表名_字段名
```

索引必须服务于明确查询场景。

唯一约束必须来自明确业务规则。

## 7. Flyway

所有表结构变更必须通过 Flyway。

命名格式：

```text
V版本号__动作_业务对象.sql
```

示例：

```text
V1__create_video_task.sql
V2__add_status_to_video_task.sql
```

不得修改已经执行过的历史 migration。

## 8. Entity / BO / VO 映射

规则：

- Entity 对应数据库表。
- BO 对应业务入参。
- VO 对应接口返回。
- Entity 不直接返回给前端。
- VO 只暴露接口契约允许的字段。

## 9. AI 禁止事项

AI 不得：

- 未更新数据库结构文档就新增表。
- 未写 Flyway migration 就改表结构。
- 直接修改已经执行过的 migration。
- 使用中文字段名。
- 混用数据库 camelCase 和 snake_case。
- 使用 FLOAT / DOUBLE 存金额。
- 把 Entity 直接返回给前端。
