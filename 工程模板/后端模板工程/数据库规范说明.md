# 数据库规范说明

本工程必须遵守：

```text
文档/工程基线/07_数据库设计规范.md
样板/代码交付/05_数据库结构样板.md
```

## 1. 默认技术选择

```text
MySQL
MyBatis-Plus
Flyway
```

如果项目使用其他数据库，必须在工程规格中说明差异。

## 2. 每表 7 要素

每张业务表默认必须包含：

```text
id
created_by
created_at
updated_by
updated_at
is_valid
is_deleted
```

含义：

- `id`：32 位字符串主键，数据库不自增。
- `created_by`：创建人，32 位字符串用户 ID，系统默认 32 位 0。
- `created_at`：创建时间，默认 trigger 维护。
- `updated_by`：更新人，32 位字符串用户 ID，系统默认 32 位 0。
- `updated_at`：更新时间，默认 trigger 维护。
- `is_valid`：是否有效，默认 1。
- `is_deleted`：是否删除，默认 0。

## 3. 主键规范

主键字段统一为：

```text
id CHAR(32) PRIMARY KEY
```

Java Entity 使用：

```text
String id
```

主键由应用层生成 32 位字符串，数据库只负责存储。

推荐使用 32 位无横线 UUID。

不得使用：

```text
BIGINT AUTO_INCREMENT
Long id
IdType.AUTO
```

## 4. 表命名

表名使用 snake_case，默认使用业务单数名。

推荐：

```text
video_task
wechat_user
user_role
role_permission
```

不推荐：

```text
VideoTask
videoTasks
t_video_task
tbl_video_task
```

## 5. 字段命名

数据库字段使用 snake_case。

Java、API、前端字段使用 lowerCamelCase。

示例：

```text
数据库字段：script_summary
Java 字段：scriptSummary
API 字段：scriptSummary
前端字段：scriptSummary
```

## 6. 时间字段 trigger

时间字段使用：

```text
created_at
updated_at
```

默认由数据库 trigger 维护：

- insert 时填充 `created_at` 和 `updated_at`。
- update 时刷新 `updated_at`。

trigger 命名：

```text
trg_表名_bi
trg_表名_bu
```

示例：

```text
trg_video_task_bi
trg_video_task_bu
```

## 7. 有效和删除字段

默认字段：

```text
is_valid
is_deleted
```

业务查询默认过滤：

```text
is_valid = 1
is_deleted = 0
```

不得再使用旧字段名：

```text
deleted
valid
```

除非历史项目已有明确约定。

## 8. 多表关联

多表查询默认使用主键 `id` 关联。

关联字段使用：

```text
业务对象_id
```

关联字段类型默认使用：

```text
CHAR(32)
```

示例：

```text
user_id -> user.id
role_id -> role.id
video_task_id -> video_task.id
created_by -> user.id
updated_by -> user.id
```

不得默认使用名称、手机号、openid、业务描述字段做关联。

## 9. 状态字段

状态字段使用：

```text
status
```

状态值必须在接口契约或数据库结构文档中说明。

推荐值：

```text
draft
pending
generating
completed
failed
```

## 10. 索引和唯一约束

普通索引：

```text
idx_表名_字段名
```

唯一索引：

```text
uk_表名_字段名
```

索引必须服务于明确查询场景。

唯一约束必须来自明确业务规则。

## 11. Flyway

所有表结构变更必须通过 Flyway。

命名格式：

```text
V版本号__动作_业务对象.sql
```

示例：

```text
V1__create_video_task.sql
V2__add_status_to_video_task.sql
```

不得修改已经执行过的历史 migration。

包含 trigger 的 migration 必须在模板验证阶段确认可执行。

## 12. Entity / BO / VO 映射

规则：

- Entity 对应数据库表。
- BO 对应业务入参。
- VO 对应接口返回。
- Entity 不直接返回给前端。
- VO 只暴露接口契约允许的字段。
- 主键 `id` 在 Entity、VO、接口路径参数中统一使用 String。

## 13. AI 禁止事项

AI 不得：

- 新建业务表却缺少 7 要素。
- 使用自增 BIGINT 主键。
- 使用 `Long id` 表达业务主键。
- 未写 Flyway migration 就改表结构。
- 使用中文字段名。
- 混用数据库 camelCase 和 snake_case。
- 使用非主键字段作为默认表关联字段。
- 把 Entity 直接返回给前端。
