# 微信小程序登录样板说明

## 1. 定位

本文件说明后端如何为微信小程序登录预留标准结构。

这是样板说明，不包含真实 appid、secret，也不直接写死微信接口配置。

## 2. 推荐流程

```text
小程序端 wx.login 获取 code
  ↓
小程序调用 POST /api/v1/wechat/session
  ↓
后端使用 code 换取 openid / unionid
  ↓
后端创建或查询本地用户
  ↓
后端创建登录态
  ↓
小程序后续请求携带登录态
```

## 3. 推荐后端分层

```text
controller/v1/WechatSessionController.java

domain/wechat/bo/WechatLoginBo.java
domain/wechat/vo/WechatLoginVo.java
domain/wechat/entity/WechatUser.java

service/IWechatSessionService.java
service/impl/WechatSessionServiceImpl.java

mapper/WechatUserMapper.java
```

## 4. 接口建议

```text
POST /api/v1/wechat/session
```

请求：

```text
code：小程序 wx.login 返回的临时 code
```

响应：

```text
user：当前登录用户基本信息
```

是否返回 token 或设置 cookie，应由项目认证方案决定。

## 5. 配置要求

后端不得提交真实微信密钥。

微信相关配置必须来自环境变量或密钥配置：

```text
WECHAT_APP_ID
WECHAT_APP_SECRET
```

## 6. 安全要求

- 不在前端保存微信密钥。
- 不在 GitHub 提交微信密钥。
- 不把 openid 直接作为业务权限判断依据。
- 登录态必须有有效期。
- 失败时不暴露微信接口原始错误细节。

## 7. AI 修改边界

AI 不得直接写死微信接口地址、appid、secret 或云托管域名。

AI 只能在接口契约明确后生成对应 Controller、Service、BO、VO、Entity、Mapper 和测试。
