# 后端命名规范说明

本工程必须遵守：

```text
文档/工程基线/06_前后端命名规范.md
```

## 1. Controller

使用业务名 + Controller：

```text
VideoTaskController
WechatSessionController
```

路径使用 `/api/v1` 前缀和小写 kebab-case：

```text
/api/v1/video-tasks
/api/v1/wechat/session
```

## 2. Entity

Entity 使用业务名：

```text
VideoTask
WechatUser
```

Entity 只表示数据库实体，不直接返回给前端。

## 3. BO

BO 使用动作 + 业务名 + Bo：

```text
CreateVideoTaskBo
UpdateVideoTaskBo
WechatLoginBo
```

## 4. VO

VO 使用业务名 + Vo：

```text
VideoTaskVo
UserVo
WechatLoginVo
```

## 5. Mapper

Mapper 使用业务名 + Mapper：

```text
VideoTaskMapper
WechatUserMapper
```

## 6. Service

Service 接口使用 I + 业务名 + Service：

```text
IVideoTaskService
IWechatSessionService
```

实现类使用业务名 + ServiceImpl：

```text
VideoTaskServiceImpl
WechatSessionServiceImpl
```

## 7. 数据库

表名和字段名使用 snake_case：

```text
video_task
script_summary
created_at
updated_at
```

索引命名：

```text
idx_video_task_status
uk_user_username
```

## 8. 禁止

不得为同一业务概念新增多个英文名。

不得让 Controller 直接依赖 Service 实现类。

不得把 Entity 直接返回给前端。
