# 认证权限成熟化说明

## 1. 定位

后端模板当前只提供最小认证样板。

真实项目如果进入多人使用、长期使用或外部用户使用阶段，必须补认证和权限成熟化设计。

## 2. 当前模板已有能力

```text
AuthController
IAuthService
AuthServiceImpl
LoginBo
LoginVo
UserVo
统一错误码
统一异常处理
```

这些只用于约束分层，不代表完整安全方案。

## 3. 真实项目必须补充

### 登录安全

- 密码哈希，不存明文密码。
- 登录失败次数限制。
- 登录态有效期。
- 登出机制。
- 401 统一处理。

### 会话策略

可选方案：

```text
HttpOnly Cookie + SameSite + CSRF 防护
Bearer Token + Refresh Token
小程序 session token
```

具体方案必须写入接口契约和安全约束。

### 权限模型

最小权限模型建议：

```text
User
Role
Permission
UserRole
RolePermission
```

第一版内部工具可以先用简单角色：

```text
admin
operator
viewer
```

### 接口鉴权

必须明确：

- 哪些接口不需要登录。
- 哪些接口需要登录。
- 哪些接口需要角色权限。
- 权限失败返回 FORBIDDEN。

## 4. 不建议第一版直接引入

除非项目明确需要，不建议第一版直接引入：

- Spring Cloud Gateway
- OAuth2 全套授权服务器
- 多租户权限平台
- 复杂菜单权限系统
- 动态数据权限系统

## 5. AI 修改边界

AI 不得自行决定认证方案。

AI 不得自行新增权限模型。

AI 只有在接口契约、安全约束和代码任务包明确后，才可以实现认证和权限模块。
