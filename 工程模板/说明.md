# 工程模板

本目录存放可复制使用的工程模板。

工程模板不是文档样板，而是被 AI 执行业务代码时使用的工程骨架。

## 1. 当前模板

```text
工程模板/Web后台模板工程/      Vue 3 + Vite + TypeScript + Element Plus
工程模板/H5模板工程/          Vue 3 + Vite + TypeScript + Vant
工程模板/微信小程序模板工程/  原生微信小程序 + TypeScript + Vant Weapp
工程模板/iOS模板工程/         SwiftUI + MVVM + CocoaPods
工程模板/后端模板工程/        Spring Boot + MyBatis-Plus + Flyway 单体后端
```

## 2. 使用原则

```text
先写项目约束文档，
再生成高保真交互预览，
再由使用者视觉确认，
再复制当前版本需要的工程模板，
再让 AI 在模板工程内按任务包开发。
```

AI 不应从 0 创建工程，也不应删除模板工程中的基础结构、统一错误处理、统一响应、检查命令和测试样例。

## 3. 多端原则

Web 后台、H5、微信小程序、iOS 是多个独立工程。

不建立前端共享层。

多端一致性通过接口契约、产品定义、界面规格、工程规格、代码任务包和检查清单保持。

## 4. 使用检查

复制模板前后应检查：

```text
文档/工程基线/03_工程模板验证记录.md
检查清单/工程模板/
检查清单/代码交付/01_页面原型确认清单.md
scripts/check-ai-guard.sh
```

如果模板状态是“未验证”或“缺件已补齐待验证”，不得宣称该模板生产可用。

## 5. 后续增强方向

```text
H5 端端到端测试
微信小程序 sitemap 和 CI 上传配置
iOS 完整 Xcode 工程或 XcodeGen/Tuist 生成方案
模板真实运行验证记录补齐
```
