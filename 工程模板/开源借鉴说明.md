# 开源借鉴说明

## 1. 目的

工程模板不是从零拍脑袋设计，而是借鉴成熟开源项目的工程结构和约束机制，再裁剪成适合 AI 受控开发的轻量模板。

原则：

```text
借骨架，不搬平台。
借机制，不搬业务。
借约束，不搬复杂度。
```

## 2. 前端借鉴来源

主要参考：

```text
vue-element-plus-admin
```

借鉴内容：

- Vue 3 + Vite + TypeScript + Element Plus 技术路线
- 后台系统 Layout 思路
- 路由 meta 思路
- lint / format / typecheck / build 命令体系
- mini 模板思路：只保留通用后台能力，不引入复杂业务模块

不直接搬入：

- 大量示例页面
- 国际化完整体系
- 图表、富文本、代码编辑器等重型依赖
- 动态权限完整体系
- 标签页、多主题等复杂后台平台能力

## 3. 后端借鉴来源

主要参考：

```text
RuoYi-Vue3
RuoYi-Vue-Plus
```

借鉴内容：

- Spring Boot 单体后台工程思路
- 统一响应对象
- 错误码枚举
- 全局异常处理
- 参数校验
- 依赖版本集中管理
- 认证和权限作为可选模块的设计思路

不直接搬入：

- 多租户
- 工作流
- OSS
- 短信
- 动态数据源
- Redis / Redisson
- 部门、岗位、菜单、字典等完整后台业务模块
- 代码生成平台能力

## 4. 当前模板定位

当前模板只做内部生产力工具 MVP 所需的最小工程骨架。

默认不追求大而全后台平台。

如果具体项目需要权限、字典、文件上传、操作日志等能力，应作为可选模块逐步引入，并先更新项目约束文档和代码任务包。
