# 微信小程序云托管调用说明

## 1. 定位

本文件说明微信小程序如何调用后端服务。

小程序是独立工程，不引用 Web/H5 代码。

后端地址、云托管服务访问方式和域名配置，以微信云托管当前控制台和官方文档为准。

## 2. 调用原则

```text
接口契约优先
配置集中管理
页面不写死后端地址
错误处理统一
登录态统一
```

## 3. 推荐目录

```text
miniprogram/
  config/
    api.ts
  services/
    request.ts
    sessionService.ts
  types/
    api.ts
    session.ts
```

## 4. 配置要求

后端服务地址必须集中配置。

不得在页面中直接写：

```text
https://xxx.example.com/api/v1/...
```

应通过配置文件或环境配置统一管理。

## 5. Request 封装要求

统一请求封装必须处理：

- API 前缀
- 请求头
- 登录态
- 统一响应格式
- 业务错误码
- 网络错误
- 401 登录失效

## 6. 接口路径

接口路径以：

```text
目标项目/docs/04_接口契约.md
```

为准。

默认后端接口前缀：

```text
/api/v1
```

## 7. 云托管访问方式

如果当前微信云托管环境支持小程序到后端服务的内网访问，优先使用内网访问。

如果需要公网域名，必须按微信小程序服务器域名、HTTPS 和云托管域名规则配置。

不要在模板中写死具体云托管域名。

## 8. AI 修改边界

AI 可以新增业务 service 和类型定义。

AI 不得：

- 在页面中散落请求逻辑。
- 在页面中写死后端地址。
- 自行发明接口路径。
- 绕过统一 request 封装。
- 绕过接口契约。
