# 前端多端基座说明

## 1. 目的

本文件定义 Web 后台、H5、微信小程序、iOS 多端模板的共同基座。

前端多端基座不能闭门造车，必须参考成熟开源工程的分层和构建体系。

## 2. 参考来源

主要参考：

```text
Vue Vben Admin
vue-element-plus-admin
Vant
Taro
Vant Weapp
SwiftUI / TCA 思路
```

## 3. 借鉴原则

```text
借构建体系，不搬复杂平台。
借分层结构，不搬业务模块。
借共享边界，不让多端互相污染。
```

## 4. 成熟项目启发

### Vue Vben Admin

借鉴点：

- pnpm workspace / monorepo 思路
- apps 与 packages 分离
- build / dev / typecheck / lint / test 命令体系
- 可按应用筛选构建
- 公共配置沉淀为 package

不直接照搬：

- turbo 全量体系
- 多 UI 适配平台
- 文档站、playground、复杂插件体系

### vue-element-plus-admin

借鉴点：

- Web 后台布局
- 路由 meta
- Element Plus 后台组件组织
- mini 模板思路

不直接照搬：

- 大量示例页面
- 动态权限完整体系
- 国际化、标签页、图表等复杂能力

### Vant

借鉴点：

- 移动端组件组织
- H5 表单、列表、弹窗、Toast、Loading 交互
- lint / test / build 命令体系

不直接照搬：

- 组件库源码
- 文档站点
- 复杂发布体系

### Taro

借鉴点：

- 跨端构建命令体系
- 多端目标隔离
- workspace 批量构建和测试思路

不直接照搬：

- 完整跨端框架能力
- 编译器和插件体系
- 多端复杂适配层

## 5. 推荐目标结构

真实产品项目建议采用：

```text
frontend/
  apps/
    web-admin/
    h5/
    weapp/
  packages/
    shared-api/
    shared-types/
    shared-tokens/
    shared-utils/
```

说明：

- Web 后台、H5、小程序分别是独立 app。
- API 类型、请求字段、错误码、设计 token、通用工具进入 shared packages。
- 不同端不得互相直接引用页面、组件和端专属 API。
- iOS 为原生工程，不直接放入 Node workspace，但要共享接口契约和设计 token 说明。

## 6. 共享层边界

允许进入共享层：

```text
API 路径常量
统一响应类型
错误码类型
业务 DTO 类型
设计 token
纯函数工具
字段校验规则
```

禁止进入共享层：

```text
Vue 组件
Element Plus 组件
Vant 组件
微信小程序页面
iOS View
端专属路由
端专属状态管理
```

## 7. 构建命令原则

每个端必须有独立命令：

```text
lint
typecheck
test
build
check
```

目标项目如采用 pnpm workspace，可增加：

```text
check:web
check:h5
check:weapp
check:all
```

但第一版模板不强制引入复杂 monorepo 工具。

## 8. AI 修改边界

AI 修改多端项目时必须遵守：

- 修改某一端页面，不得改其他端页面。
- 修改共享类型，必须同步检查所有调用端。
- 修改接口字段，必须先更新接口契约。
- 修改设计 token，必须说明影响哪些端。
- 不得把 Web 后台组件复制到 H5 或小程序。
- 不得把小程序端 API 写进 Web/H5。

## 9. 一句话原则

```text
共享 API、类型、设计 token 和纯工具；
隔离页面、组件、路由、状态和端能力。
```
