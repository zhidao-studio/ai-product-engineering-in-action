# 目标与边界

## 1. 目标一句话

```text
定义一套文档驱动的 AI 产品交付方法论，
提供可复用的约束框架、工程基线、代码交付模板和检查清单，
让非技术人能学会并实际使用。
```

## 2. 三个目标层级

| 优先级 | 目标 | 说明 |
|---:|---|---|
| 1 | 方法论目标 | 定义文档驱动的 AI 产品交付流程 |
| 2 | 工具化目标 | 提供可复用的约束框架、工程基线、代码交付模板和检查清单 |
| 3 | 教学目标 | 让非技术人能学会并实际使用 |

## 3. 目标读者

- 有产品想法但不会写代码的人
- 想用 AI 做产品但担心失控的人
- 产品经理、运营、设计师、内容创作者
- 会一点技术但缺少产品交付流程的人
- 希望从 0 建立前后端工程并用 AI 交付的人

## 4. 核心问题

如果没有约束框架和工程基线，AI 容易自行扩功能、自行定风格、自行选技术方案、自行搭工程目录、自行引入依赖，并最终产出能跑但不可维护的代码。

## 5. 当前不做什么

当前阶段不做：

- 不做某个具体产品的完整实现
- 不绑定某个具体 AI 工具
- 不以小程序、H5 或 App 样例作为主线
- 不引入 Spring Cloud
- 不引入复杂微服务和云原生架构

## 6. 最终交付物

```text
方法论文档
工程基线文档
可复用约束框架模板
可复用代码交付模板
可执行检查清单
AI / Agent / Review 提示词
填好的示例
```
