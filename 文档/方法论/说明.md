# 方法论

本目录用于定义“文档驱动的 AI 产品交付方法论”。

这里不放具体产品代码，也不围绕某个样例工程展开。

## 目录结构

| 文件 | 作用 |
|---|---|
| `01_目标与边界.md` | 定义目标、边界和读者 |
| `02_核心原则.md` | 定义核心原则 |
| `03_约束框架分层结构.md` | 定义约束框架分层结构，以及和代码交付的关系 |
| `04_如何构造约束框架.md` | 定义如何构造约束框架 |
| `05_覆盖检查.md` | 定义如何检查约束是否覆盖主要 AI 产出维度 |

## 方法论目标

```text
定义一套文档驱动的产品交付流程，
让 AI 从产品想法到代码交付的产出可约束、可验收、可追溯。
```

## 和工程基线的关系

```text
文档/方法论/    解释方法论和交付流程
文档/工程基线/  定义具体工程基线和质量红线
```

方法论只回答“为什么要这样组织交付”。

工程基线回答“AI 在什么工程规范里写代码”。
