# 仓库结构说明

## 1. 目的

本文件用于定义当前仓库的目录职责，避免同一类内容在多个目录重复维护。

## 2. 顶层目录职责

| 目录 | 职责 | 不应该放什么 |
|---|---|---|
| `文档/` | 方法论、操作手册、工程基线、控制系统、结构说明、教学化说明 | 空白样板、检查清单、指令、完整案例、真实工程代码 |
| `样板/` | 可复用空白样板、控制系统元数据样板 | 方法论解释、长篇教学内容、填好的案例、真实工程代码 |
| `工程模板/` | 可复制使用的 Web、H5、微信小程序、iOS、后端多类独立工程模板 | 方法论正文、案例正文、无关业务系统 |
| `检查清单/` | 可执行检查清单 | 方法论正文、样板正文、案例正文 |
| `指令/` | AI / Agent / Review 指令 | 方法论文档、样板、检查清单 |
| `scripts/` | AI 门禁脚本、本地 hook 安装脚本、CI 调用入口 | 方法论文档、业务代码、工程模板 |
| `.github/` | GitHub Actions 门禁工作流 | 方法论文档、业务代码、样板 |
| `案例/` | 填好的参考案例 | 空白样板、仓库方法论主文档 |

## 3. 文档目录职责

```text
文档/说明.md              文档导航
文档/仓库结构说明.md      仓库结构说明
文档/操作手册/            面向第一次使用者的操作流程
文档/方法论/              方法论
文档/工程基线/            工程基线、质量红线、模板验证、命名规范、数据库规范、码值规范
文档/控制系统/            系统总览、阶段状态机和控制系统说明
文档/教学化内容/          后续教学化内容
```

## 4. 样板目录职责

```text
样板/控制系统/    AI Context Pack、contracts、写权限策略、人话报告等控制元数据样板
样板/约束框架/    产品交付约束框架样板
样板/代码交付/    代码交付样板：接口、数据库、基础设施、任务包等
```

样板只回答：应该填写什么、AI 应该被什么约束。

## 5. 工程模板目录职责

```text
工程模板/Web后台模板工程/      Vue 3 + Vite + TypeScript + Element Plus 轻量后台模板
工程模板/H5模板工程/          Vue 3 + Vite + TypeScript + Vant 移动端 H5 模板
工程模板/微信小程序模板工程/  原生微信小程序 + TypeScript + Vant Weapp 模板
工程模板/iOS模板工程/         SwiftUI + MVVM 轻量 iOS 模板
工程模板/后端模板工程/        Spring Boot + MyBatis-Plus + Flyway 单体后端模板
工程模板/多端独立工程说明.md  多端前端独立工程原则
工程模板/开源借鉴说明.md      Web 后台工程模板的开源参考来源
工程模板/移动端借鉴说明.md    H5、微信小程序、iOS 的开源参考来源
```

工程模板只回答：AI 应该在什么工程骨架里开发。

只复制当前产品版本需要的端。

多个前端端是独立工程，不建立前端共享层。

## 6. 检查清单目录职责

```text
检查清单/约束框架/    约束框架完整性检查
检查清单/代码交付/    原型确认、前端、后端、接口、数据库、基础设施、模板适配检查
检查清单/工程模板/    多端工程模板和独立工程原则检查
检查清单/生产级/      非部署生产级边界检查
```

检查清单只回答：是否完整、是否可以进入下一步、哪里缺约束。

## 7. 指令目录职责

```text
指令/通用大模型/    产品定义、界面规格、原型预览、工程规格、接口数据、任务包等生成指令
指令/执行代理/      Agent 执行指令
指令/检查/          检查指令
```

指令只回答：如何让 AI 执行一个受约束的动作。

## 8. 门禁目录职责

```text
scripts/check-ai-guard.sh               单目标项目门禁入口
scripts/run-ai-guard-all.sh             全目标项目门禁入口
scripts/install-ai-guard-hook.sh        本地 pre-commit hook 安装脚本
.github/workflows/ai-guard.yml          GitHub Actions 门禁工作流
```

门禁只回答：AI 的修改是否可以进入下一步。

## 9. 案例目录职责

```text
案例/01_最小约束框架案例/          最小填好案例
案例/02_全栈生产力工具案例/        带 Web、后端、接口、数据库和登录的 MVP 案例
案例/03_AI视频任务管理端到端案例/  端到端跑通案例说明
```

案例只回答：样板填完以后大概长什么样，以及项目文档如何串联。

```text

需要引用时引用源文件，不复制一份新的。
```
