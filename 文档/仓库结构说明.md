# 仓库结构说明

## 1. 目的

本文件用于定义当前仓库的目录职责，避免同一类内容在多个目录重复维护。

## 2. 顶层目录职责

| 目录 | 职责 | 不应该放什么 |
|---|---|---|
| `文档/` | 方法论、工程基线、结构说明、教学化说明 | 空白模板、检查清单、提示词、完整示例 |
| `模板/` | 可复用空白模板 | 方法论解释、长篇教学内容、填好的示例 |
| `检查清单/` | 可执行检查清单 | 方法论正文、模板正文、示例正文 |
| `提示词/` | AI / Agent / Review 提示词 | 方法论文档、模板、检查清单 |
| `示例/` | 填好的参考示例 | 空白模板、仓库方法论主文档 |

## 3. 文档目录职责

```text
文档/说明.md              文档导航
文档/仓库结构说明.md      仓库结构说明
文档/方法论/              方法论
文档/工程基线/            工程基线和质量红线
文档/教学化内容/          后续教学化内容
```

`文档/` 不保存可复制模板、不保存检查清单、不保存提示词、不保存完整示例。

## 4. 模板目录职责

```text
模板/约束框架/    产品交付约束框架模板
模板/代码交付/    代码交付模板
```

模板只回答：应该填写什么、AI 应该被什么约束。

## 5. 检查清单目录职责

```text
检查清单/约束框架/    约束框架完整性检查
检查清单/代码交付/    工程基线和代码交付检查
```

检查清单只回答：是否完整、是否可以进入下一步、哪里缺约束。

## 6. 提示词目录职责

```text
提示词/通用大模型/    通用大模型起草提示词
提示词/执行代理/      Agent 执行提示词
提示词/审查/          审查提示词
```

提示词只回答：如何让 AI 执行一个受约束的动作。

## 7. 示例目录职责

```text
示例/01_最小约束框架示例/    最小填好示例
```

示例只回答：模板填完以后大概长什么样。

## 8. 当前唯一基线来源

当前工程技术基线只以这里为准：

```text
文档/工程基线/01_工程基线总说明.md
```

工程质量红线只以这里为准：

```text
文档/工程基线/02_工程质量红线.md
```

其他目录如需引用技术基线，只能引用这两个文件，不应重复维护另一套基线说明。

## 9. 当前唯一阅读入口

仓库总入口：

```text
README.md
```

文档入口：

```text
文档/说明.md
```

不要重新创建：

```text
docs/
templates/
checklists/
prompts/
examples/
```

## 10. 一句话原则

```text
一个内容类型只放一个地方；
需要引用时引用源文件，不复制一份新的。
```
