# 仓库结构说明

## 1. 目的

本文件用于定义当前仓库的目录职责，避免同一类内容在多个目录重复维护。

## 2. 顶层目录职责

| 目录 | 职责 | 不应该放什么 |
|---|---|---|
| `文档/` | 方法论、操作手册、工程基线、结构说明、教学化说明 | 空白样板、检查清单、指令、完整案例 |
| `样板/` | 可复用空白样板 | 方法论解释、长篇教学内容、填好的案例 |
| `检查清单/` | 可执行检查清单 | 方法论正文、样板正文、案例正文 |
| `指令/` | AI / Agent / Review 指令 | 方法论文档、样板、检查清单 |
| `案例/` | 填好的参考案例 | 空白样板、仓库方法论主文档 |

## 3. 文档目录职责

```text
文档/说明.md              文档导航
文档/仓库结构说明.md      仓库结构说明
文档/操作手册/            面向第一次使用者的操作流程
文档/方法论/              方法论
文档/工程基线/            工程基线和质量红线
文档/教学化内容/          后续教学化内容
```

`文档/` 不保存可复制样板、不保存检查清单、不保存指令、不保存完整案例。

## 4. 样板目录职责

```text
样板/约束框架/    产品交付约束框架样板
样板/代码交付/    代码交付样板
```

样板只回答：应该填写什么、AI 应该被什么约束。

## 5. 检查清单目录职责

```text
检查清单/约束框架/    约束框架完整性检查
检查清单/代码交付/    工程基线和代码交付检查
```

检查清单只回答：是否完整、是否可以进入下一步、哪里缺约束。

## 6. 指令目录职责

```text
指令/通用大模型/    通用大模型起草指令
指令/执行代理/      Agent 执行指令
指令/检查/          检查指令
```

指令只回答：如何让 AI 执行一个受约束的动作。

## 7. 案例目录职责

```text
案例/01_最小约束框架案例/      最小填好案例
案例/02_全栈生产力工具案例/    带前端、后端、接口、数据库和登录的 MVP 案例
```

案例只回答：样板填完以后大概长什么样，以及项目文档如何串联。

## 8. 当前唯一基线来源

当前工程技术基线只以这里为准：

```text
文档/工程基线/01_工程基线总说明.md
```

工程质量红线只以这里为准：

```text
文档/工程基线/02_工程质量红线.md
```

其他目录如需引用技术基线，只能引用这两个文件，不应重复维护另一套基线说明。

## 9. 当前唯一阅读入口

仓库总入口：

```text
README.md
```

文档入口：

```text
文档/说明.md
```

不要重新创建：

```text
docs/
templates/
checklists/
prompts/
examples/
模板/
提示词/
示例/
```

## 10. 一句话原则

```text
一个内容类型只放一个地方；
需要引用时引用源文件，不复制一份新的。
```
