# 阶段状态机

## 1. 目的

阶段状态机用于控制 AI 在不同阶段允许做什么、禁止做什么、必须等待谁确认。

它解决的问题是：

```text
AI 读到一个目标后，不能直接越级生成完整项目。
AI 必须先判断当前阶段，再输出当前阶段允许的产物。
```

## 2. 核心原则

```text
阶段先行，动作受限。
当前阶段未确认，不得生成后续阶段产物。
用户确认是阶段推进的唯一依据。
AI 只能在当前阶段允许的输出范围内工作。
```

## 3. 阶段列表

```text
idea
product_definition_draft
product_definition_confirmed
ui_spec_draft
prototype_generated
prototype_confirmed
engineering_spec_confirmed
contract_confirmed
task_package_confirmed
coding
verification
acceptance
delivered
```

## 4. 阶段定义

### 4.1 idea

含义：只有产品想法，还没有结构化产品定义。

允许输出：

```text
问题清单
选项清单
推荐方案
产品定义草案
```

禁止输出：

```text
工程规格
接口契约
数据库结构
代码任务包
完整案例目录
工程代码
```

进入下一阶段条件：

```text
生成产品定义草案。
用户确认产品范围或要求继续调整。
```

### 4.2 product_definition_draft

含义：已有产品定义草案，但尚未确认。

允许输出：

```text
产品定义修订建议
产品范围差异说明
待确认问题和选项
```

禁止输出：

```text
正式界面规格
工程规格
接口契约
数据库结构
代码任务包
工程代码
```

进入下一阶段条件：

```text
用户确认产品定义。
```

### 4.3 product_definition_confirmed

含义：产品定义已确认，可以进入界面规格。

允许输出：

```text
界面规格草案
页面清单
页面流程
状态说明
```

禁止输出：

```text
工程代码
数据库结构
完整后端接口实现
```

进入下一阶段条件：

```text
用户确认界面规格草案。
```

### 4.4 ui_spec_draft

含义：已有界面规格草案，但尚未生成高保真交互预览。

允许输出：

```text
高保真交互预览 HTML
原型说明
模拟数据
页面状态样例
```

禁止输出：

```text
正式工程代码
后端接口实现
数据库 migration
```

进入下一阶段条件：

```text
高保真交互预览可打开、可点击、可被使用者查看。
```

### 4.5 prototype_generated

含义：高保真交互预览已生成，但尚未确认。

允许输出：

```text
原型修改建议
页面差异说明
使用者确认清单
```

禁止输出：

```text
复制工程模板
正式编码
接口实现
数据库实现
```

进入下一阶段条件：

```text
使用者确认原型就是预期样子。
```

### 4.6 prototype_confirmed

含义：原型已确认，可以进入工程规格。

允许输出：

```text
工程规格
端类型选择
工程模板选择
技术边界说明
```

禁止输出：

```text
未确认工程规格前直接编码
```

进入下一阶段条件：

```text
工程规格确认。
```

### 4.7 engineering_spec_confirmed

含义：工程规格已确认，可以生成接口、数据库和任务包。

允许输出：

```text
接口契约
数据库结构
代码任务包草案
AI 写权限策略
```

禁止输出：

```text
未确认接口和数据库前直接编码
```

进入下一阶段条件：

```text
接口契约和数据库结构确认。
```

### 4.8 contract_confirmed

含义：接口契约和数据库结构已确认。

允许输出：

```text
代码任务包
允许修改范围
验证计划
```

禁止输出：

```text
超出任务包范围的代码修改
```

进入下一阶段条件：

```text
代码任务包确认。
```

### 4.9 task_package_confirmed

含义：代码任务包已确认，可以进入工程执行。

允许输出：

```text
按任务包修改代码
执行测试
输出验证结果
```

禁止输出：

```text
扩展任务范围
重建工程
修改保护文件
跳过检查清单
```

进入下一阶段条件：

```text
代码修改完成并进入验证。
```

### 4.10 coding

含义：AI 正在按任务包编码。

允许输出：

```text
代码 diff
修改说明
本地验证命令
```

禁止输出：

```text
未经批准修改保护文件
未经批准新增依赖
未经批准修改数据库基线
```

进入下一阶段条件：

```text
代码提交后自动验证开始。
```

### 4.11 verification

含义：自动化验证或人工检查正在执行。

允许输出：

```text
验证结果
失败原因
人话报告
修复选项
```

禁止输出：

```text
隐藏失败
忽略检查清单
未经确认连续重试超过限制
```

进入下一阶段条件：

```text
验证通过，或用户选择修复方案。
```

### 4.12 acceptance

含义：使用者验收阶段。

允许输出：

```text
变更摘要
功能演示说明
验收清单
回退选项
```

禁止输出：

```text
未经验收标记交付完成
```

进入下一阶段条件：

```text
使用者确认验收通过。
```

### 4.13 delivered

含义：当前迭代已交付。

允许输出：

```text
交付记录
变更记录
后续迭代建议
```

禁止输出：

```text
继续修改已交付内容但不生成新任务包
```

## 5. 阶段越级规则

AI 不得越级生成后续阶段文件。

示例：

```text
当前是 idea 阶段：不得生成工程规格、接口契约、数据库结构、代码任务包。
当前是 prototype_generated 阶段：不得复制工程模板或写正式代码。
当前是 engineering_spec_confirmed 阶段：不得跳过接口契约和数据库结构直接编码。
```

## 6. 阶段推进记录

每个目标项目应维护阶段记录：

```text
目标项目/AI_CONTEXT_PACK/00_当前阶段.md
```

记录内容：

```text
当前阶段
已确认文件
未确认文件
允许输出
禁止输出
下一步进入条件
```

## 7. 一句话原则

```text
AI 先判断阶段，再判断动作；阶段未确认，不得越级生成。
```
