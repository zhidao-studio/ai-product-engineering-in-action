# 从产品想法到第一个页面

## 1. 适用场景

你现在只有一个产品想法，还没有代码、没有工程、没有设计稿。

目标不是一次性做完整产品，而是先走到：

```text
产品定义已确认
界面规格已确认
高保真交互预览已确认
工程规格已确认
工程模板已复制
第一个页面任务包已确认
可以让 AI 在模板工程内写第一个页面
```

## 2. 你需要准备什么

最少准备一句话产品想法：

```text
我想做一个……，帮助……解决……问题。
```

最好再准备：

- 一个你喜欢的参考网站 URL
- 一张你喜欢的页面截图
- 一个你不喜欢的反面例子
- 你明确不想做的功能

## 3. 第一步：打开 AI 工具

打开你选择的大模型工具。

可以是 ChatGPT、Claude、Gemini、Kimi、通义、豆包，或其他你能稳定使用的模型。

第一阶段先不要打开编码 Agent。

原因：

```text
先写约束文档，再做交互预览确认，再复制工程模板，最后让 Agent 在模板工程内写代码。
```

## 4. 第二步：生成逐层计划

复制这个文件：

```text
指令/通用大模型/01_构造约束框架指令.md
```

把你的产品想法填进去。

AI 这一步只应该输出逐层构造计划，不应该直接生成 7 层完整文档。

## 5. 第三步：生成产品定义

复制这个文件：

```text
指令/通用大模型/02_产品定义约束指令.md
```

把 AI 输出保存到你的目标项目：

```text
目标项目/docs/01_产品定义.md
```

确认前，不进入下一步。

## 6. 第四步：生成界面规格

复制这个文件：

```text
指令/通用大模型/03_界面规格约束指令.md
```

输入：

```text
目标项目/docs/01_产品定义.md
参考截图或参考网站 URL
不喜欢的设计说明
```

把 AI 输出保存为：

```text
目标项目/docs/02_界面规格.md
```

确认前，不进入原型预览阶段。

## 7. 第五步：生成高保真交互预览

这一步用于在写正式代码前，用最低成本确认页面长什么样、交互顺不顺。

复制这个文件：

```text
指令/通用大模型/03_5_高保真交互预览指令.md
```

输入：

```text
目标项目/docs/01_产品定义.md
目标项目/docs/02_界面规格.md
参考截图或参考网站 URL
不喜欢的设计说明
```

让 AI 生成一个单文件 HTML 原型，并保存到：

```text
目标项目/prototypes/01_高保真交互预览.html
```

这个原型必须满足：

- 可以直接用浏览器打开
- 不接真实后端
- 不需要安装依赖
- 使用模拟数据
- 可以点击主要导航、按钮、表单、列表、详情、弹窗或状态切换
- 能看出页面布局、信息层级、视觉风格和主要交互

使用者必须打开这个 HTML 原型进行视觉确认。

如果发现不符合预期，先改原型，不要进入工程阶段。

确认标准：

```text
这个页面就是我要的方向。
主要布局已经对。
主要按钮和入口已经对。
主要交互流程已经对。
可以进入工程阶段。
```

确认后，将确认结果记录到：

```text
目标项目/docs/02_5_原型预览确认记录.md
```

## 8. 第六步：生成工程规格

复制这个文件：

```text
指令/通用大模型/04_工程规格约束指令.md
```

输入：

```text
目标项目/docs/01_产品定义.md
目标项目/docs/02_界面规格.md
目标项目/docs/02_5_原型预览确认记录.md
文档/工程基线/01_工程基线总说明.md
工程模板/说明.md
```

把 AI 输出保存为：

```text
目标项目/docs/03_工程规格.md
```

## 9. 第七步：如果需要后端，生成接口和数据库

如果产品需要后端或保存数据，复制这个文件：

```text
指令/通用大模型/05_接口与数据约束指令.md
```

把 AI 输出保存为：

```text
目标项目/docs/04_接口契约.md
目标项目/docs/05_数据库结构.md
```

如果第一版是纯前端，可以跳过这一步。

## 10. 第八步：复制工程模板

不要让 AI 从 0 创建工程。

根据工程规格复制对应模板：

```text
工程模板/Web后台模板工程/       →  目标项目/web-admin/
工程模板/后端模板工程/          →  目标项目/backend/
工程模板/H5模板工程/            →  目标项目/h5/
工程模板/微信小程序模板工程/    →  目标项目/weapp/
工程模板/iOS模板工程/           →  目标项目/ios/
```

只复制当前版本需要的端。

多个前端端是独立工程，不建立前端共享层。

复制后不要先改模板结构，先生成代码任务包。

## 11. 第九步：生成第一个代码任务包

复制这个文件：

```text
指令/通用大模型/06_任务包约束指令.md
```

要求 AI 先只生成第一个任务，例如：

```text
TASK-WEB-001：基于 Web 后台模板工程建立首页静态结构
```

把结果保存为：

```text
目标项目/docs/06_代码任务包.md
```

任务包必须写清楚：

- 基于哪个工程模板
- 原型确认记录是什么
- 允许修改哪些文件
- 不处理哪些内容
- 必须通过哪些检查命令
- 是否需要新增或更新测试

## 12. 第十步：开始让 AI 写第一个页面

这时才可以打开编码 Agent，例如 Codex、Claude Code、Cursor、其他 AI 编程工具。

给它的输入必须包括：

```text
目标项目/docs/01_产品定义.md
目标项目/docs/02_界面规格.md
目标项目/docs/02_5_原型预览确认记录.md
目标项目/docs/03_工程规格.md
目标项目/docs/06_代码任务包.md
文档/工程基线/01_工程基线总说明.md
文档/工程基线/02_工程质量红线.md
对应工程模板 README
指令/执行代理/工程任务指令.md
```

要求它：

```text
只执行当前任务包。
在模板工程内开发。
以原型预览确认为视觉和交互参考。
不重建工程。
不删除统一请求封装、错误处理、设计变量和基础配置。
完成后说明如何启动和验证。
```

## 13. 第十一步：验收第一个页面

复制检查清单：

```text
检查清单/约束框架/02_使用者检查清单.md
检查清单/代码交付/02_前端检查清单.md
```

检查：

- 页面是否符合产品定义
- 页面是否符合界面规格
- 页面是否符合原型预览确认记录
- 是否没有新增不做清单里的功能
- 是否基于工程模板开发
- 是否能运行
- 是否能通过构建

## 14. 一句话流程

```text
想法 → 产品定义 → 界面规格 → 高保真交互预览 → 使用者确认 → 工程规格 → 接口/数据 → 复制工程模板 → 任务包 → AI 在模板内写第一个页面 → 检查清单验收
```
