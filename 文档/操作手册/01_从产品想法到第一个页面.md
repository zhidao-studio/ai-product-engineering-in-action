# 从产品想法到第一个页面

## 1. 适用场景

你现在只有一个产品想法，还没有代码、没有工程、没有设计稿。

目标不是一次性做完整产品，而是先走到：

```text
产品定义已确认
界面规格已确认
工程规格已确认
工程模板已复制
第一个页面任务包已确认
可以让 AI 在模板工程内写第一个页面
```

## 2. 你需要准备什么

最少准备一句话产品想法：

```text
我想做一个……，帮助……解决……问题。
```

最好再准备：

- 一个你喜欢的参考网站 URL
- 一张你喜欢的页面截图
- 一个你不喜欢的反面例子
- 你明确不想做的功能

## 3. 第一步：打开 AI 工具

打开你选择的大模型工具。

可以是 ChatGPT、Claude、Gemini、Kimi、通义、豆包，或其他你能稳定使用的模型。

第一阶段先不要打开编码 Agent。

原因：

```text
先写约束文档，再复制工程模板，最后让 Agent 在模板工程内写代码。
```

## 4. 第二步：生成逐层计划

复制这个文件：

```text
指令/通用大模型/01_构造约束框架指令.md
```

把你的产品想法填进去。

AI 这一步只应该输出逐层构造计划，不应该直接生成 7 层完整文档。

## 5. 第三步：生成产品定义

复制这个文件：

```text
指令/通用大模型/02_产品定义约束指令.md
```

把 AI 输出保存到你的目标项目：

```text
目标项目/docs/01_产品定义.md
```

确认前，不进入下一步。

## 6. 第四步：生成界面规格

复制这个文件：

```text
指令/通用大模型/03_界面规格约束指令.md
```

输入：

```text
目标项目/docs/01_产品定义.md
参考截图或参考网站 URL
不喜欢的设计说明
```

把 AI 输出保存为：

```text
目标项目/docs/02_界面规格.md
```

## 7. 第五步：生成工程规格

复制这个文件：

```text
指令/通用大模型/04_工程规格约束指令.md
```

输入：

```text
目标项目/docs/01_产品定义.md
目标项目/docs/02_界面规格.md
文档/工程基线/01_工程基线总说明.md
```

把 AI 输出保存为：

```text
目标项目/docs/03_工程规格.md
```

## 8. 第六步：如果需要后端，生成接口和数据库

如果产品需要后端或保存数据，复制这个文件：

```text
指令/通用大模型/05_接口与数据约束指令.md
```

把 AI 输出保存为：

```text
目标项目/docs/04_接口契约.md
目标项目/docs/05_数据库结构.md
```

如果第一版是纯前端，可以跳过这一步。

## 9. 第七步：复制工程模板

不要让 AI 从 0 创建工程。

根据工程规格复制模板：

```text
工程模板/前端模板工程/  →  目标项目/frontend/
工程模板/后端模板工程/  →  目标项目/backend/
```

如果第一版不需要后端，只复制前端模板工程。

复制后不要先改模板结构，先生成代码任务包。

## 10. 第八步：生成第一个代码任务包

复制这个文件：

```text
指令/通用大模型/06_任务包约束指令.md
```

要求 AI 先只生成第一个任务，例如：

```text
TASK-FE-001：基于前端模板工程建立首页静态结构
```

把结果保存为：

```text
目标项目/docs/06_代码任务包.md
```

任务包必须写清楚：

- 基于哪个工程模板
- 允许修改哪些文件
- 不处理哪些内容
- 必须通过哪些检查命令
- 是否需要新增或更新测试

## 11. 第九步：开始让 AI 写第一个页面

这时才可以打开编码 Agent，例如 Codex、Claude Code、Cursor、其他 AI 编程工具。

给它的输入必须包括：

```text
目标项目/docs/01_产品定义.md
目标项目/docs/02_界面规格.md
目标项目/docs/03_工程规格.md
目标项目/docs/06_代码任务包.md
文档/工程基线/01_工程基线总说明.md
文档/工程基线/02_工程质量红线.md
工程模板/前端模板工程/README.md
指令/执行代理/工程任务指令.md
```

要求它：

```text
只执行当前任务包。
在模板工程内开发。
不重建工程。
不删除统一请求封装、错误处理、设计变量和基础配置。
完成后说明如何启动和验证。
```

## 12. 第十步：验收第一个页面

复制检查清单：

```text
检查清单/约束框架/02_使用者检查清单.md
检查清单/代码交付/02_前端检查清单.md
```

检查：

- 页面是否符合产品定义
- 页面是否符合界面规格
- 是否没有新增不做清单里的功能
- 是否基于工程模板开发
- 是否能运行
- 是否能通过构建

## 13. 一句话流程

```text
想法 → 产品定义 → 界面规格 → 工程规格 → 接口/数据 → 复制工程模板 → 任务包 → AI 在模板内写第一个页面 → 检查清单验收
```
