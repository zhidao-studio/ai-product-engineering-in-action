# AI 平台工作模式与提问策略

## 1. 目的

不同 AI 平台的默认工作方式不一样。

在本套文档控制系统里，不应简单要求 AI “少提问”或“直接执行”。

正确做法是区分两个阶段：

```text
规划审查阶段：先通读文档，再提问、给选项、列 ToDo，让使用者选择。
工程执行阶段：按已确认任务包直接执行，只在真阻塞时提问。
```

## 2. 两种工作模式

### 2.1 规划审查模式

适用于：

```text
第一次接入项目
通读已有文档工程
发现文档缺口
生成下一步执行计划
拆分代码任务包
进入编码前让使用者选择方案
```

这个阶段应该提问。

理想输出不是评价文档，而是：

```text
我读到了什么
我发现哪些缺口
哪些问题需要你确认
每个问题给 2-3 个可选方案
我推荐哪个方案以及原因
确认后 ToDo 怎么排
哪些可以立即做
哪些要等你选择后再做
```

### 2.2 工程执行模式

适用于：

```text
代码任务包已经确认
允许修改范围已经确认
接口契约已经确认
数据库结构已经确认
原型确认记录已经确认
```

这个阶段不应该反复提问。

除非遇到真正阻塞，否则应按任务包直接执行。

## 3. Kimi Coding 类平台的用法

如果平台会先通读文档、主动提问、生成选择项和 ToDo，这是适合本系统的规划审查行为。

这种行为应该保留。

使用方式：

```text
先让它通读项目文档和工程模板。
要求它不要写代码。
要求它输出问题清单、选择项、推荐方案和 ToDo。
使用者选择后，再让它生成或执行代码任务包。
```

## 4. Qwen 类平台的约束

如果平台只是在评价文档，例如：

```text
这是一个规范文档
这个文档结构清晰
这个文档用于指导开发
```

这不是本系统想要的输出。

应该用前置指令把它拉入规划审查模式：

```text
不要评价文档本身。
不要只总结这是什么文档。
请通读这些文档后，站在产品负责人、架构师和执行代理的角度，输出：
1. 你认为当前已经确定的内容
2. 你认为还缺失或冲突的内容
3. 需要我选择的问题清单
4. 每个问题给 2-3 个可选方案
5. 你的推荐方案和理由
6. 等我选择后的 ToDo List
7. 暂时不要写代码
```

## 5. 规划审查阶段允许提问

规划审查阶段，AI 必须主动提出高质量问题。

问题必须满足：

```text
和当前文档或工程直接相关
能影响后续代码生成
能帮助确定产品、设计、接口、数据库、工程或任务边界
每个问题必须给可选方案
每个问题必须说明推荐方案
```

不允许提出空泛问题：

```text
你想做什么？
还有什么需求？
你希望怎么做？
是否继续？
```

## 6. 选项格式

每个问题必须按以下格式输出：

```text
问题 N：……

为什么要确认：……

选项 A：……
优点：……
缺点：……
适合：……

选项 B：……
优点：……
缺点：……
适合：……

推荐：选择 A / B
原因：……
```

## 7. ToDo List 格式

规划审查阶段最后必须输出 ToDo：

```text
待你选择：
- [ ] 问题 1：选择 A / B / C
- [ ] 问题 2：选择 A / B / C

你选择后我执行：
- [ ] 更新产品定义
- [ ] 更新界面规格
- [ ] 更新接口契约
- [ ] 更新数据库结构
- [ ] 更新代码任务包
- [ ] 再进入工程执行

可以直接执行：
- [ ] 修正文档中的明显不一致
- [ ] 补充缺失检查清单
```

## 8. 工程执行阶段允许提问的情况

只有以下情况允许执行代理提问：

```text
缺少必须输入文件，且无法从上下文推断。
任务包没有说明目标端或目标模块。
两个已确认文档存在直接冲突。
执行会删除用户代码或造成不可逆修改。
需要真实密钥、账号、证书、生产环境地址。
测试或构建需要用户本地环境才能继续。
```

## 9. 工程执行阶段不允许提问的情况

以下情况不得提问，必须按文档和任务包执行：

```text
产品定义已经写明。
界面规格已经写明。
原型确认记录已经写明。
工程规格已经写明。
工程模板已经指定。
接口契约已经写明。
数据库结构已经写明。
代码任务包已经写明允许修改范围。
检查清单已经写明验收标准。
```

## 10. 推荐平台使用顺序

### 需求、设计、架构、任务拆分阶段

优先使用具备主动提问能力的平台。

目标输出：

```text
问题清单
方案选项
推荐方案
ToDo List
```

### 明确任务包后的代码执行阶段

优先使用执行稳定、少废话、能严格按文件修改的平台。

目标输出：

```text
修改文件
验证结果
风险点
待确认事项
```

## 11. 一句话原则

```text
规划阶段要会问，执行阶段要会做；会问不是问题，乱评价和不产出选择项才是问题。
```
