# 微信云托管适配方案

## 1. 定位

本文件定义在选择微信生态作为运行环境时，Web 后台、后端和微信小程序如何适配微信云托管。

这是可选部署目标，不是默认唯一部署方式。

具体可用能力、控制台配置项、计费规则和构建入口，以微信云托管当前官方控制台和官方文档为准。

## 2. 适用场景

适合以下项目：

```text
Web 后台 + Spring Boot 后端 + 微信小程序
H5 + Spring Boot 后端 + 微信小程序
以微信生态为主要入口的内部工具或轻量业务系统
```

不适合一开始就用于：

```text
复杂多云部署
强 Kubernetes 运维
多地域高可用
大型微服务体系
```

## 3. 推荐部署形态

```text
GitHub 仓库
  ↓
微信云托管构建
  ↓
Web 静态站点服务
Spring Boot 容器服务
微信小程序调用后端服务
```

## 4. Web 后台部署

Web 后台使用：

```text
工程模板/Web后台模板工程/
```

构建方式：

```text
npm install
npm run build
```

产物目录通常为：

```text
dist/
```

部署目标：

```text
云托管静态网站服务
```

注意：

- Web 后台是独立工程。
- 不和 H5、小程序共享前端代码。
- API 地址通过环境变量或构建配置注入。
- 自定义域名、HTTPS、缓存策略以云托管控制台为准。

## 5. H5 部署

H5 使用：

```text
工程模板/H5模板工程/
```

构建方式：

```text
npm install
npm run build
```

产物目录通常为：

```text
dist/
```

部署目标：

```text
云托管静态网站服务
```

注意：

- H5 是独立工程。
- H5 优先保证移动端触屏体验。
- H5 和 Web 后台的一致性来自接口契约，不来自共享代码。

## 6. Spring Boot 后端部署

后端使用：

```text
工程模板/后端模板工程/
```

推荐部署为：

```text
云托管容器服务
```

构建方式：

```text
mvn package
Dockerfile 构建镜像
容器启动 Spring Boot 应用
```

Dockerfile 应由具体项目生成，不建议在方法论仓库中写死生产参数。

后端必须满足：

- 使用 `/api/v1` 作为默认 API 前缀。
- Controller 只依赖 Service 接口。
- Service 使用 interface + impl。
- 不引入 Spring Cloud。
- 配置通过环境变量注入。
- 不提交真实密钥、数据库密码、微信密钥。

## 7. 微信小程序接入

微信小程序使用：

```text
工程模板/微信小程序模板工程/
```

小程序调用后端时，应优先使用微信云托管推荐的服务访问方式。

如果当前环境支持小程序到云托管服务的内网访问，应优先使用内网调用，减少公网暴露面。

如果需要公网域名，则按微信小程序服务器域名、HTTPS 和云托管域名规则配置。

注意：

- 小程序是独立工程。
- 小程序不引用 Web/H5 代码。
- 接口路径和字段以 `目标项目/docs/04_接口契约.md` 为准。
- 后端服务地址不得硬编码在页面里，应通过配置文件或环境配置统一管理。

## 8. GitHub 仓库建议结构

如果一个仓库同时包含 Web、H5、小程序和后端，建议：

```text
project-root/
  docs/
  web-admin/
  h5/
  weapp/
  backend/
  infra/
  README.md
```

每个端独立构建。

不要建立前端共享层。

## 9. 构建配置建议

### Web 后台

```text
服务类型：静态网站
源码目录：web-admin
构建命令：npm install && npm run build
产物目录：dist
```

### H5

```text
服务类型：静态网站
源码目录：h5
构建命令：npm install && npm run build
产物目录：dist
```

### 后端

```text
服务类型：容器服务
源码目录：backend
构建方式：Dockerfile
启动端口：以 Spring Boot 配置为准
健康检查：/api/v1/status
```

### 微信小程序

```text
源码目录：weapp
构建方式：微信开发者工具 / 小程序 CI
后端访问：按云托管推荐方式配置
```

## 10. 环境变量建议

后端至少需要：

```text
SERVER_PORT
SPRING_PROFILES_ACTIVE
DATABASE_URL
DATABASE_USERNAME
DATABASE_PASSWORD
```

如果接入微信能力，还需要按业务补充：

```text
WECHAT_APP_ID
WECHAT_APP_SECRET
WECHAT_TOKEN
WECHAT_AES_KEY
```

所有敏感配置必须放在云托管环境变量或密钥配置中，不得提交到 GitHub。

## 11. AI 生成代码时的限制

AI 不得：

- 直接写死云托管域名。
- 直接写死微信密钥。
- 在前端页面中散落后端地址。
- 为了部署引入 Spring Cloud。
- 为了部署把多端前端合并成一个工程。
- 生成和云托管当前控制台不一致的绝对化配置说明。

AI 必须：

- 保持多端独立工程。
- 后端配置外置。
- 接口契约优先。
- 后端保留 `/api/v1/status` 健康检查接口。
- 在部署说明中标注需要以微信云托管当前控制台为准。

## 12. 一句话原则

```text
微信云托管只解决运行和托管，
不改变多端独立工程、不改变接口契约优先、不改变后端单体基线。
```
