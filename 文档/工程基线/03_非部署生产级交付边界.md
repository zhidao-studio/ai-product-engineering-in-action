# 非部署生产级交付边界

## 1. 目的

本文件定义在不考虑发布部署的前提下，一个 AI 生成产品至少需要达到什么程度，才可以称为“接近生产级”。

这里不包含云平台部署、应用商店发布、证书、域名、CI/CD、灰度、监控告警等平台相关内容。

## 2. 可以称为接近生产级的条件

必须同时满足：

```text
产品边界清楚
工程模板已使用
代码任务包已执行
核心功能可运行
核心流程可验收
核心接口有契约
核心数据有结构
核心错误有处理
核心安全有约束
核心测试可执行
检查清单已通过
```

## 3. 不允许称为生产级的情况

出现以下任一情况，不得称为生产级：

- 没有产品定义
- 没有界面规格
- 没有工程规格
- 没有接口契约却开发了前后端联调
- 没有数据库结构却写入持久化数据
- 没有使用工程模板，从 0 让 AI 自行搭工程
- 没有代码任务包
- 没有测试任务
- 没有错误处理
- 没有输入校验
- 没有安全约束
- 没有检查清单验收

## 4. 分级判断

### Demo 级

```text
页面能打开，功能能演示，但工程边界、测试、安全和验收不足。
```

### 工程化 MVP

```text
基于工程模板开发，核心功能可运行，接口和数据可追溯，有基础测试和检查清单。
```

### 接近生产级

```text
工程化 MVP 之上，补齐认证、安全、错误处理、测试覆盖、数据约束、多端体验和使用验收。
```

### 完整生产级

```text
接近生产级之上，还需要发布部署、监控告警、备份恢复、权限深化、审计、压测、回滚和运维流程。
```

本仓库当前优先覆盖到“接近生产级”的非部署阶段。

## 5. 一句话判断

```text
文档定义边界，模板固定结构，任务包控制变更，测试和检查清单验证结果。
四者缺一，不应称为接近生产级。
```
