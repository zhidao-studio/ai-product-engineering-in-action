# 前后端命名规范

## 1. 目的

本文件定义前端、后端、API、数据库和任务包的统一命名规范。

命名规范用于约束 AI 不随意发明名称，避免同一概念在不同文件中出现多个叫法。

## 2. 参考原则

本规范参考成熟工程的常见约定：

```text
Vue / Vite / Element Plus / Vant 工程约定
Vue Vben Admin 的应用分层和命名习惯
微信小程序目录和页面命名习惯
Swift API Design Guidelines 的大小写习惯
Spring Boot 分层命名习惯
MyBatis-Plus Mapper / Entity 命名习惯
RuoYi-Vue-Plus 的 controller / domain / mapper / service / service.impl 分层习惯
REST API 资源命名习惯
关系型数据库 snake_case 命名习惯
```

原则：

```text
同一业务概念，全链路只用一套名称。
前端显示名可以中文，代码名必须英文。
代码名必须稳定、可搜索、可推导。
命名要表达业务语义，不表达实现细节。
```

## 3. 业务词汇表

每个项目必须先维护业务词汇表。

示例：

| 中文名 | 英文名 | 说明 |
|---|---|---|
| 视频任务 | VideoTask | 一条 AI 视频制作任务 |
| 用户 | User | 登录用户 |
| 会话 | Session | 登录会话 |

AI 生成代码前，必须优先使用业务词汇表中的英文名。

## 4. 前端命名规范

### 4.1 目录命名

前端目录使用小写 kebab-case 或固定复数目录名。

推荐：

```text
src/pages/
src/components/
src/services/
src/stores/
src/router/
src/types/
src/styles/
src/utils/
```

不得混用：

```text
src/page/
src/Pages/
src/apiService/
src/commonComponents/
```

### 4.2 Vue 页面和组件命名

Vue 单文件组件使用 PascalCase。

页面组件以 Page 结尾：

```text
LoginPage.vue
DashboardPage.vue
VideoTaskListPage.vue
VideoTaskDetailPage.vue
```

业务组件使用业务名 + 组件类型：

```text
VideoTaskForm.vue
VideoTaskTable.vue
VideoTaskCard.vue
```

基础状态组件使用固定命名：

```text
LoadingState.vue
EmptyState.vue
ErrorState.vue
```

### 4.3 前端 Service 命名

前端 API 调用文件使用业务名 + Service：

```text
sessionService.ts
videoTaskService.ts
userService.ts
```

导出函数使用动词 + 业务对象：

```text
login
listVideoTasks
getVideoTask
createVideoTask
updateVideoTask
```

不得使用含糊名称：

```text
doRequest
submitData
getList
handleApi
```

### 4.4 前端 Store 命名

Pinia Store 文件使用业务名 + Store：

```text
userStore.ts
sessionStore.ts
videoTaskStore.ts
```

Store hook 使用：

```text
useUserStore
useSessionStore
useVideoTaskStore
```

### 4.5 前端 Type 命名

TypeScript 类型使用 PascalCase。

请求类型以 Request 结尾：

```text
LoginRequest
CreateVideoTaskRequest
UpdateVideoTaskRequest
```

响应类型以 Response 结尾：

```text
LoginResponse
VideoTaskResponse
```

业务视图模型可以使用 Model 或 ViewModel：

```text
VideoTaskModel
VideoTaskViewModel
```

## 5. 微信小程序命名规范

### 5.1 页面目录

小程序页面目录使用小写 kebab-case。

```text
pages/home/
pages/video-task-list/
pages/video-task-detail/
```

每个页面保持同名四件套：

```text
index.ts
index.wxml
index.wxss
index.json
```

### 5.2 小程序 Service

小程序服务文件使用业务名 + Service：

```text
sessionService.ts
videoTaskService.ts
```

统一请求封装必须叫：

```text
request.ts
```

页面不直接调用 `wx.request`。

## 6. iOS 命名规范

Swift 类型使用 PascalCase。

Feature 目录使用业务名：

```text
Features/VideoTask/
```

View 以 View 结尾：

```text
VideoTaskListView
VideoTaskDetailView
VideoTaskFormView
```

ViewModel 以 ViewModel 结尾：

```text
VideoTaskListViewModel
VideoTaskDetailViewModel
```

Service 以 Service 结尾：

```text
VideoTaskService
SessionService
```

属性和方法使用 lowerCamelCase。

## 7. 后端命名规范

### 7.1 包名

Java 包名全部小写。

```text
com.example.template.controller.v1
com.example.template.domain.video.entity
com.example.template.domain.video.bo
com.example.template.domain.video.vo
com.example.template.mapper
com.example.template.service
com.example.template.service.impl
```

不得使用大写包名或复数混乱包名。

### 7.2 Controller 命名

Controller 使用业务名 + Controller：

```text
AuthController
VideoTaskController
WechatSessionController
```

Controller 路径使用 REST 资源名，统一 `/api/v1` 前缀。

```text
/api/v1/session
/api/v1/video-tasks
/api/v1/wechat/session
```

资源名使用小写 kebab-case，集合资源优先用复数。

### 7.3 Entity 命名

Entity 使用业务名：

```text
VideoTask
User
WechatUser
```

Entity 只表示数据库实体，不直接返回给前端。

### 7.4 BO 命名

业务入参对象使用动作 + 业务名 + Bo：

```text
CreateVideoTaskBo
UpdateVideoTaskBo
WechatLoginBo
```

BO 必须承载参数校验。

### 7.5 VO 命名

接口返回对象使用业务名 + Vo：

```text
VideoTaskVo
LoginVo
UserVo
WechatLoginVo
```

VO 只返回接口契约允许的字段。

### 7.6 Mapper 命名

Mapper 使用业务名 + Mapper：

```text
VideoTaskMapper
UserMapper
WechatUserMapper
```

Mapper 只做数据访问，不写业务判断。

### 7.7 Service 命名

Service 接口使用 I + 业务名 + Service：

```text
IAuthService
IVideoTaskService
IWechatSessionService
```

实现类使用业务名 + ServiceImpl：

```text
AuthServiceImpl
VideoTaskServiceImpl
WechatSessionServiceImpl
```

Controller 只依赖 Service 接口，不依赖实现类。

## 8. API 命名规范

API 路径使用小写 kebab-case。

推荐：

```text
GET /api/v1/video-tasks
GET /api/v1/video-tasks/{id}
POST /api/v1/video-tasks
PUT /api/v1/video-tasks/{id}
```

不得使用：

```text
/getVideoTaskList
/videoTask/getList
/api/videoTask/list
```

请求字段和响应字段使用 lowerCamelCase：

```text
title
topic
scriptSummary
storyboardSummary
createdAt
updatedAt
```

错误码使用 UPPER_SNAKE_CASE：

```text
INVALID_REQUEST
UNAUTHORIZED
FORBIDDEN
NOT_FOUND
INTERNAL_ERROR
```

## 9. 数据库命名规范

数据库表名使用 snake_case 单数或业务单数名。

推荐：

```text
video_task
wechat_user
user_role
role_permission
```

字段名使用 snake_case：

```text
id
title
topic
script_summary
storyboard_summary
created_at
updated_at
```

索引使用：

```text
idx_表名_字段名
uk_表名_字段名
```

示例：

```text
idx_video_task_status
idx_video_task_created_at
uk_user_username
```

## 10. 任务包命名规范

任务编号使用：

```text
TASK-端或层-序号
```

示例：

```text
TASK-WEB-001
TASK-H5-001
TASK-WEAPP-001
TASK-IOS-001
TASK-BE-001
TASK-DB-001
TASK-INT-001
```

任务名称必须包含动作和对象：

```text
实现视频任务列表页
实现视频任务创建接口
新增 video_task 表结构
```

不得使用：

```text
改一下页面
补接口
处理逻辑
```

## 11. AI 命名规则

AI 生成任何代码前，必须先检查：

```text
是否已有同义命名？
是否符合业务词汇表？
是否符合本命名规范？
是否会导致前后端字段不一致？
是否会导致数据库字段不可追溯？
```

AI 不得因为局部方便新增同义词。

例如已有 `VideoTask`，不得再新增：

```text
VideoJob
TaskVideo
AiVideoTask
VideoMission
```

除非产品定义明确它们是不同业务概念。

## 12. 一句话原则

```text
业务词汇先定，前后端同名，接口字段同名，数据库可追溯，AI 不造同义词。
```
