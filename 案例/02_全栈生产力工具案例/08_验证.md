# 验证

## 1. 功能检查

| 检查项 | 通过标准 |
|---|---|
| 登录 | 正确账号能登录，错误账号有错误提示 |
| 任务列表 | 登录后能看到任务列表或空状态 |
| 新建任务 | 能创建视频任务并返回列表 |
| 任务详情 | 能查看任务标题、主题、脚本摘要、分镜摘要 |
| 状态更新 | 能修改任务状态并保存 |

## 2. 接口检查

- [ ] 所有接口路径符合 `04_接口契约.md`
- [ ] 响应字段符合接口契约
- [ ] 错误码符合接口契约
- [ ] 前端没有调用未定义接口

## 3. 数据库检查

- [ ] 表结构符合 `05_数据库结构.md`
- [ ] 不存明文密码
- [ ] 不向前端返回 password_hash
- [ ] 迁移文件可追溯

## 4. 前端工程检查

- [ ] 页面放在 `src/pages/`
- [ ] API 调用放在 `src/services/`
- [ ] 类型定义放在 `src/types/`
- [ ] 页面包含 loading / empty / error 状态
- [ ] `npm run build` 可通过

## 5. 后端工程检查

- [ ] 业务逻辑放在 service
- [ ] 数据访问放在 repository
- [ ] 请求响应对象使用 dto
- [ ] 错误响应统一
- [ ] `./mvnw test` 可通过

## 6. 追溯检查

每个代码改动必须能追溯到：

```text
产品定义
界面规格
工程规格
接口契约
数据库结构
代码任务包
```

## 7. 结论

```text
是否通过：通过 / 有条件通过 / 不通过
必须修复：
是否进入使用验收：是 / 否
```
