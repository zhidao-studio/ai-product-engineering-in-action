# 代码任务包

## 1. 任务拆分原则

- 一个任务只处理一个目标
- 前端、后端、数据库、联调、测试分开
- 每个任务必须有允许修改范围和禁止范围
- 未完成当前任务验收，不进入下一任务

## 3. 示例任务：TASK-BE-002

目标：实现登录接口。

允许修改：

```text
backend/src/main/java/**/api/
backend/src/main/java/**/service/
backend/src/main/java/**/repository/
backend/src/main/java/**/dto/
backend/src/main/java/**/exception/
backend/src/main/resources/application.yml
```

禁止修改：

```text
frontend/
infra/
接口契约未定义的其他接口
```

验收标准：

- POST `/api/auth/login` 可调用
- 正确账号完成登录并返回 user 信息
- 登录态按接口契约设置
- 错误账号返回 UNAUTHORIZED
- 不返回 password_hash
- 不引入 Spring Cloud

## 4. 示例任务：TASK-FE-003

目标：实现任务列表页。

允许修改：

```text
frontend/src/pages/
frontend/src/services/
frontend/src/types/
frontend/src/router/
```

禁止修改：

```text
backend/
infra/
数据库结构
接口契约
```

验收标准：

- 页面能展示任务列表
- 支持 loading / empty / error 状态
- API 调用放在 services
- 401 行为符合接口契约
- 字段来自接口契约
- 不新增接口契约外字段

## 5. 示例测试任务：TASK-BE-TEST-001

目标：验证后端登录和任务接口。

验收标准：

- 登录成功用例通过
- 登录失败用例通过
- 未登录访问任务接口返回 UNAUTHORIZED
- 创建任务参数校验用例通过
- 查询任务详情用例通过

## 6. 示例测试任务：TASK-FE-TEST-001

目标：验证前端核心交互。

验收标准：

- 登录失败时展示错误提示
- 未登录进入任务页时跳转登录页
- 任务列表 loading / empty / error 状态可验证
- 新建任务表单必填校验可验证

## 7. 执行规则

每个任务执行前，必须把相关项目文档和工程任务指令一起提供给 AI。

每个任务完成后，AI 必须输出：

```text
修改了哪些文件
为什么修改
如何启动
如何验证
是否新增或更新测试
风险点
```

## 8. 变更记录

每个任务完成后，必须在目标项目 `docs/` 下追加变更记录，包含：任务编号、修改文件、修改原因、验证结果、遗留风险。
