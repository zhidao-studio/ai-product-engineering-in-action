# 代码任务包

## 1. 任务拆分原则

- 一个任务只处理一个目标
- Web 后台、后端、数据库、联调、测试分开
- 每个任务必须有允许修改范围和禁止范围
- 未完成当前任务验收，不进入下一任务
- 所有代码任务必须基于工程模板执行，不从 0 搭工程

## 2. 基于的工程模板

本案例基于：

```text
工程模板/Web后台模板工程/
工程模板/后端模板工程/
```

如果后续增加移动端入口，可以再引入：

```text
工程模板/H5模板工程/
工程模板/微信小程序模板工程/
工程模板/iOS模板工程/
```

## 4. 示例任务：TASK-BE-002

目标：基于后端模板的 `auth/` 样板调整登录接口。

允许修改：

```text
backend/src/main/java/**/auth/
backend/src/main/java/**/common/
backend/src/main/java/**/exception/
backend/src/test/java/**/auth/
```

禁止修改：

```text
frontend/
infra/
接口契约未定义的其他接口
后端工程基础结构
```

验收标准：

- 登录接口路径符合接口契约
- 正确账号完成登录并返回 user 信息
- 错误账号返回 UNAUTHORIZED
- 不返回 password_hash
- 认证相关 DTO 有参数校验
- 对应测试通过
- 不引入 Spring Cloud

## 5. 示例任务：TASK-WEB-003

目标：基于 Web 后台模板实现任务列表页。

允许修改：

```text
frontend/src/pages/
frontend/src/services/
frontend/src/types/
frontend/src/router/
frontend/src/components/
```

禁止修改：

```text
backend/
infra/
数据库结构
接口契约
Web 后台模板基础结构
```

验收标准：

- 页面能展示任务列表
- 支持 loading / empty / error 状态
- API 调用放在 services
- 401 行为符合接口契约
- 字段来自接口契约
- 不新增接口契约外字段
- 前端检查命令通过

## 6. 示例测试任务：TASK-BE-TEST-001

目标：验证后端登录和任务接口。

验收标准：

- 登录成功用例通过
- 登录失败用例通过
- 未登录访问任务接口返回 UNAUTHORIZED
- 创建任务参数校验用例通过
- 查询任务详情用例通过

## 7. 示例测试任务：TASK-WEB-TEST-001

目标：验证 Web 后台核心交互。

验收标准：

- 登录失败时展示错误提示
- 未登录进入任务页时跳转登录页
- 任务列表 loading / empty / error 状态可验证
- 新建任务表单必填校验可验证
- 不绕过统一请求封装

## 8. 执行规则

每个任务执行前，必须把相关项目文档、工程模板 README 和工程任务指令一起提供给 AI。

每个任务完成后，AI 必须输出：

```text
修改了哪些文件
为什么修改
如何启动
如何验证
是否新增或更新测试
风险点
```

## 9. 变更记录

每个任务完成后，必须在目标项目 `docs/` 下追加变更记录，包含：任务编号、修改文件、修改原因、验证结果、遗留风险。
