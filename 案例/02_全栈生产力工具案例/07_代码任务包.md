# 代码任务包

## 1. 任务拆分原则

- 一个任务只处理一个目标
- 前端、后端、数据库、联调分开
- 每个任务必须有允许修改范围和禁止范围
- 未完成当前任务验收，不进入下一任务

## 2. 任务列表

| 编号 | 类型 | 目标 | 依赖文档 |
|---|---|---|---|
| TASK-INFRA-001 | infra | 建立本地数据库和环境变量样例 | 基础设施规范 / 数据库结构 |
| TASK-BE-001 | backend | 初始化 Spring Boot 单体工程 | 工程规格 / 工程基线 |

| TASK-FE-002 | frontend | 实现登录页和登录状态 | 界面规格 / 接口契约 |
| TASK-FE-003 | frontend | 实现任务列表页 | 界面规格 / 接口契约 |
| TASK-FE-004 | frontend | 实现新建任务和详情页 | 界面规格 / 接口契约 |
| TASK-INT-001 | integration | 前后端联调 | 全部项目文档 |

## 3. 示例任务：TASK-BE-002

目标：实现登录接口。

允许修改：

```text
backend/src/main/java/**/api/
backend/src/main/java/**/service/
backend/src/main/java/**/repository/
backend/src/main/java/**/dto/
backend/src/main/java/**/exception/
backend/src/main/resources/application.yml
```

禁止修改：

```text
frontend/
infra/
接口契约未定义的其他接口
```

验收标准：

- POST `/api/auth/login` 可调用
- 正确账号返回 token 和 user
- 错误账号返回 UNAUTHORIZED
- 不返回 password_hash
- 不引入 Spring Cloud

## 4. 示例任务：TASK-FE-003

目标：实现任务列表页。

允许修改：

```text
frontend/src/pages/
frontend/src/services/
frontend/src/types/
frontend/src/router/
```

禁止修改：

```text
backend/
infra/
数据库结构
接口契约
```

验收标准：

- 页面能展示任务列表
- 支持 loading / empty / error 状态
- API 调用放在 services
- 字段来自接口契约
- 不新增接口契约外字段

## 5. 执行规则

每个任务执行前，必须把相关项目文档和工程任务指令一起提供给 AI。

每个任务完成后，AI 必须输出：

```text
修改了哪些文件
为什么修改
如何启动
如何验证
风险点
```
