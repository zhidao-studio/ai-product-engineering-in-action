# 基础设施规范

## 1. 本地开发目标

第一版只约束本地开发和调试，不做生产级云部署。

## 2. 软件清单

| 软件 | 用途 | 安装方式 |
|---|---|---|
| Node.js | 前端运行环境 | `brew install node` |
| Java | 后端运行环境 | `brew install openjdk` |
| nginx | 本地反向代理 | `brew install nginx` |
| ngrok | 本地公网调试 | 按官方方式安装 |
| Docker Desktop | 数据库容器 | 手动安装 |

## 3. 端口规划

| 服务 | 端口 |
|---|---:|
| 前端 Vite | 5173 |
| 后端 Spring Boot | 8080 |
| nginx 本地入口 | 8088 |
| PostgreSQL | 5432 |

## 4. nginx 代理规则

```text
/          → http://localhost:5173
/api       → http://localhost:8080/api
```

nginx 用于提供统一入口，方便 ngrok 暴露。

## 5. ngrok 使用规则

ngrok 指向 nginx 端口：

```text
ngrok http 8088
```

不得暴露数据库端口。

不得把 ngrok 公网地址写死进前端或后端代码。

## 6. 环境变量

目标项目必须提供：

```text
.env.example
```

至少包含：

```text
DATABASE_URL=
DATABASE_USERNAME=
DATABASE_PASSWORD=
JWT_SECRET=
VITE_API_BASE_URL=
```

不得提交真实 `.env` 文件。
