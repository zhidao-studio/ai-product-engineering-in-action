# 全栈生产力工具案例

## 1. 案例目标

本案例用于证明这套约束框架不只能做纯前端演示，也能约束一个带前端、后端、接口、数据库和登录能力的内部生产力工具 MVP。

案例产品：

```text
AI 视频任务管理后台
```

## 2. 案例边界

本案例是 MVP，不是生产级 SaaS。

第一版做：

- 单用户登录
- 视频任务列表
- 新建视频任务
- 查看任务详情
- 更新任务状态
- 保存任务脚本和分镜摘要

第一版不做：

- 多租户
- 在线支付
- 视频大模型真实调用
- 多人协作权限
- 消息通知
- 云端生产部署

## 3. 案例文件

| 文件 | 作用 |
|---|---|

| `08_验证.md` | 功能、接口、数据、工程检查 |
| `09_使用验收.md` | 使用者验收流程 |

## 4. 证明点

本案例走通：

```text
产品定义 → 界面规格 → 工程规格 → 接口契约 → 数据库结构 → 任务包 → 验证 → 使用验收 → 交付迭代
```
