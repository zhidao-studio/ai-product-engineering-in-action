# 案例

本目录存放填好的参考案例。

案例的作用不是展示完整代码工程，而是展示样板应该如何填写、如何串联项目约束文档。

## 目录说明

```text
案例/01_最小约束框架案例/      纯前端最小案例，用于理解 7 层约束
案例/02_全栈生产力工具案例/    带前端、后端、接口、数据库和登录的 MVP 案例
```

## 使用方式

1. 先看 `案例/01_最小约束框架案例/`，理解约束框架的最小形态。
2. 再看 `案例/02_全栈生产力工具案例/`，理解真实 MVP 如何串联前端、后端、接口、数据库和任务包。
3. 对照 `样板/` 填写自己的项目版本。
4. 用 `检查清单/` 检查完整性。
