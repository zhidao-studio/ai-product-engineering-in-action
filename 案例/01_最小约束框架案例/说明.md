# 最小约束框架案例

## 1. 案例目标

这个案例不开发完整产品，只展示如何从一个产品想法构造最小可用约束框架。

产品想法：

```text
做一个页面，帮助用户收藏自己喜欢的地点。
```

## 2. 案例文件

| 文件 | 对应层级 | 作用 |
|---|---|---|
| `01_产品定义.md` | 产品定义约束 | 锁定做什么、不做什么、成功标准 |
| `02_界面规格.md` | 界面规格约束 | 锁定视觉风格、页面模块和状态 |
| `03_工程规格.md` | 工程规格约束 | 锁定字段、前端范围、后端范围和接口方案 |
| `04_任务执行.md` | 任务执行约束 | 锁定 AI 本次只能做什么 |
| `05_验证.md` | 验证约束 | 判断 AI 产物是否符合文档和任务 |
| `06_使用验收.md` | 使用验收约束 | 判断真实用户是否能完成核心流程 |
| `07_交付迭代.md` | 交付迭代约束 | 约束上线、反馈、变更和迭代 |

## 3. 边界

本案例不做登录、不做后端、不做真实 API、不做地图、不写代码。
