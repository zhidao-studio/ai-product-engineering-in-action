# 工程基线摘要

## 1. 多端原则

```text
多个工程，独立建设。
接口一致，代码不共享。
各端体验，各端负责。
```

## 2. 默认工程模板

```text
Web 后台：工程模板/Web后台模板工程/
H5：工程模板/H5模板工程/
微信小程序：工程模板/微信小程序模板工程/
iOS：工程模板/iOS模板工程/
后端：工程模板/后端模板工程/
```

只复制当前版本需要的端。

## 3. 后端基线

```text
Java 17+
Spring Boot 3
MyBatis-Plus
Flyway
MySQL 优先
单体应用
不使用 Spring Cloud
```

## 4. 数据库基线

```text
主键 id：CHAR(32)，应用层生成 32 位字符串
每表 7 要素：id、created_by、created_at、updated_by、updated_at、is_valid、is_deleted
时间字段：默认 trigger 维护
关联字段：默认使用主键 id 关联，关联字段类型 CHAR(32)
```

## 5. 码值基线

```text
数据库码值是源头。
后端常量必须和数据库码值一致。
接口契约必须记录码值。
前端只做展示翻译，不改变码值。
```

## 6. 命名基线

```text
业务词汇先定。
前后端同名。
接口字段同名。
数据库字段可追溯。
AI 不造同义词。
```

## 7. 原型确认基线

```text
界面规格后必须生成高保真交互预览。
使用者视觉确认前，不得进入正式工程编码。
原型代码不得直接复制到正式工程。
```

## 8. AI 执行基线

```text
先判断阶段。
再判断允许输出。
再判断允许修改范围。
最后执行当前任务。
```
