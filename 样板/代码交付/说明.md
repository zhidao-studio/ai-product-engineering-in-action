# 代码交付样板

本目录用于把约束框架继续落到代码交付。

如果项目从 0 开始，不能直接让 AI 写代码。必须先定义工程基线、前端规范、后端规范、接口契约、数据库规范、基础设施规范、代码任务包和代码验收方式。

## 样板清单

| 文件 | 作用 |
|---|---|
| `01_项目总工程结构样板.md` | 项目总工程结构 |
| `02_前端工程规范样板.md` | 前端工程规范 |
| `03_后端工程规范样板.md` | 后端工程规范 |
| `04_接口契约样板.md` | 前后端接口契约 |
| `05_数据库结构样板.md` | 数据库和数据模型规范 |
| `06_基础设施规范样板.md` | 基础设施规范 |
| `07_代码任务包样板.md` | AI 编码任务包 |
| `08_代码验收清单样板.md` | 代码交付验收清单 |

## 基线来源

当前项目推荐技术基线以这里为准：

```text
文档/工程基线/01_工程基线总说明.md
```

工程质量红线以这里为准：

```text
文档/工程基线/02_工程质量红线.md
```
