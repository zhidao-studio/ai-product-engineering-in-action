# 基础设施规范样板

## 1. 基础设施目标

本文件用于约束本地开发环境、反向代理、内网穿透和基础服务启动方式。

当前基线：

```text
macOS
Homebrew
nginx
ngrok
Docker Compose（可选）
```

## 2. 软件清单

| 软件 | 用途 | 安装方式 |
|---|---|---|
| Node.js | 前端运行环境 | `brew install node` |
| Java | 后端运行环境 | `brew install openjdk` |
| nginx | 本地反向代理 | `brew install nginx` |
| ngrok | 本地服务公网访问 | 按官方方式安装 |
| Docker Desktop | 本地数据库和依赖服务 | 按项目需要 |

## 3. 本地端口规划

| 服务 | 默认端口 | 说明 |
|---|---:|---|
| 前端 Vite | 5173 | 本地前端开发 |
| 后端 Spring Boot | 8080 | 本地 API 服务 |
| nginx | 8088 / 80 | 本地反向代理 |
| 数据库 | 5432 / 3306 | 按数据库类型决定 |

## 4. nginx 和 ngrok 规则

nginx 用于本地统一入口、代理前端页面、代理后端 API，并给 ngrok 暴露统一端口。

ngrok 用于临时公网访问、Webhook 调试和外部设备访问本机服务。

不得直接暴露数据库，不提交 ngrok token，不把公网地址写死进代码。

## 5. CORS 规则

如果前端和后端不同源，必须明确 CORS 配置。

必须填写：

| 项目 | 内容 |
|---|---|
| 允许的前端源 | 例如 `http://localhost:5173`、`http://localhost:8088` |
| 允许的方法 | GET / POST / PUT / DELETE 等 |
| 是否允许凭证 | 是 / 否 |
| 本地开发配置 |  |
| 临时调试配置 |  |
| 生产配置 |  |

规则：

- 不允许默认使用 `Access-Control-Allow-Origin: *`。
- Cookie 登录态必须明确凭证策略。
- 本地、调试和生产配置必须区分。
- CORS 配置应集中放在后端配置中，不得散落在业务代码里。
