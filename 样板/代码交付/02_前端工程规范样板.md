# 前端工程规范样板

## 1. 技术选择

前端采用 Vue 体系。

推荐基线：

```text
Vue 3
Vite
TypeScript
Vue Router
Pinia
ESLint
Prettier
Vitest
Playwright 或 Cypress（二选一）
```

初始化方式：

```text
npm create vue@latest
```

## 2. 目录结构

```text
frontend/
  src/
    pages/
    components/
    services/
    interceptors/
    stores/
    router/
    types/
    utils/
    styles/
    App.vue
    main.ts
  public/
  package.json
  vite.config.ts
```

## 3. 目录职责

| 目录 | 职责 |
|---|---|
| `pages/` | 页面级组件 |
| `components/` | 可复用 UI 组件 |
| `services/` | API 调用和数据访问封装 |
| `interceptors/` | 统一请求和错误拦截 |
| `stores/` | Pinia 状态管理 |
| `router/` | 路由配置 |
| `types/` | TypeScript 类型定义 |
| `utils/` | 通用工具函数 |
| `styles/` | 全局样式和设计 token |

## 4. 编码规则

- 页面组件只能放在 `pages/`。
- 可复用组件只能放在 `components/`。
- 接口调用只能放在 `services/`。
- 请求和响应错误拦截只能放在 `interceptors/` 或统一请求封装中。
- 类型定义必须放在 `types/`。
- 页面里不能直接写 `fetch`。
- 页面里不能硬编码后端地址。
- 不允许每个页面各自复制一套 API 错误处理逻辑。

## 5. 状态管理规则

- 只有跨页面共享状态才进入 Pinia store。
- 页面内部临时状态使用 `ref` 或 `reactive`。
- 登录状态、用户信息、全局配置可以进入 store。
- 接口缓存状态只有在多个页面复用时才进入 store。
- 不得把所有表单字段无差别放入 store。

## 6. 错误处理模式

前端必须有统一错误处理模式：

- API 失败必须经过统一请求封装或拦截器处理。
- 401 必须按接口契约处理：跳转登录页、提示登录失效，或执行刷新流程。
- 网络错误统一展示可理解提示，不展示原始异常堆栈。
- 表单校验错误优先在字段附近展示。
- 系统级错误可以使用全局 toast 或页面错误状态。
- 不允许每个接口散落 `try-catch` 后各自写不同提示。

## 7. 页面状态

每个核心页面至少考虑：loading、empty、success、error。

## 8. 响应式规则

默认断点：

```text
768px   移动端 / 平板分界
1024px  平板 / 桌面分界
1440px  大屏优化分界
```

必须说明项目采用移动端优先还是桌面端优先。

如果有侧边导航，必须说明移动端如何收起或切换为顶部导航。
