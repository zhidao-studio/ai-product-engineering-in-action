## 4. 表结构定义

### 4.1 表名：video_task

表说明：视频任务表。

字段清单：

| 字段名 | 类型 | 是否非空 | 默认值 / 维护方式 | Java 字段 | API 字段 | 说明 |
|---|---|---|---|---|---|---|
| id | BIGINT | 是 | 主键 | id | id | 主键 |
| title | VARCHAR(200) | 是 | 无 | title | title | 任务标题 |
| topic | VARCHAR(200) | 是 | 无 | topic | topic | 视频主题 |
| script_summary | VARCHAR(1000) | 否 | NULL | scriptSummary | scriptSummary | 脚本摘要 |
| storyboard_summary | VARCHAR(1000) | 否 | NULL | storyboardSummary | storyboardSummary | 分镜摘要 |
| status | VARCHAR(50) | 是 | draft | status | status | 任务状态 |
| created_by | BIGINT | 是 | 0 | createdBy | 不默认暴露 | 创建人 |
| created_at | TIMESTAMP | 是 | trigger 维护 | createdAt | createdAt | 创建时间 |
| updated_by | BIGINT | 是 | 0 | updatedBy | 不默认暴露 | 更新人 |
| updated_at | TIMESTAMP | 是 | trigger 维护 | updatedAt | updatedAt | 更新时间 |
| is_valid | TINYINT(1) | 是 | 1 | validFlag | 不默认暴露 | 是否有效，码值 1/0 |
| is_deleted | TINYINT(1) | 是 | 0 | removedFlag | 不默认暴露 | 是否删除，码值 1/0 |

7 要素检查：

| 要素 | 字段 | 码值 | 是否具备 |
|---|---|---|---|
| 主键 id | id | 无 | 是 |
| 创建人 | created_by | 用户 id，系统默认 0 | 是 |
| 创建时间 | created_at | trigger 维护 | 是 |
| 更新人 | updated_by | 用户 id，系统默认 0 | 是 |
| 更新时间 | updated_at | trigger 维护 | 是 |
| 是否有效 | is_valid | 1 有效，0 无效 | 是 |
| 是否删除 | is_deleted | 0 未删除，1 已删除 | 是 |

主键：

```text
id
```

索引：

| 索引名 | 字段 | 类型 | 查询场景 |
|---|---|---|---|
| idx_video_task_status | status | 普通索引 | 按状态筛选任务 |
| idx_video_task_created_at | created_at | 普通索引 | 按创建时间排序 |
| idx_video_task_created_by | created_by | 普通索引 | 按创建人查询任务 |
| idx_video_task_valid_deleted | is_valid, is_deleted | 普通索引 | 默认过滤有效且未删除数据 |

唯一约束：

```text
无
```

状态枚举：

| 值 | 说明 |
|---|---|
| draft | 草稿 |
| pending | 等待处理 |
| generating | 生成中 |
| completed | 已完成 |
| failed | 失败 |

关联规则：

```text
created_by -> user.id
updated_by -> user.id
```

多表查询必须使用主键 `id` 关联。

是否使用 trigger：是

是否逻辑删除：是，使用 `is_deleted`

是否业务有效控制：是，使用 `is_valid`

对应 Entity：


对应 Mapper：

## 5. Flyway migration

对应 migration 文件：

```text
src/main/resources/db/migration/V1__create_video_task.sql
```

规则：

- 新增表必须新增 migration。
- 修改表结构必须新增 migration。
- 不得修改已经执行过的 migration。
- 如果 migration 中包含 trigger，必须在模板验证阶段确认可执行。

## 8. AI 生成约束

AI 生成或修改数据库结构时，必须：

- 使用本文件定义的表名和字段名。
- 每张业务表保留 7 要素。
- 时间字段默认使用 trigger 维护。
- 多表查询默认使用主键 id 关联。
- 码值以数据库为源头，后端常量必须一致。
- 同步更新接口契约。
- 同步更新 Entity、BO、VO、Mapper 和 Service。
- 新增 Flyway migration。
- 不直接把 Entity 返回给前端。
