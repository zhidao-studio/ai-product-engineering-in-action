# AI 执行规则

## 1. 当前仓库目标

本仓库的目标是：

```text
定义一套文档驱动的 AI 产品交付方法论，
提供约束框架、工程基线、工程模板、代码任务包和检查清单，
让非技术人能学会并实际使用。
```

## 2. 目录职责

```text
文档/        方法论、工程基线、操作手册、教学化说明
样板/        约束框架样板和代码交付样板
工程模板/    前端模板工程和后端模板工程
检查清单/    约束框架检查清单和代码交付检查清单
指令/        AI 执行指令
案例/        填好的案例
```

根目录保留 `README.md`、`AGENTS.md`。

工程模板内部允许保留生态标准文件名，例如 `package.json`、`pom.xml`、`tsconfig.json`、`src/main.ts`。

## 3. 工程交付流程

```text
先写项目约束文档
再复制工程模板
再生成代码任务包
再在模板工程内开发业务代码
最后用检查清单和项目命令验证
```

## 4. 代码交付必须参考

```text
文档/工程基线/01_工程基线总说明.md
文档/工程基线/02_工程质量红线.md
工程模板/前端模板工程/
工程模板/后端模板工程/
样板/代码交付/07_代码任务包样板.md
检查清单/代码交付/
```

## 5. 输出要求

每次修改后说明：

1. 修改了哪些文件。
2. 为什么这样改。
3. 如何验证。
4. 还有什么风险或遗留项。
