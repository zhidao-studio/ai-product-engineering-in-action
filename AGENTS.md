# AI 执行规则

## 1. 仓库目标

本仓库用于定义一套阶段受控的 AI 产品交付协议。

目标是：

```text
人负责意图和决策。
AI 负责受控执行。
系统负责阶段、契约、边界、门禁和反馈。
```

## 2. 新任务默认入口

所有新任务默认先使用：

```text
指令/通用大模型/00_默认启动口令.md
```

普通使用者不得直接让 AI 跳过控制系统去写代码、改文件或执行命令。

AI 接到任务后，必须先判断：

```text
当前模式：使用者模式 / 开发者模式
当前阶段
当前 AI 角色
允许做什么
禁止做什么
需要人确认什么
准备修改哪些文件
准备执行哪些验证
```

## 3. 入口体验原则

```text
给使用者：简单、清晰、高体验、少选择、少技术细节。
给开发者：稳定、可维护、可验证、可追踪、可回滚。
```

使用者模式下，少暴露技术细节，多输出人话解释、选项和推荐方案。

开发者模式下，保留必要技术细节，输出验证方法、影响范围、回滚方式和维护风险。

## 5. 结构冻结规则

当前整体结构已经冻结。

结构冻结不是禁止新增文件，而是禁止新增另一套横向体系、另一套总览、另一套同类方法论或不归属的散文件。

后续只允许两类动作：

```text
1. 把 文档/控制系统/00_系统总览.md 中已经定义的节点做实。
2. 把软约束升级成结构化契约或可执行门禁。
```

新增文件前必须先判断：

```text
它属于使用者主线、AI 主线、工程主线、治理主线中的哪一条？
它属于文档层、契约层、门禁层、反馈层中的哪一层？
它是否能合并到已有文件？
它是否有执行价值？
它是否会成为新的唯一入口？
```

如果不能明确归层，或者可以合并到已有文件，不要新增文件。

## 6. 目录职责

```text
文档/        方法论、工程基线、控制系统、操作手册、教学化说明
样板/        约束框架样板、代码交付样板、控制系统元数据样板
工程模板/    Web、H5、微信小程序、iOS、后端多类独立工程模板
检查清单/    约束框架检查清单、原型确认清单、代码交付检查清单
指令/        AI 执行指令、通用大模型指令、检查指令
案例/        填好的案例
scripts/     AI 门禁脚本
.github/     GitHub Actions 门禁工作流
```

根目录只保留必要入口文件，例如：

```text
README.md
AGENTS.md
```

## 7. iOS 基线

当前 iOS 默认基线是：

```text
SwiftUI
MVVM
CocoaPods
XCTest
```

iOS 依赖管理默认使用：

```text
Podfile
Podfile.lock
.xcworkspace
```

业务任务不得擅自切换到 Swift Package Manager，不得擅自新增 `Package.swift`。

如需切换依赖管理方式，必须进入开发者模式，并由开发者确认工程基线变更。

## 8. AI 角色

本仓库统一使用六类 AI 角色：

```text
规划 AI：问题清单、选项、产品定义、任务拆分建议。
原型 AI：界面规格、高保真原型、页面状态、交互说明。
工程 AI：工程规格、OpenAPI、Schema、任务包、模板内代码。
验证 AI：构建、测试、门禁、失败分析、人话报告。
开发者 AI：模板修复、脚本增强、CI 修复、基线维护。
治理 AI：风险记录、成本记录、回滚建议、演化记录。
```

执行顺序：

```text
先判断当前阶段
再判断当前模式和 AI 角色
再读取目标项目 AI_CONTEXT_PACK
再确认允许修改范围
再生成或修改产物
再执行验证和门禁
最后输出人话报告
```

## 9. 禁止事项

任何 AI 角色都不得：

```text
跳过当前阶段
跳过原型确认
跳过契约确认
跳过任务包
跳过门禁
越界修改保护路径
新增白名单外依赖
私自改变技术栈
为了解决一个问题重建整个工程
业务任务中擅自从 CocoaPods 切换到 Swift Package Manager
把技术日志直接丢给使用者决策
替使用者决定上线、回滚或继续投入
```

## 10. 交付前必须验证

对目标项目完成修改后，必须执行：

```bash
bash scripts/check-ai-guard.sh <目标项目目录>
```

如果门禁不通过，不得声明任务完成。

当前模板验证状态以这里为准：

```text
文档/工程基线/03_工程模板验证记录.md
```

如果模板状态是“未验证”或“缺件已补齐待验证”，不得宣称模板已经生产可用。

## 11. 输出要求

每次修改后说明：

```text
当前模式：使用者模式 / 开发者模式
当前 AI 角色：规划 AI / 原型 AI / 工程 AI / 验证 AI / 开发者 AI / 治理 AI
修改了哪些文件
为什么这样改
如何验证
AI 门禁脚本是否通过
是否遵守冻结结构
风险或遗留项
```

如果验证失败，必须输出：

```text
失败分类：
发生位置：
影响范围：
是否阻塞：
人话解释：
可选方案：
推荐方案：
需要谁确认：使用者 / 开发者 / 不需要
```
