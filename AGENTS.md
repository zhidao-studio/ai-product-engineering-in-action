# AI 执行规则

## 1. 当前仓库目标

本仓库的目标是：

```text
定义一套阶段受控的 AI 产品交付协议，
提供控制系统、约束框架、原型预览、工程基线、工程模板、代码任务包和检查清单，
让使用者能把产品想法推进到可验收的工程化交付。
```

## 2. 先读总览

执行任何新增、修改或整理动作前，必须先读取：

```text
文档/控制系统/00_系统总览.md
```

本仓库当前只分四层：

```text
文档层
契约层
门禁层
反馈层
```

新增文件前必须先判断它属于哪一层。

如果不能明确归层，或者可以合并到已有文件，不要新增文件。

## 3. 目录职责

```text
文档/        方法论、工程基线、控制系统、操作手册、教学化说明
样板/        约束框架样板、代码交付样板、控制系统元数据样板
工程模板/    Web、H5、微信小程序、iOS、后端多类独立工程模板
检查清单/    约束框架检查清单、原型确认清单、代码交付检查清单
指令/        AI 执行指令、通用大模型指令、检查指令
案例/        填好的案例
```

根目录保留 `README.md`、`AGENTS.md`。

工程模板内部允许保留生态标准文件名，例如 `package.json`、`pom.xml`、`Package.swift`、`project.config.json`、`tsconfig.json`、`src/main.ts`。

## 4. 工程交付流程

```text
先判断当前阶段
再写项目约束文档
再生成高保真交互预览
再由使用者视觉确认
再生成工程规格
再复制对应端工程模板
再生成代码任务包
再在模板工程内开发业务代码
最后用检查清单和项目命令验证
```

不得跳过阶段状态机。

不得跳过原型预览确认直接进入正式编码。

## 6. 输出要求

每次修改后说明：

1. 修改了哪些文件。
2. 为什么这样改。
3. 如何验证。
4. 还有什么风险或遗留项。
