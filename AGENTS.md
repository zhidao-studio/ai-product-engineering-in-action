# AI 执行规则

本仓库是《AI 产品工程实战》课程与实战案例仓库。

## 1. 基本原则

1. 每次只完成一个明确任务。
2. 修改前必须先阅读当前目录下的 README、AGENTS 和相关 docs。
3. 不允许随意重构无关目录。
4. 不允许删除课程文档。
5. 不允许把示例工程改成复杂工程。
6. 不允许引入大而全的框架。
7. 不允许把 token、secret、账号、密码、AppID 私密信息写入仓库。
8. 所有示例代码必须保持适合教学，优先清晰，不追求炫技。

## 2. 文档规则

1. 课程对外文档使用中文优先。
2. 文件名和目录名保留英文，便于 GitHub、工具链和 AI 识别。
3. 文档标题必须有中文名称。
4. 每个模板都要能让非开发者照着填写。
5. 不要写空泛概念，必须有例子、检查清单或交付物。

## 3. 小程序示例规则

1. 第一版只做 5 个页面：首页、详情页、收藏页、设置页、关于页。
2. 第一版只使用 Mock 数据和本地存储。
3. 不接入后端、登录、支付、地图、云开发。
4. 不引入复杂状态管理。
5. 页面必须考虑正常、空、错误、加载状态。

## 4. 完成任务后必须输出

1. 修改了哪些文件。
2. 实现了什么。
3. 如何运行或检查。
4. 是否有风险。
5. 后续建议。
