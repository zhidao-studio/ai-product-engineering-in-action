# 后端检查清单

## 1. 技术栈

- [ ] 使用 Java
- [ ] 使用 Spring Boot
- [ ] 使用 Maven 或 Maven Wrapper
- [ ] 明确不使用 Spring Cloud
- [ ] 后端是单体应用

## 2. 目录结构

- [ ] `controller/v1/` 只放 API v1 控制器
- [ ] `domain/*/bo/` 放业务入参对象
- [ ] `domain/*/vo/` 放接口返回对象
- [ ] `service/` 放 Service 接口
- [ ] `service/impl/` 放 Service 实现
- [ ] `repository/` 或 `mapper/` 放数据访问
- [ ] `exception/` 放异常和错误响应
- [ ] `config/` 放配置

## 3. API 版本和多端

- [ ] 新接口默认使用 `/api/v1`
- [ ] 不兼容变更新增 `/api/v2`，不破坏 v1
- [ ] 多端共用接口契约
- [ ] 如需客户端类型，使用 `X-Client-Type`
- [ ] 客户端类型没有在业务逻辑中形成大量条件分支

## 4. 编码约束

- [ ] Controller 没有业务逻辑
- [ ] Controller 没有直接访问数据库
- [ ] Controller 只依赖 Service 接口
- [ ] Service 存在 interface 和 impl
- [ ] Service 没有处理 HTTP 细节
- [ ] Repository / Mapper 没有业务判断
- [ ] 错误响应格式统一
- [ ] 配置不包含真实密钥

## 5. 验证

- [ ] `./mvnw test` 可通过
- [ ] `./mvnw spring-boot:run` 可启动
- [ ] API 返回符合契约
- [ ] 错误响应符合契约
- [ ] Service 接口和实现均有对应测试计划
