# 后端检查清单

## 1. 技术栈

- [ ] 使用 Java 17+
- [ ] 使用 Spring Boot 3
- [ ] 使用 Maven 或 Maven Wrapper
- [ ] 使用 Spring Web
- [ ] 使用 Spring Validation
- [ ] 使用 MyBatis-Plus 作为默认数据访问框架
- [ ] 使用 Flyway 管理数据库结构变更
- [ ] 明确不使用 Spring Cloud
- [ ] 后端是单体应用

## 2. 目录结构

- [ ] `controller/v1/` 只放 API v1 控制器
- [ ] `domain/*/entity/` 放数据库实体对象
- [ ] `domain/*/bo/` 放业务入参对象
- [ ] `domain/*/vo/` 放接口返回对象
- [ ] `mapper/` 放 MyBatis-Plus Mapper
- [ ] `service/` 放 Service 接口
- [ ] `service/impl/` 放 Service 实现
- [ ] `exception/` 放异常和错误响应
- [ ] `config/` 放配置
- [ ] `resources/db/migration/` 放 Flyway migration

## 3. API 版本和多端

- [ ] 新接口默认使用 `/api/v1`
- [ ] 不兼容变更新增 `/api/v2`，不破坏 v1
- [ ] 多端共用接口契约
- [ ] 如需客户端类型，使用 `X-Client-Type`
- [ ] 客户端类型没有在业务逻辑中形成大量条件分支

## 4. 编码约束

- [ ] Controller 没有业务逻辑
- [ ] Controller 没有直接访问数据库
- [ ] Controller 只依赖 Service 接口
- [ ] Service 存在 interface 和 impl
- [ ] Service 没有处理 HTTP 细节
- [ ] Mapper 没有业务判断
- [ ] Entity 没有直接暴露给前端
- [ ] BO 有参数校验
- [ ] VO 只返回接口契约允许的字段
- [ ] 错误响应格式统一
- [ ] 配置不包含真实密钥

## 5. 数据库约束

- [ ] 新表有 Flyway migration
- [ ] 主键、索引、非空字段已定义
- [ ] 创建时间和更新时间字段已定义
- [ ] 字段命名和接口契约可追溯
- [ ] 不直接修改生产表结构说明而跳过 migration

## 6. 验证

- [ ] `./mvnw test` 可通过
- [ ] `./mvnw package` 可通过
- [ ] `./mvnw spring-boot:run` 可启动
- [ ] API 返回符合契约
- [ ] 错误响应符合契约
- [ ] Service 接口和实现均有对应测试计划
- [ ] CRUD 样板可作为新业务模块参考
