# 前端检查清单

## 1. 技术栈

- [ ] 使用 Vue 3
- [ ] 使用 Vite
- [ ] 使用 TypeScript
- [ ] 使用 Vue Router
- [ ] 状态管理使用 Pinia
- [ ] 使用 ESLint 和 Prettier

## 2. 目录结构

- [ ] 页面放在 `src/pages/`
- [ ] 组件放在 `src/components/`
- [ ] API 调用放在 `src/services/`
- [ ] 请求和错误拦截放在统一位置
- [ ] 状态放在 `src/stores/`
- [ ] 类型放在 `src/types/`
- [ ] 样式放在 `src/styles/`

## 3. 命名规范

- [ ] 已阅读 `文档/工程基线/06_前后端命名规范.md`
- [ ] 页面组件使用 PascalCase，并以 Page 结尾
- [ ] 业务组件使用业务名 + 组件类型
- [ ] Service 文件使用业务名 + Service
- [ ] Store 文件使用业务名 + Store
- [ ] Store hook 使用 `useXxxStore`
- [ ] TypeScript 类型使用 PascalCase
- [ ] 请求类型以 Request 结尾
- [ ] 响应类型以 Response 结尾
- [ ] API 字段使用 lowerCamelCase
- [ ] 没有为同一业务概念新增多个英文名

## 4. 编码约束

- [ ] 页面中没有直接散落 fetch
- [ ] API 地址没有硬编码
- [ ] 字段来自数据规范和接口契约
- [ ] 页面状态包含 loading / empty / success / error
- [ ] 样式符合设计规格
- [ ] 只有跨页面共享状态进入 store
- [ ] 页面内临时状态没有被无差别放入 store

## 5. 错误处理

- [ ] API 错误经过统一拦截或统一请求封装
- [ ] 401 行为符合接口契约
- [ ] 网络错误展示用户可理解提示
- [ ] 表单校验错误在字段附近或表单区域展示
- [ ] 不向用户展示原始异常堆栈
- [ ] 没有每个接口各写一套不同错误处理逻辑

## 6. 验证

- [ ] `npm install` 可执行
- [ ] `npm run dev` 可启动
- [ ] `npm run build` 可通过
- [ ] 关键页面可访问
- [ ] 关键交互可完成
- [ ] 前端核心交互有测试任务或测试说明
