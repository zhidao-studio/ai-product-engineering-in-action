# 常见问题

## 1. 不会开发真的能学吗？

可以。

但本课程不是承诺“完全不用思考”，而是教你用文档控制需求实现。

你需要学会：

- 写清需求边界
- 拆页面和流程
- 写任务清单
- 约束 Agent 执行
- 审查和验收结果

## 2. 这门课是不是教我用 AI 一键生成产品？

不是。

课程核心是：

```text
通过一整套完整文档，控制需求如何被实现。
```

AI / Agent 只是执行层，不是决策层。

## 3. 为什么不直接让 Agent 写代码？

因为直接写代码会失控。

常见问题：

- Agent 自行扩功能
- Agent 自行决定页面
- Agent 自行发明字段
- Agent 自行引入依赖
- 最后代码看起来很多，但无法判断是否正确

所以必须先有文档控制体系。

## 4. 我能不能用别的模型或 Agent？

可以。

本课程不绑定某一个 Agent。

你可以使用 Codex、Cursor、Windsurf、Kimi Coding、Trae、MarsCode、GitHub Copilot 或其他代码生成 Agent。

前提是它必须遵守：

- 依赖文档
- 任务边界
- 允许修改范围
- 禁止修改范围
- 验收标准

## 5. 为什么第一版优先 H5？

H5 只是第一版承载形态，不是课程核心。

优先 H5 的原因是：

- 浏览器即可运行
- 不需要平台账号
- 不需要专门开发者工具
- 更容易讲清页面、数据、交互和文档映射

后续可以扩展到微信小程序、iOS App、Web App 等形态。

## 6. 文档是不是浪费时间？

不是。

文档不是说明书，而是控制器。

```text
先写文档，
再用文档控制 Agent，
最后用文档验收结果。
```

没有文档，看似快，实际上更容易返工。

## 7. 写完文档后，Agent 还会不会出错？

会。

文档不能保证 Agent 永远正确，但可以让错误更容易定位：

- 是 PRD 不清楚？
- 是页面清单缺失？
- 是用户流程断了？
- 是数据字段没定义？
- 是任务范围太大？
- 是 Agent 没按任务执行？

## 8. 如果需求变了怎么办？

不能直接改代码。

正确流程：

```text
先判断是否属于当前版本
↓
更新相关文档
↓
更新任务清单
↓
再让 Agent 实现
↓
最后审查和验收
```

## 9. 什么情况下不能进入实现？

以下情况不要进入实现：

- 没有项目总说明
- 没有 PRD
- 没有页面清单
- 没有用户流程
- 没有数据规范
- 没有技术结构
- 没有任务清单
- 没有验收标准

## 10. 最重要的一句话是什么？

```text
需求没有进文档，就不能进代码。
代码不能回溯到文档，就不能通过验收。
```
