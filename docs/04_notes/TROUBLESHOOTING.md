# 问题排查手册

## 1. Agent 生成的代码跑不起来怎么办？

按顺序做：

1. 复制完整报错。
2. 不要让 Agent 大范围重写。
3. 检查当前任务和允许修改范围。
4. 用修复提示词让 Agent 只修当前错误。
5. 如果连续修 3 次还不对，停止修复，回到文档和任务定义。

修复提示词：

```text
请只修复下面这个报错，不要做无关重构。

当前任务：
【粘贴 TASK】

相关文档：
【粘贴相关 PRD / 页面清单 / 用户流程 / 数据规范】

允许修改：
【列出允许修改文件】

禁止修改：
- 不改无关文件
- 不新增功能
- 不引入新依赖
- 不改变数据字段
- 不改变页面范围

报错：
【粘贴报错】

完成后请说明：
1. 根因
2. 修改文件
3. 对应文档
4. 验证方式
```

## 2. 怎么判断是 Agent 的问题还是文档的问题？

优先检查：

- 是否有项目总说明。
- 是否有 PRD。
- 是否有页面清单。
- 是否有用户流程。
- 是否有数据规范。
- 是否有技术结构。
- 当前任务是否太大。
- 是否写清允许修改范围。
- 是否写清禁止修改范围。
- Agent 是否修改了无关文件。

## 3. Agent 越修越乱怎么办？

立刻停止。

处理方式：

1. 查看 git diff。
2. 回滚无关修改。
3. 检查当前任务是否太大。
4. 检查相关文档是否缺失。
5. 把任务拆小。
6. 重新给 Agent 明确允许修改范围。
7. 必要时回到上一个 commit。

## 4. 什么时候该放弃当前实现方案？

满足任一情况就要停：

- Agent 连续 3 次修复失败。
- 任务已经超出第一版范围。
- 需要引入复杂后端、登录、支付、地图等未批准能力。
- 一个小功能变成多个大改动。
- 你已经看不懂改了什么。
- 代码无法回溯到文档。

## 5. 页面显示空白怎么办？

检查：

1. 是否打开了正确目录。
2. 是否有控制台报错。
3. 数据是否为空。
4. 字段名是否和数据规范一致。
5. 页面结构是否来自页面清单。
6. 交互逻辑是否来自用户流程。
7. Agent 是否改了无关文件。

## 6. 功能能跑，但感觉不对怎么办？

不要只看能不能跑。

继续检查：

- 是否符合 PRD。
- 是否符合页面清单。
- 是否符合用户流程。
- 是否符合数据规范。
- 是否满足验收标准。
- 用户动作后是否有结果。

如果不符合，必须转成修复任务。

## 7. 需求变了怎么办？

不要直接改代码。

正确流程：

```text
判断是否属于当前版本
↓
修改相关文档
↓
更新任务清单
↓
让 Agent 按新任务实现
↓
再审查和验收
```

## 8. 一句话原则

```text
先查文档，再查任务，最后查代码。
```
