# 术语表

| 英文 | 中文 | 解释 |
|---|---|---|
| PRD | 产品需求文档 | 说明产品功能、场景和验收标准 |
| MVP | 最小可行版本 | 第一版只做最关键功能，用来验证产品是否值得继续 |
| Prompt | 提示词 | 给 AI 的任务说明 |
| Agent | AI 执行助手 | 能按任务执行的 AI 工具 |
| Mock Data | 模拟数据 | 后端或真实数据未准备好前，用于开发和验证流程的数据 |
| Code Review | 代码审查 | 检查代码是否可靠、是否符合任务要求 |
| Checklist | 检查清单 | 按项检查是否完成 |
| Changelog | 变更记录 | 记录每次修改 |
| Sample | 样例 | 课程中提供的可参考、可运行、可跟做的标准案例 |
| Example Project | 样例工程 | 课程中用于教学和实战的完整工程 |
| Practical Sample | 实战样例 | 跟随课程一步一步完成的真实练习项目 |

## 课程统一表述

本课程统一使用：

- 样例
- 实战样例
- 样例工程
- 第一版可运行样例
- 第一版可运行样例产品

避免使用：

- Demo
- App Demo
- 小程序 Demo
- 演示版本
- 演示样品

原因：

课程不是展示一个临时效果，而是带学员完成一个结构清晰、可运行、可继续维护的样例产品。
