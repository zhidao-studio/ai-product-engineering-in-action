|---|---|---|
| Document Control System | 文档控制体系 | 用一整套文档控制需求如何被实现 |

| Prompt | 提示词 | 给 AI / Agent 的任务说明 |

| AI Coding Agent | 代码生成 Agent | 能读取工程、修改代码、修复问题的 AI 工具，例如 Codex、Cursor、Windsurf、Kimi Coding、Trae、Copilot 等 |
| Mock Data | 模拟数据 | 后端或真实数据未准备好前，用于开发和验证流程的数据 |

| Code Review | 代码审查 | 检查代码是否可靠、是否符合文档和任务要求 |
| Human Acceptance | 模拟用户验收 | 从真实用户视角检查产品是否能理解、能操作、能完成核心流程 |
| Checklist | 检查清单 | 按项检查是否完成 |
| Changelog | 变更记录 | 记录每次修改 |
| Sample | 样例 | 课程中提供的可参考、可运行、可跟做的标准案例 |
| Example Project | 样例工程 | 课程中用于教学和实战的完整工程 |
| Practical Sample | 实战样例 | 跟随课程一步一步完成的真实练习项目 |

## 文档控制体系

本课程中的“文档”不是代码写完后的说明书，而是需求实现前的控制器。

它用于控制：

- 产品边界
- 功能范围
- 页面范围
- 用户流程
- 数据结构
- 技术结构
- Agent 执行范围
- 验收标准

一句话：

```text
先写文档，
再用文档控制 Agent，
最后用文档验收结果。
```

## Agent 和 AI 编码工具的关系

`Agent` 是更宽泛的概念，表示“能按任务执行的 AI 助手”。

本课程中更准确的说法是：

```text
代码生成 Agent
```

它特指能进入工程、读取文件、修改代码、修复问题的工具，例如：

- Codex
- Cursor
- Windsurf
- Kimi Coding
- Trae / MarsCode
- GitHub Copilot

因此，学员可以这样理解：

```text
Agent 是大概念；
代码生成 Agent 是执行层；
文档控制体系是控制层。
```

## 课程统一表述

本课程统一使用：

- 样例
- 实战样例
- 样例工程
- 第一版可运行样例
- 第一版可运行样例产品
- Agent
- 文档控制体系
- 文档到代码映射

避免使用：

- Demo
- App Demo
- 小程序 Demo
- 演示版本
- 演示样品

原因：

课程不是展示一个临时效果，而是带学员完成一套结构清晰、可运行、可继续维护的文档控制和实现流程。
