# 非技术用户决策树

## 1. 是否进入文档控制体系

```text
这个想法是否已经验证过目标用户和真实痛点？
├─ 是：进入项目总说明
└─ 否：先做产品想法验证
```

```text
能否说清第一版核心闭环？
├─ 能：进入 PRD
└─ 不能：继续压缩范围
```

## 2. 产品复杂度判断

```text
你的第一版有几个页面？
├─ 1-3 个页面：适合做第一版可运行样例
├─ 4-5 个页面：适合作为课程第一版
├─ 6-8 个页面：需要砍功能，只保留核心闭环
└─ 8 个以上：先不要实现，重新拆 MVP
```

## 3. 功能复杂度判断

```text
第一版是否需要登录？
├─ 不需要：优先不做登录
└─ 需要：先问是否可以用本地数据替代
```

```text
第一版是否需要后端？
├─ 不需要：先用 Mock 数据和本地存储
└─ 需要：先确认是否必须，能否延后
```

```text
第一版是否需要支付 / IM / 直播 / 复杂地图？
├─ 是：第一版先砍掉
└─ 否：可以继续进入文档拆解
```

## 4. 文档完整度判断

```text
是否已有项目总说明和 PRD？
├─ 有：继续拆页面和流程
└─ 没有：不能进入实现
```

```text
是否已有页面清单、用户流程、数据规范？
├─ 有：继续写技术结构和任务清单
└─ 没有：不能让 Agent 写代码
```

```text
是否已有文档到代码映射？
├─ 有：可以进入审查和验收
└─ 没有：实现后无法判断是否正确
```

## 5. Agent 执行判断

```text
一个任务是否能一句话说清？
├─ 能：继续检查任务边界
└─ 不能：先拆小任务
```

```text
任务是否写清依赖文档、允许修改、禁止修改、验收标准？
├─ 有：可以交给 Agent 执行
└─ 没有：先补任务清单
```

```text
Agent 连续修 3 次还不对？
├─ 是：停止继续修，回到文档和任务定义
└─ 否：继续小步修复
```

## 6. 是否继续项目

```text
给 3-5 个目标用户试用后，有人愿意继续反馈吗？
├─ 有：进入下一版迭代
├─ 没有：重新验证痛点
└─ 反馈完全看不懂：先优化产品表达和核心流程
```

## 7. 第一版上限规则

非技术用户第一版建议遵守：

- 页面不超过 5 个
- 核心流程不超过 1 条
- 不做登录
- 不做支付
- 不做复杂后端
- 不做复杂权限
- 不同时使用多个 Agent 改同一任务
- 需求没有进入文档，不进入实现
- 代码不能回溯到文档，不通过验收
