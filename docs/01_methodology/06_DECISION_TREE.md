# 非技术用户决策树

## 1. 产品复杂度判断

```text
你的第一版有几个页面？
├─ 1-3 个页面：可以直接做 Demo
├─ 4-5 个页面：适合作为课程第一版
├─ 6-8 个页面：需要砍功能，只保留核心闭环
└─ 8 个以上：先不要开发，重新拆 MVP
```

## 2. 功能复杂度判断

```text
第一版是否需要登录？
├─ 不需要：优先不做登录
└─ 需要：先问是否可以用本地数据替代
```

```text
第一版是否需要后端？
├─ 不需要：先用 Mock 数据和本地存储
└─ 需要：先确认接口最少需要几个
```

```text
第一版是否需要支付 / IM / 直播 / 复杂地图？
├─ 是：第一版先砍掉
└─ 否：可以继续
```

## 3. AI 执行判断

```text
一个任务是否能一句话说清？
├─ 能：可以交给 AI 编码工具
└─ 不能：先让 ChatGPT 拆小
```

```text
一个任务是否有验收标准？
├─ 有：可以开始
└─ 没有：先补验收标准
```

```text
AI 连续修 3 次还不对？
├─ 是：停止继续修，回到任务定义
└─ 否：继续小步修复
```

## 4. 是否继续项目

```text
给 3-5 个目标用户试用后，有人愿意继续反馈吗？
├─ 有：进入下一版迭代
├─ 没有：重新验证痛点
└─ 反馈完全看不懂：先优化产品表达和核心流程
```

## 5. 第一版上限规则

非技术用户第一版建议遵守：

- 页面不超过 5 个
- 核心流程不超过 1 条
- 不做登录
- 不做支付
- 不做复杂后端
- 不做复杂权限
- 不同时使用多个 AI 编码工具改同一功能
