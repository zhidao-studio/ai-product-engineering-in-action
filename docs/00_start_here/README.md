# 从这里开始

## 1. 本仓库解决什么问题

本仓库不是一个具体产品样例，也不是某个 AI 编码工具教程。

它要定义一套：

```text
文档驱动的 AI 产品交付方法论。
```

目标是提供可复用的 harness 模板和 checklist，让非技术人能通过文档工程约束 AI，完成可验收、可追溯、可维护的产品交付。

## 2. 最重要的一句话

```text
不是让 AI 替你做产品，
而是用 harness 约束 AI 完成可验收的产品交付。
```

## 3. 三个目标层级

| 优先级 | 目标 | 说明 |
|---:|---|---|
| 1 | 方法论目标 | 定义文档驱动的 AI 产品交付流程 |
| 2 | 工具化目标 | 提供可复用的 harness 模板和 checklist |
| 3 | 教学目标 | 让非技术人能学会并实际使用 |

## 4. 你应该先读什么

建议顺序：

```text
1. docs/01_methodology/01_GOAL_AND_SCOPE.md
2. docs/01_methodology/02_CORE_PRINCIPLES.md
3. docs/01_methodology/03_HARNESS_LAYERS.md
4. docs/01_methodology/04_HOW_TO_BUILD_HARNESS.md
5. templates/harness/
6. checklists/harness/
7. examples/01_minimal_harness/
```

## 5. 读完后你应该能做什么

你应该能：

- 判断一个需求是否值得进入 AI 交付。
- 用文档锁定做什么和不做什么。
- 用设计规格锁定页面长什么样。
- 用工程规格锁定前端、后端、接口怎么实现。
- 用任务 harness 控制 AI 每次只做什么。
- 用 checklist 判断 AI 是否做对。
- 用变更规则保证后续迭代不失控。
