# 工具准备与成本意识

## 1. 为什么要讲成本

非开发者用 AI 做产品，容易只看到“AI / Agent 能执行任务”，忽略真实成本。

成本不只是钱，还包括：

- 文档成本
- 学习成本
- 工具费用
- 时间成本
- 试错成本
- 发布成本
- 维护成本

## 2. 第一版建议工具

| 工具 | 用途 | 成本建议 |
|---|---|---|
| ChatGPT / 通用大模型 | 梳理想法、生成文档、拆任务、模拟验收 | 可先用已有订阅 |
| 文档编辑器 | 编写项目总说明、PRD、页面清单、任务清单 | Markdown 即可 |
| 代码生成 Agent | 按任务实现代码 | 先选一个主力，不要同时开太多 |
| 审查模型 | 检查实现是否符合文档 | 可用 DeepSeek / ChatGPT 等 |
| GitHub | 托管文档和代码，记录变更 | 免费版足够 |
| VS Code | 查看文档和代码 | 免费 |
| 浏览器 | 预览 H5 第一版样例 | 免费 |

## 3. 第一版不建议花钱的地方

- 不急着买服务器。
- 不急着买域名。
- 不急着做付费 UI 设计。
- 不急着接支付。
- 不急着做云同步。
- 不急着做复杂后端。
- 不急着上平台专属能力。

## 4. 第一版建议策略

```text
先写文档控制体系
↓
先用 Mock 数据
↓
先跑通本地或浏览器功能
↓
先验证用户是否能完成核心流程
↓
再决定是否接后端、云服务或平台能力
```

## 5. 成本检查问题

在开始做之前问自己：

1. 这个产品第一版是否可以不用后端？
2. 是否可以不用登录？
3. 是否可以不用地图？
4. 是否可以不用支付？
5. 是否可以先用 H5 或轻量页面验证？
6. 是否已经写清不做清单？
7. 是否已经写清任务边界？
8. 是否可以先给 3-5 个真实用户试用？

## 6. 文档成本不是浪费

很多人会觉得写文档慢。

但没有文档会带来更高成本：

```text
Agent 理解错需求
↓
实现越界
↓
反复返工
↓
不知道哪里错
↓
最终无法维护
```

文档的作用是提前降低返工成本。

## 7. 一句话原则

```text
第一版省的是复杂技术成本，
不能省的是需求控制成本。
```
