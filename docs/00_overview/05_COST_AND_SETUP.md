# 工具准备与成本意识

## 1. 为什么要讲成本

非开发者用 AI 做产品，容易只看到“AI 能写代码”，忽略真实成本。

成本不只是钱，还包括：

- 学习成本
- 工具费用
- 时间成本
- 试错成本
- 发布成本
- 维护成本

## 2. 第一版建议工具

| 工具 | 用途 | 成本建议 |
|---|---|---|
| ChatGPT / 通用大模型 | 需求整理、任务拆分、验收 | 可先用已有订阅 |
| AI 编码工具 | 进入工程改代码 | 先选一个主力，不要同时开太多 |
| DeepSeek | 审查代码、检查逻辑 | 可作为低成本审查模型 |
| Kimi | 长文档、竞品资料、中文文案 | 可作为资料整理工具 |
| 微信开发者工具 | 小程序开发和预览 | 免费 |
| GitHub | 代码托管 | 免费版足够 |
| VS Code | 文档和代码查看 | 免费 |

## 3. 第一版不建议花钱的地方

- 不急着买服务器。
- 不急着买域名。
- 不急着做付费 UI 设计。
- 不急着接支付。
- 不急着做云同步。
- 不急着做复杂后端。

## 4. 第一版建议策略

```text
先用 Mock 数据
↓
先跑通本地功能
↓
先验证用户是否需要
↓
再决定是否接后端和云服务
```

## 5. 成本检查问题

在开始做之前问自己：

1. 这个产品第一版是否可以不用后端？
2. 是否可以不用登录？
3. 是否可以不用地图？
4. 是否可以不用支付？
5. 是否可以先用 5 个页面验证？
6. 是否可以先给 3-5 个真实用户试用？
