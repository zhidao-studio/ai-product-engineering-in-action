# 课程总说明

## 课程名称

AI 产品工程实战

副标题：

构造 AI 驱动交付 Harness

## 一句话定位

这是一套面向非开发者的 AI 产品工程实战教程。

课程目标不是教你让 AI 一次性生成产品，而是教你构造一套可复用的产品交付 Harness，用产品定义、设计规格、工程规格、任务执行、验证、体验验收、交付迭代等多层约束，控制 AI 如何交付产品。

核心链路：

```text
产品定义锁边界
↓
设计规格锁视觉和交互
↓
工程规格锁前端 / 后端 / 接口
↓
任务 Harness 锁本次执行范围
↓
验证 Harness 判断是否做对
↓
体验验收判断用户能不能用
↓
交付迭代保证变更可控
```

## 目标用户

- 有产品想法，但不会开发的人
- 会一点技术，但无法独立完成完整产品的人
- 想用 AI 做网站、小工具、H5、微信小程序、App 的个人创业者
- 产品经理、运营、设计师、内容创作者
- 想用 AI 做副业产品的人

## 课程承诺

学完后，学员应该能做到：

1. 构造产品定义 Harness：产品验证、项目总说明、PRD。
2. 构造设计规格 Harness：视觉风格、配色、组件、布局、原型。
3. 构造工程规格 Harness：数据规范、前端规格、后端 API、数据库规格、技术架构。
4. 构造任务执行 Harness：任务清单、允许修改、禁止修改、验收标准、输出格式。
5. 构造验证 Harness：代码审查、功能测试、视觉比对、数据一致性、文档追溯。
6. 构造体验验收 Harness：用户验收脚本、体验标准、边缘场景、多设备清单。
7. 构造交付与迭代 Harness：部署清单、变更协议、反馈分类、迭代记录。
8. 判断 AI 产物是文档漏洞，还是 Agent 没遵守约束。

## 课程不承诺

- 不承诺一键生成商业级产品。
- 不承诺完全不用思考、不用验收。
- 不承诺学完就能开发大型复杂系统。
- 不承诺产品一定赚钱。
- 不鼓励第一版就做复杂产品。

## 第一版实战样例定位

第一版实战样例优先采用 H5 页面 / 轻量 Web App。

原因：

- 浏览器即可运行。
- 不需要平台账号和开发者工具。
- 文件结构简单，适合讲清 Harness 如何约束产品交付。
- 后续可以迁移到微信小程序、iOS App、Web App 等更多形态。

注意：

```text
H5 只是第一版承载形态，不是课程核心。
课程核心是：构造 AI 驱动交付 Harness。
```

## 最小闭环

第一版产品只验证一个最小闭环：

```text
用户看到内容
↓
进入详情
↓
执行核心动作
↓
看到动作结果
```

不同产品形态可以替换为不同样例。

但无论形态如何变化，Harness 架构不变。

## 课程核心观点

本课程不只讲“如何让 Agent 执行任务”，更重要的是讲：

- 如何定义产品边界
- 如何锁定设计规格
- 如何拆清前端、后端、接口工程边界
- 如何用任务 Harness 控制 Agent 执行范围
- 如何用验证 Harness 判断是否做对
- 如何用体验验收 Harness 判断用户能不能用
- 如何用交付迭代 Harness 管理上线后的变化

一句话：

```text
不是用 AI 替代文档，
而是用 Harness 控制 AI。
```
