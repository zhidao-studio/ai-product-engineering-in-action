# 课程总说明

## 课程名称

AI 产品工程实战

副标题：

用完整文档控制需求实现

## 一句话定位

这是一套面向非开发者的 AI 产品工程实战教程。

课程目标不是教你让 AI 一次性生成产品，而是教你用一整套文档控制需求如何被实现。

核心链路：

```text
需求进入文档
↓
文档拆成任务
↓
任务约束 Agent
↓
Agent 执行任务
↓
代码回溯文档
↓
验收判断结果
```

## 目标用户

- 有产品想法，但不会开发的人
- 会一点技术，但无法独立完成完整产品的人
- 想用 AI 做网站、小工具、H5、微信小程序、App 的个人创业者
- 产品经理、运营、设计师、内容创作者
- 想用 AI 做副业产品的人

## 课程承诺

学完后，学员应该能做到：

1. 把模糊想法整理成《项目总说明》。
2. 写出基础《产品需求文档》。
3. 拆出《页面清单》和《用户流程》。
4. 写出《数据规范》和《技术结构》。
5. 把需求拆成 Agent 可以执行的小任务。
6. 用任务清单约束 Agent 的实现范围。
7. 用《文档到代码映射》审查 Agent 产物。
8. 用模拟用户验收判断产品是否真正可交付。

## 课程不承诺

- 不承诺一键生成商业级产品。
- 不承诺完全不用思考、不用验收。
- 不承诺学完就能开发大型复杂系统。
- 不承诺产品一定赚钱。
- 不鼓励第一版就做复杂产品。

## 第一版实战样例定位

第一版实战样例优先采用 H5 页面 / 轻量 Web App。

原因：

- 浏览器即可运行。
- 不需要平台账号和开发者工具。
- 文件结构简单，适合讲清“文档如何控制实现”。
- 后续可以迁移到微信小程序、iOS App、Web App 等更多形态。

注意：

```text
H5 只是第一版承载形态，不是课程核心。
课程核心是：通过完整文档控制需求实现。
```

## 最小闭环

第一版产品只验证一个最小闭环：

```text
用户看到内容
↓
进入详情
↓
执行核心动作
↓
看到动作结果
```

不同产品形态可以替换为不同样例。

但无论形态如何变化，文档控制链路不变。

## 课程核心观点

本课程不只讲“如何让 Agent 执行任务”，更重要的是讲：

- 如何验证产品想法
- 如何定义需求边界
- 如何拆清楚页面、流程和数据
- 如何用文档控制 Agent 的执行范围
- 如何审查 Agent 产出
- 如何模拟真实用户验收
- 如何持续维护和迭代

一句话：

```text
不是用 AI 替代文档，
而是用文档控制 AI。
```
