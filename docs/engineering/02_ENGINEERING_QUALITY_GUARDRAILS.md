# 工程质量红线

## 1. 目的

本文件用于补充工程基线，明确 AI 写代码时必须遵守的质量红线。

它关注：

```text
框架依赖
工程分层
安全
可维护性
测试
配置
日志
错误处理
```

## 2. 框架依赖红线

### 前端

允许：

```text
Vue 3
Vite
TypeScript
Vue Router
Pinia
ESLint
Prettier
Vitest
Playwright 或 Cypress
```

需要审批后才能引入：

```text
UI 组件库
图表库
动画库
请求库
工具函数库
```

禁止 AI 自行引入：

```text
React
Angular
Nuxt
大型 UI 套件
低维护频率依赖
不明确来源的依赖
```

### 后端

允许：

```text
Spring Boot
Spring Web
Spring Validation
Spring Data JPA 或 MyBatis（二选一，按项目决定）
Spring Security（需要登录认证时再引入）
Flyway 或 Liquibase（二选一，按项目决定）
```

禁止 AI 自行引入：

```text
Spring Cloud
注册中心
配置中心
服务网关
消息队列
分布式事务
复杂微服务框架
```

## 3. 分层红线

### 前端分层

```text
pages       页面级组件
components  可复用组件
services    API 调用
types       类型定义
stores      状态管理
styles      样式和设计 token
utils       通用工具
```

红线：

- 页面不能直接写 API 请求。
- 页面不能直接硬编码后端地址。
- 组件不能包含业务流程编排。
- services 不能包含 UI 逻辑。
- stores 不能直接操作 DOM。

### 后端分层

```text
api         Controller，请求响应
service     业务逻辑
repository  数据访问
model       实体或领域对象
dto         请求和响应对象
config      配置
exception   异常和错误响应
```

红线：

- Controller 不写业务逻辑。
- Controller 不直接访问数据库。
- Repository 不写业务判断。
- Service 不处理 HTTP 细节。
- DTO 不直接复用数据库实体给前端。

## 4. 安全红线

AI 不得：

- 提交真实密码、token、密钥。
- 把 ngrok 地址写死进代码。
- 把数据库端口暴露到公网。
- 在前端保存敏感 token。
- 关闭安全校验来解决问题。
- 忽略输入校验。
- 直接拼接 SQL。
- 返回后端异常堆栈给前端。

必须做到：

- 使用 `.env.example` 说明环境变量。
- 使用统一错误响应。
- 对请求参数做校验。
- 对外部输入做合法性检查。
- 对 CORS 做明确配置，不使用无限制通配。

## 5. 可维护性红线

AI 不得：

- 把所有代码写进一个文件。
- 一次任务重构大量无关文件。
- 新增无文档说明的工具函数。
- 复制粘贴大量相似逻辑。
- 用魔法字符串表达业务状态。
- 删除注释或 README 中的关键启动说明。

必须做到：

- 文件职责单一。
- 类型定义清楚。
- 错误处理集中。
- 业务状态使用枚举或常量。
- 代码改动能追溯到任务包。

## 6. 测试红线

前端至少需要：

- 构建通过
- 核心组件或核心流程测试
- 关键页面状态测试

后端至少需要：

- Service 单元测试
- API 基础测试
- 参数校验测试
- 错误响应测试

AI 不得用“未测试”作为完成结论。

## 7. 配置和环境红线

- `.env` 不提交。
- `.env.example` 必须提交。
- 本地端口必须和 infra 文档一致。
- 配置不得散落在业务代码里。
- 前端后端地址通过环境变量或统一配置读取。

## 8. 日志和错误处理红线

后端：

- 使用统一日志方式。
- 不打印敏感信息。
- 不把异常堆栈返回给前端。
- 错误码必须可读、可追踪。

前端：

- 用户看到的是可理解提示。
- 开发日志不能影响用户界面。
- API 失败必须有错误状态。

## 9. AI 执行红线

AI 每次代码交付必须说明：

```text
改了哪些文件
为什么改
对应哪个任务包
如何启动
如何验证
风险点
是否需要补 harness 或工程规范
```

## 10. 一句话原则

```text
工程质量红线不是为了限制开发，
而是为了防止 AI 写出能跑但不可维护的代码。
```
