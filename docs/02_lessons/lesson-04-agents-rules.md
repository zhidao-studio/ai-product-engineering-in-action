# 第 4 课：把文档转成技术结构和 Agent 任务边界

## 学习目标

学完本课，学员应该能把页面、流程和数据规范继续转成：

- 技术结构
- 任务清单
- Agent 执行规则

## 本节核心观点

Agent 很强，但必须有边界。

没有规则，它可能：

- 改无关文件
- 引入无关依赖
- 重构整个项目
- 新增文档没有定义的功能
- 自行改变数据结构

所以 Agent 执行前，必须先用文档规定技术结构和任务边界。

## 本节内容

1. 用技术结构控制文件职责。
2. 用任务清单控制 Agent 每次改什么。
3. 用 Agent 执行规则控制允许和禁止事项。
4. 明确 Agent 可以替换，但文档控制流程不能省。

## 学员产物

完成：

```text
ARCHITECTURE.md
TASKS.md
AGENTS.md
```

## 常见错误

1. 技术结构由 Agent 自己决定。
2. 任务没有允许修改范围。
3. 任务没有禁止修改范围。
4. 一个任务做太多事情。
5. 同时让多个 Agent 改同一个任务。

## 课后作业

为你的项目写技术结构、任务清单和 Agent 执行规则，并检查：

```text
每个文件职责是否清楚？
每个任务是否只做一件事？
Agent 是否知道哪些不能改？
```
