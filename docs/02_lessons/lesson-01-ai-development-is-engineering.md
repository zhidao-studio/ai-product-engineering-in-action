# 第 1 课：核心不是写代码，是用文档控制需求实现

## 学习目标

学完本课，学员应该理解本课程的核心主线：

```text
通过一整套完整文档，控制需求如何被实现。
```

## 本节核心观点

AI / Agent 可以帮助执行任务，但不能替你做产品判断。

可维护产品不是一句“帮我做一个产品”就够了。

它需要：

- 产品边界
- 功能范围
- 页面清单
- 用户流程
- 数据规范
- 技术结构
- 任务清单
- 文档到代码映射
- 审查和验收

## 本节内容

本节讲清楚：

1. 为什么不能跳过文档直接实现。
2. 文档工程为什么是控制层。
3. Agent 为什么只是执行层。
4. 学员为什么必须负责判断和验收。
5. 后续每一节课如何逐步建立文档控制体系。

## 学员产物

完成一张《文档控制体系总览》。

## 常见错误

1. 直接让 Agent 做完整产品。
2. 没有写清第一版边界。
3. 没有写“不做什么”。
4. 需求没有进入文档就进入实现。
5. Agent 完成后不做文档映射和用户验收。

## 课后作业

用一句话写出你的产品想法，并回答：

```text
这个想法需要哪些文档控制？
哪些内容不能交给 Agent 自行决定？
```
