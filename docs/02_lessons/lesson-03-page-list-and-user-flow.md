# 第 3 课：用页面、流程和数据继续控制需求

## 学习目标

学完本课，学员应该能把 PRD 拆成三份控制文档：

- 页面清单
- 用户流程
- 数据规范

## 本节核心观点

Agent 实现页面前，必须先有页面清单、用户流程和数据规范。

如果这些文档不清楚，Agent 就会自行发挥：

- 自行新增页面
- 自行设计跳转
- 自行发明字段
- 自行决定用户动作后的结果

## 本节内容

1. 用页面清单控制页面数量和页面职责。
2. 用用户流程控制动作和结果闭环。
3. 用数据规范控制字段和存储方式。
4. 检查核心流程是否闭环。
5. 防止 Agent 自行新增页面、字段和流程。

## 学员产物

完成：

```text
PAGE_LIST.md
UX_FLOW.md
DATA_SPEC.md
```

## 常见错误

1. 只写页面名，不写入口和目标。
2. 用户动作后没有结果承接。
3. 数据字段由 Agent 自行发明。
4. 页面数量失控。
5. 页面和 PRD 功能对不上。

## 课后作业

把你的 PRD 拆成页面、流程和数据规范，并检查：

```text
每个页面是否来自 PRD？
每个动作是否有结果？
每个字段是否有使用位置？
```
