# 第 3 课：把产品拆成页面和用户流程

## 学习目标

学完本课，学员应该能理解本节在 AI 产品工程流程中的位置，并完成对应产物。

## 本节核心观点

AI 写页面前，必须先有页面清单。页面清单不清楚，AI 就会自己发挥。每个核心页面都要考虑 Loading、Empty、Error、Success。

## 演示内容

围绕“钓鱼露营地点收藏小程序”演示：

1. 输入什么文档。
2. 使用什么提示词。
3. AI 输出什么。
4. 学员如何检查。
5. 如何进入下一步。

## 学员产物

完成 `PAGE_LIST.md` 和 `UX_FLOW.md`。

## 常见错误

1. 范围过大。
2. 输出不可验收。
3. 没有写“不做什么”。
4. AI 写完后不运行、不审查。

## 课后作业

把本节产物套用到你自己的产品想法上，并用检查清单自查一遍。
