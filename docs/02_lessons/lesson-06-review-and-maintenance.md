# 第 6 课：用文档映射和用户验收判断实现是否正确

## 学习目标

学完本课，学员应该能判断 Agent 产物是否真正实现了需求，并能把问题转成下一轮任务。

## 本节核心观点

产品最终交付给人，不是交付给 Agent。

Agent 完成任务后，需要三层检查：

- 任务边界检查
- 文档到代码映射检查
- 模拟用户验收

能运行不代表实现正确；代码还必须符合文档和用户流程。

## 本节内容

1. 检查 Agent 是否只做当前任务。
2. 检查代码是否能回溯到文档。
3. 运行验证功能是否能用。
4. 模拟用户完成核心流程。
5. 把发现的问题转成明确任务。
6. 判断是否可以进入下一任务或真实用户试用。

## 学员产物

完成：

```text
REVIEW.md
DOC_TO_CODE_MAPPING.md
HUMAN_ACCEPTANCE_CHECKLIST.md
CHANGELOG.md
```

## 常见错误

1. Agent 说完成就算完成。
2. 只看能不能运行，不看是否符合文档。
3. 发现问题后直接让 Agent 自由优化。
4. 没有把问题转成任务。
5. 需求变更直接改代码，不先改文档。

## 课后作业

对你的当前实现做一次审查，并输出结论：

```text
是否通过：通过 / 有条件通过 / 不通过
原因：
必须修复：
可选优化：
是否可以进入下一任务：是 / 否
```
