# 第 5 课：让 Agent 按文档和任务实现

## 学习目标

学完本课，学员应该能把一个任务交给 Agent，并确认 Agent 在文档边界内执行。

## 本节核心观点

Agent 实现不要按“功能大包”推进，要按“小任务闭环”推进。

一个任务必须包含：

- 任务编号
- 任务目标
- 依赖文档
- 允许修改范围
- 禁止修改范围
- 验收标准

## 本节内容

1. 从任务清单里选择一个任务。
2. 把依赖文档写入 Agent 输入。
3. 写清允许修改范围。
4. 写清禁止修改范围。
5. 让 Agent 完成后说明修改文件、对应文档、验证方式和风险点。
6. 出错时只修当前问题，不重做整个产品。

## 学员产物

完成：

```text
标准 Agent 任务输入
Agent 修改说明
实现验证记录
```

## 常见错误

1. 让 Agent 一次做完整产品。
2. 不提供依赖文档。
3. 不写允许修改范围。
4. 不写禁止修改范围。
5. Agent 出错后让它重构全部。

## 课后作业

从你的任务清单里选择一个任务，写成标准 Agent 输入，并检查：

```text
是否只做一个任务？
是否列出依赖文档？
是否写清允许和禁止修改范围？
是否有可验收标准？
```
