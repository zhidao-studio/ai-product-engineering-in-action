# 第 5 课手把手演示：拆任务并让 AI 编码工具执行

## 本节目标

把第一版样例产品拆成 AI 编码工具可以执行的小任务，并完成一个明确任务。

## 起始输入

已有文档：

```text
00_PROJECT_BRIEF.md
01_PRD.md
02_PAGE_LIST.md
03_UX_FLOW.md
AGENTS.md
```

本节要生成：

```text
05_TASKS.md
```

并执行：

```text
TASK-002：实现首页地点列表
```

## 第一步：让 AI 生成任务清单

使用提示词：

```text
请根据下面的项目文档，把项目拆成 AI 编码工具可以执行的小任务。

要求：
1. 一个任务只做一件事。
2. 每个任务必须能独立验收。
3. 每个任务必须写允许修改范围。
4. 每个任务必须写禁止修改范围。
5. 每个任务必须有验收标准。
6. 任务数量控制在 5-12 个。
7. 如果任务太大，请继续拆小。

项目文档：
【粘贴项目总说明、PRD、页面清单、用户流程、AGENTS.md】
```

## 第二步：AI 应该输出什么

合格任务清单：

| 任务编号 | 任务名称 | 状态 | 说明 |
|---|---|---|---|
| TASK-001 | 初始化小程序基础结构 | TODO | app 配置和基础页面 |
| TASK-002 | 实现首页地点列表 | TODO | 使用 mock 数据 |
| TASK-003 | 实现详情页 | TODO | 展示详情和收藏按钮 |
| TASK-004 | 实现本地收藏服务 | TODO | 使用本地存储 |
| TASK-005 | 实现收藏页 | TODO | 展示已收藏地点 |
| TASK-006 | 实现设置页和关于页 | TODO | 基础说明页面 |
| TASK-007 | 代码审查和模拟用户验收 | TODO | 检查交付质量 |

## 第三步：选择一个任务执行

选择：

```text
TASK-002：实现首页地点列表
```

给 AI 编码工具的任务输入：

```text
请根据 AGENTS.md 和案例 docs，只完成当前任务。

当前任务：
TASK-002：实现首页地点列表。

目标：
在首页展示 mock 地点数据。

允许修改：
- examples/01_wechat_mini_program_favorites/miniapp/pages/home/
- examples/01_wechat_mini_program_favorites/miniapp/services/placeService.js
- examples/01_wechat_mini_program_favorites/miniapp/mock/places.js

禁止修改：
- app.json
- 其他页面
- 第三方依赖
- 登录、后端、地图、支付相关能力

验收标准：
1. 首页能显示地点列表。
2. 每个地点展示名称、地区、标签、摘要。
3. 点击地点能进入详情页。
4. 空数据时有空状态。
5. 完成后输出修改文件、验证方式和风险点。
```

## 第四步：检查 AI 输出是否合格

重点检查：

- 是否只修改允许目录？
- 是否新增了无关依赖？
- 是否改了 app.json？
- 是否把详情页、收藏页一起大改？
- 是否输出了修改文件列表？
- 是否给出验证方式？
- 是否说明风险点？

## 第五步：不合格时怎么追问

如果 AI 改了无关文件：

```text
你修改了无关文件。请回退无关修改，只保留 TASK-002 需要的变更。
允许修改范围只有：
- pages/home/
- services/placeService.js
- mock/places.js
```

如果运行报错：

```text
请只修复下面这个报错，不要做无关重构。
请说明根因、修改文件和验证方式。

报错：
【粘贴报错】
```

如果连续 3 次修不好：

```text
停止继续修复。
请重新分析 TASK-002 是否拆得太大，给我一个更小的修复任务。
```

## 第六步：合格产物

最终应该得到：

```text
首页地点列表可显示
地点 mock 数据可读取
点击地点可以进入详情页
AI 输出修改文件和验证方式
```

## 本节完成标准

- `05_TASKS.md` 已生成。
- 当前任务边界清楚。
- AI 编码工具只完成一个任务。
- 首页能运行。
- 可以进入代码审查。
