# 截图补充指南

## 1. 为什么需要截图

图文课程的核心不是堆文字，而是让学员看到：

- 当前在哪一步
- 应该打开哪个文件
- 应该复制哪段提示词
- AI 输出应该长什么样
- 微信开发者工具里应该看到什么结果

## 2. 截图命名建议

建议后续建立目录：

```text
docs/02_lessons/visual/assets/
├── lesson-01/
├── lesson-02/
├── lesson-03/
├── lesson-04/
├── lesson-05/
└── lesson-06/
```

截图命名：

```text
lesson-02-step-01-input-idea.png
lesson-02-step-02-ai-output-project-brief.png
lesson-02-step-03-check-prd.png
```

## 3. 每节课至少需要哪些截图

### 第 1 课

- 仓库 README 首页
- AI 工具分工表
- 学习路径图

### 第 2 课

- 输入产品想法
- AI 输出项目总说明
- AI 输出 PRD

### 第 3 课

- 页面清单输出
- 用户流程图
- 小程序 5 页面结构

### 第 4 课

- AGENTS.md 文件
- AI 执行规则重点标注
- 禁止事项标注

### 第 5 课

- TASKS.md 任务清单
- AI 编码工具任务输入
- 微信开发者工具首页效果

### 第 6 课

- DeepSeek 审查输入
- 代码审查报告
- 模拟用户验收报告
- 问题转任务示例

## 4. 截图标注建议

每张图尽量标注：

- 第几步
- 看哪里
- 点哪里
- 复制哪里
- 判断什么
- 完成后进入哪一步

## 5. 不建议截图的内容

不要公开：

- 个人 API Key
- GitHub Token
- 真实账号密码
- 私有 AppID
- 未公开的商业信息
