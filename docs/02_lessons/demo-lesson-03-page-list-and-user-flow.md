# 第 3 课手把手演示：从 PRD 到页面清单和用户流程

## 本节目标

把《产品需求文档》拆成：

1. 页面清单
2. 页面状态
3. 页面跳转关系
4. 用户主流程

## 起始输入

使用第 2 课得到的产品需求：

```text
钓鱼露营地点收藏小程序

第一版功能：
1. 首页地点列表
2. 地点详情
3. 收藏功能
4. 收藏页
5. 设置页
6. 关于页

第一版不做：
登录、后端、地图、发布、评论、私信、支付、云同步。
```

## 第一步：让 AI 生成页面清单

使用提示词：

```text
请根据下面的 PRD，生成《页面清单》和《用户流程》。

要求：
1. 第一版页面不超过 5 个。
2. 每个页面必须有明确入口。
3. 每个页面必须有用户动作。
4. 每个核心页面必须考虑 Loading / Empty / Error / Success。
5. 必须给出页面跳转关系。
6. 如果页面过多，请主动建议合并或砍掉。

PRD：
【粘贴产品需求文档】
```

## 第二步：AI 应该输出什么

合格页面清单：

| 页面编号 | 页面名称 | 页面入口 | 页面目标 | 第一版是否做 |
|---|---|---|---|---|
| P001 | 首页 | 小程序启动页 / Tab | 展示地点列表 | 是 |
| P002 | 详情页 | 首页地点卡片 | 展示地点详情和收藏按钮 | 是 |
| P003 | 收藏页 | Tab | 展示已收藏地点 | 是 |
| P004 | 设置页 | Tab | 展示基础设置 | 是 |
| P005 | 关于页 | 设置页 | 展示课程样例说明 | 是 |

合格用户流程：

```text
用户打开小程序
↓
首页浏览地点列表
↓
点击地点卡片
↓
进入详情页
↓
点击收藏
↓
进入收藏页
↓
查看已收藏地点
```

## 第三步：检查 AI 输出是否合格

重点检查：

- 页面是否超过 5 个？
- 每个页面是否有入口？
- 是否有核心流程？
- 首页、详情页、收藏页是否有页面状态？
- 是否偷偷加回登录、地图、后端、支付？
- 收藏后是否有结果页面承接？

## 第四步：不合格时怎么追问

如果 AI 输出了太多页面：

```text
页面太多了，请压缩到 5 个以内。

要求：
1. 保留首页、详情页、收藏页、设置页、关于页。
2. 不新增登录页。
3. 不新增发布页。
4. 不新增地图页。
5. 重新输出页面清单和用户流程。
```

如果 AI 没有写页面状态：

```text
请为首页、详情页、收藏页补充 Loading / Empty / Error / Success 四种状态。
每种状态用一句话说明用户看到什么。
```

## 第五步：合格产物

最终应该得到：

```text
02_PAGE_LIST.md
03_UX_FLOW.md
```

其中页面数量控制在：

```text
5 个以内
```

核心流程必须闭环：

```text
首页 → 详情页 → 收藏 → 收藏页查看
```

## 本节完成标准

- 页面清单完整。
- 页面状态完整。
- 用户流程闭环。
- 第一版复杂度没有扩大。
- 可以进入第 4 课写 AI 执行规则。
