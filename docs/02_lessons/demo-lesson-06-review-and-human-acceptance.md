# 第 6 课手把手演示：代码审查、运行验证和模拟用户验收

## 本节目标

判断 AI 编码工具交付的内容是否真的可以给人使用。

本节完成后，你会得到：

1. 代码审查报告
2. 运行验证结果
3. 模拟用户验收报告
4. 下一轮修改任务

## 起始输入

你已经完成一个任务，例如：

```text
TASK-002：实现首页地点列表
```

AI 编码工具已经输出：

```text
修改文件列表
实现说明
验证方式
风险点
```

## 第一步：做代码审查

使用提示词：

```text
请作为代码审查员，审查这次 AI 代码变更。

任务说明：
【粘贴 TASK-002】

代码变更说明或 diff：
【粘贴 AI 编码工具输出】

项目规则：
【粘贴 AGENTS.md】

请重点检查：
1. 是否符合当前任务。
2. 是否修改了无关文件。
3. 是否引入了不必要依赖。
4. 是否有硬编码敏感信息。
5. 是否缺少 Loading / Empty / Error / Success 状态。
6. 是否存在明显 bug。
7. 是否适合非技术学员继续维护。
8. 是否可以进入模拟用户验收。
```

## 第二步：AI 应该输出什么

合格代码审查报告：

```markdown
# 代码审查报告

## 1. 总体结论

有条件通过。

## 2. 必须修改项

暂无。

## 3. 建议优化项

首页顶部可以补充样例数据说明。

## 4. 风险点

当前使用 mock 数据，不是真实地点库。

## 5. 测试建议

检查首页列表、空数据状态、点击跳转详情页。

## 6. 是否可以进入模拟用户验收

可以。
```

## 第三步：运行验证

自己打开微信开发者工具，检查：

| 检查项 | 合格标准 |
|---|---|
| 小程序能打开 | 无白屏 |
| 首页能显示地点 | 至少显示 1 条地点 |
| 点击能跳转 | 能进入详情页 |
| 收藏能操作 | 按钮有反馈 |
| 收藏页能显示 | 收藏后能看到数据 |

## 第四步：模拟用户验收

使用提示词：

```text
请你模拟一个真实目标用户，使用这个产品，并从用户视角检查交付内容是否准确。

产品说明：
【粘贴项目总说明】

页面清单：
【粘贴页面清单】

核心流程：
【粘贴用户流程】

当前交付说明：
【粘贴当前已完成的功能说明】

请检查：
1. 第一次打开是否看得懂？
2. 用户是否知道下一步点哪里？
3. 页面文案是否清楚？
4. 核心流程是否闭环？
5. 是否有逻辑断点？
6. 用户是否会中途放弃？
7. 是否适合交给真实用户试用？
```

## 第五步：AI 应该输出什么

合格模拟用户验收报告：

```markdown
# 模拟用户验收报告

## 1. 总体结论

有条件通过。

## 2. 用户视角体验

用户可以从首页看到地点列表，也能进入详情页并收藏。

## 3. 主要卡点

首页没有说明“这些地点是样例数据”，用户可能误以为是真实地点库。

## 4. 必须修改项

在首页顶部增加说明：当前为课程实战样例数据，后续可接入真实地点。

## 5. 可选优化项

地点卡片可以增加“适合露营 / 适合钓鱼”标签。

## 6. 是否适合真实用户试用

可以小范围试用，但需要说明当前是样例数据。
```

## 第六步：把问题转成任务

不要直接说：

```text
帮我优化一下。
```

要转成任务：

```text
TASK-007：首页增加样例数据说明

目标：
在首页顶部增加说明文案，告诉用户当前为课程实战样例数据。

允许修改：
- pages/home/

禁止修改：
- 其他页面
- 数据服务
- app.json

验收标准：
首页顶部能看到说明文案，且不影响地点列表展示。
```

## 第七步：检查是否可以交给真实用户

通过标准：

- 用户能看懂这是做什么的。
- 用户能完成首页 → 详情页 → 收藏 → 收藏页。
- 没有明显死路。
- 样例数据已明确说明。
- 代码审查没有阻塞问题。
- 模拟用户验收没有严重逻辑断点。

## 本节完成标准

- 代码审查已完成。
- 运行验证已完成。
- 模拟用户验收已完成。
- 问题已转成下一轮任务。
- 可以判断是否进入真实用户试用。
