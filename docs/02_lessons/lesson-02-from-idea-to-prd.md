# 第 2 课：把产品想法放进文档控制体系

## 学习目标

学完本课，学员应该能把一句模糊想法整理成两份控制文档：

- 项目总说明
- 产品需求文档

## 本节核心观点

非技术用户最容易失败的原因不是不会写代码，而是需求太大、范围太乱。

第一版必须先回答：

- 给谁用
- 解决什么问题
- 第一版只做什么
- 第一版明确不做什么
- 做到什么算完成

这些问题没有进入文档，就不能进入实现。

## 本节内容

1. 把产品想法写成项目总说明。
2. 用项目总说明控制产品边界。
3. 把项目总说明继续写成 PRD。
4. 用 PRD 控制功能范围和验收标准。
5. 明确 Agent 不得自行扩展的内容。

## 学员产物

完成：

```text
PROJECT_BRIEF.md
PRD.md
```

## 常见错误

1. 目标用户太泛。
2. 第一版功能太多。
3. 没有不做清单。
4. 验收标准不可检查。
5. 让 Agent 自行补功能。

## 课后作业

把你的产品想法写成项目总说明和 PRD，并检查：

```text
是否写清第一版边界？
是否写清不做什么？
是否写清验收标准？
```
