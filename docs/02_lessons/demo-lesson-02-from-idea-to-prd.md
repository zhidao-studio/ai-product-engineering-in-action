# 第 2 课手把手样例：从想法到产品需求

## 本节目标

用“钓鱼露营地点收藏小程序”这个实战样例，演示如何从一个模糊想法整理出：

1. 项目总说明
2. 产品需求文档
3. 第一版做什么
4. 第一版不做什么
5. 验收标准

## 起始想法

```text
我想做一个小程序，帮大家收藏适合钓鱼和露营的地点。
```

这个想法还太粗，需要进一步整理。

## 第一步：把想法交给 AI

使用提示词：

```text
请根据我的产品想法，帮我整理成《项目总说明》。

要求：
1. 明确第一版做什么、不做什么，遵守 MVP 原则。
2. 目标用户要具体到一类人，不要写“所有人”。
3. 核心场景要描述用户在什么情况下、为了解决什么问题来用。
4. 成功标准要可验收。
5. 第一版页面数量建议不超过 5 个。
6. 输出 Markdown 格式。
7. 最后给出 3 个最大风险提醒。

我的想法是：
我想做一个小程序，帮大家收藏适合钓鱼和露营的地点。
```

## 第二步：检查 AI 输出是否合格

重点检查：

- 目标用户是否具体？
- 第一版是否控制在 5 个页面以内？
- 是否明确不做登录、支付、地图、后端？
- 成功标准是否可验收？
- 是否能进入页面清单阶段？

## 第三步：不合格时如何追问

如果 AI 输出太大，可以继续追问：

```text
这个范围还是太大，请帮我压缩成第一版可运行样例。
要求：
1. 页面不超过 5 个。
2. 不做登录。
3. 不做后端。
4. 不做地图。
5. 只用模拟数据和本地存储。
6. 输出新的第一版范围。
```

## 第四步：合格产物

最终应该得到：

```text
项目名称：
钓鱼露营地点收藏小程序

目标用户：
喜欢周末去上海周边钓鱼、露营、短途户外的人。

第一版做：
首页地点列表、详情页、本地收藏、收藏页、设置页、关于页。

第一版不做：
登录、后端、地图、发布、评论、私信、支付、云同步。

成功标准：
用户可以打开小程序，浏览地点，进入详情，收藏地点，并在收藏页查看。
```

## 本节完成标准

完成后，你应该有：

- `PROJECT_BRIEF.md`
- `PRD.md`
- 第一版做什么
- 第一版不做什么
- 可以进入第 3 课页面清单阶段
