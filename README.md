# 阶段受控的 AI 产品交付协议

Document-Driven AI Product Delivery Harness

## 1. 仓库定位

本仓库用于沉淀一套约束 AI 作为生产力的产品交付系统。

它的核心目标是：

```text
让人提出产品意图，
让系统把意图变成结构化契约，
让 AI 在工程模板和任务包内执行，
让门禁检查 AI 是否越界，
让结果以人能决策的方式反馈回来。
```

```text

2. 把某个软约束升级成结构化契约或可执行门禁。
```

## 3. 新任务默认开口

新用户不要直接让 AI 写代码、改文件或执行命令。

每次开始新任务，先使用：

```text
指令/通用大模型/00_默认启动口令.md
```

它会强制 AI 先判断：

```text
当前模式：使用者模式 / 开发者模式
当前阶段
当前 AI 角色
允许做什么
禁止做什么
需要人确认什么
准备修改哪些文件
准备执行哪些验证
```

入口原则：

```text
给使用者：简单、清晰、高体验、少选择、少技术细节。
给开发者：稳定、可维护、可验证、可追踪、可回滚。
```

## 5. 总路径

```text
人说出产品想法
↓
系统判断当前阶段
↓
系统把想法转成结构化问题、选项和文档
↓
人确认产品方向和界面预期
↓
系统冻结阶段性契约
↓
AI 在工程模板和任务包内执行
↓
机器门禁检查是否越界
↓
失败时翻译成人话选项
↓
人做产品决策
↓
系统进入下一轮交付
```

## 6. 四条主线

```text
使用者主线：想法 → 选择 → 原型确认 → 验收
AI 主线：读上下文 → 判阶段 → 判边界 → 执行任务
工程主线：文档 → 原型 → 契约 → 模板 → 编码 → 验证
治理主线：文档约束 → 结构化契约 → 机器门禁 → 人话反馈
```

## 9. 当前工程基线

```text
Web 后台：Vue 3 + Vite + TypeScript + Vue Router + Pinia + Element Plus
H5：Vue 3 + Vite + TypeScript + Vant
小程序：原生微信小程序 + TypeScript + Vant Weapp
iOS：SwiftUI + MVVM + CocoaPods
后端：Java + Spring Boot + MyBatis-Plus + Flyway 单体应用，不使用 Spring Cloud
数据库：MySQL 优先；主键 32 位字符串；每表 7 要素；时间 trigger 维护
码值：数据库码值是源头，后端常量一致，接口契约记录，前端只翻译不改码
部署：本地开发优先，微信云托管按需适配
多端原则：多个工程，独立建设；接口一致，代码不共享
```
