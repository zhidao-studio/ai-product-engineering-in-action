# AI 产品工程实战

AI Product Engineering in Action

这是一套面向非开发者的 AI 产品工程实战教程。

目标不是教你“AI 一键生成产品”，而是教你如何用 AI 按工程流程做出可维护的产品第一版。

## 第一版实战样例

H5 页面 / 轻量 Web App：钓鱼露营地点收藏

页面：

- 首页
- 详情页
- 收藏页
- 设置页
- 关于页

第一版不做：

- 登录
- 后端
- 地图
- 发布
- 评论
- 私信
- 支付
- 云同步
- 小程序平台能力

为什么先做 H5：

- 不需要微信小程序 AppID。
- 不需要微信开发者工具。
- 浏览器即可运行和预览。
- 非技术学员更容易理解页面、路由、数据和交互。
- 后续可以迁移到微信小程序、iOS App 或 Web App。

## 学习路径

1. 进入 `docs/02_lessons/visual/` 阅读图文课程。
2. 完成《产品想法验证》。
3. 写《项目总说明》。
4. 写《产品需求文档》。
5. 拆《页面清单》。
6. 写《用户流程》。
7. 写《AI 执行规则》。
8. 拆《任务清单》。
9. 让 AI 编码工具按任务写 H5 代码。
10. 用 AI 做代码审查。
11. 用“模拟用户验收”检查交付体验。
12. 运行、修复、真实用户试用和迭代。

## AI 工具说明

本课程不绑定某一个 AI 编码工具。

你可以使用：

- Codex
- Cursor
- Windsurf
- Kimi Coding
- Trae / MarsCode
- GitHub Copilot
- 其他可以读取工程、修改文件、执行任务的 AI 编码工具

核心不是工具名称，而是流程：

> 需求清楚、任务清楚、规则清楚、验收清楚。

## 仓库目录

```text
docs/          课程文档和方法论
templates/     可复制填写的文档模板
prompts/       给 ChatGPT / AI 编码工具 / DeepSeek / Kimi 的提示词
examples/      实战样例，包含文档和代码
tools/         工具脚本和配置
archive/       草稿、历史版本和归档内容
```

## 核心分工

| 工具 | 作用 |
|---|---|
| ChatGPT / 通用大模型 | 产品设计、需求拆解、架构设计、任务拆分、最终验收 |
| AI 编码工具 | 进入工程写代码、改文件、修 bug、执行任务 |
| DeepSeek / 审查模型 | 代码审查、风险检查、逻辑检查 |
| Kimi / 长上下文模型 | 长文档整理、中文说明、竞品分析、文案辅助 |
| 学员 | 做决定、控范围、运行验证、确认是否完成 |

一句话：

> 大模型想清楚，编码工具写出来，审查模型查问题，学员做决定。

## 课程原则

- 不神化 AI
- 不承诺一键生成商业产品
- 每一步都有文档
- 每一个任务都要可验收
- AI 写完不等于完成，必须运行和审查
- 最终交付给人，所以必须模拟真实用户体验
- 第一版只做最小闭环，不做大而全
