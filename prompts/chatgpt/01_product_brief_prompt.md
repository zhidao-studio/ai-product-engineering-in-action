# 项目总说明提示词

请根据我的产品想法，帮我整理成《项目总说明》。

这份文档的作用不是做介绍，而是控制后续需求实现边界。

## 要求

1. 明确第一版做什么、不做什么，遵守最小闭环原则。
2. 目标用户要具体到一类人，不要写“所有人”。
3. 核心场景要描述用户在什么情况下、为了解决什么问题来用。
4. 成功标准要可验收，不要写“体验好”“功能完善”这种虚词。
5. 第一版页面数量建议不超过 5 个。
6. 明确哪些内容禁止 Agent 在实现时自行扩展。
7. 输出 Markdown 格式。
8. 最后给出 3 个最大风险提醒。

## 输出格式

```markdown
# 项目总说明

## 1. 项目名称
## 2. 一句话说明
## 3. 目标用户
## 4. 用户痛点
## 5. 核心场景
## 6. 第一版做什么
## 7. 第一版不做什么
## 8. 成功标准
## 9. Agent 不得自行扩展的内容
## 10. 最大风险
```
