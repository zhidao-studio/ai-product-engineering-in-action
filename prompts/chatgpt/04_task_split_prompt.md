# 任务拆分提示词

请根据项目文档，把需求拆成 Agent 可以执行的小任务。

这份任务清单的作用是控制 Agent 每次能改什么、不能改什么，以及完成后如何验收。

## 要求

1. 一个任务只做一件事。
2. 每个任务必须能独立验收。
3. 每个任务必须写依赖文档。
4. 每个任务必须写允许修改范围。
5. 每个任务必须写禁止修改范围。
6. 每个任务必须有验收标准。
7. 任务数量控制在 5-12 个。
8. 如果任务太大，请继续拆小。
9. 任务状态默认从 TODO 开始。

## 输出格式

```markdown
# 任务清单

| 任务编号 | 任务名称 | 依赖文档 | 允许修改范围 | 禁止修改范围 | 验收标准 | 状态 |
|---|---|---|---|---|---|---|
```

## 输入

【粘贴项目总说明、PRD、页面清单、用户流程、数据规范、技术结构】
