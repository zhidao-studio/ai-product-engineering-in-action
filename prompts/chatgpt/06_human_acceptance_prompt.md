# 模拟用户验收提示词

请你模拟一个真实目标用户，使用这个产品，并检查交付内容是否准确。

验收不仅要看用户体验，还要看当前实现是否符合项目文档。

## 输入

项目总说明：
【粘贴项目总说明】

PRD：
【粘贴产品需求文档】

页面清单：
【粘贴页面清单】

用户流程：
【粘贴用户流程】

数据规范：
【粘贴数据规范】

文档到代码映射：
【粘贴文档到代码映射】

当前交付说明：
【粘贴 Agent 输出、代码变更或功能说明】

## 检查维度

1. 第一次打开是否看得懂？
2. 用户是否知道下一步点哪里？
3. 页面文案是否清楚？
4. 核心流程是否闭环？
5. 用户动作后是否有结果？
6. 当前实现是否符合页面清单？
7. 当前实现是否符合用户流程？
8. 当前实现是否使用了数据规范中的字段？
9. 当前实现是否满足 PRD 验收标准？
10. 是否有逻辑断点？
11. 用户是否会中途放弃？
12. 是否适合交给真实用户试用？

## 输出格式

```markdown
# 模拟用户验收报告

## 1. 总体结论
通过 / 有条件通过 / 不通过

## 2. 用户视角体验

## 3. 文档一致性结论

## 4. 主要卡点

## 5. 必须修改项

## 6. 可选优化项

## 7. 是否适合真实用户试用

## 8. 需要转成的新任务
```
