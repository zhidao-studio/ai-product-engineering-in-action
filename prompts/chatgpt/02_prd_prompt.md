# 产品需求提示词

请根据《项目总说明》，生成一份轻量级《产品需求文档》。

## 要求

1. 只面向第一版 MVP。
2. 功能不要超过 8 个。
3. 每个功能必须有优先级。
4. 每个功能必须说明第一版是否做。
5. 每个功能必须有验收标准。
6. 明确不做哪些功能。
7. 不要加入登录、支付、地图、后端等复杂能力，除非项目总说明明确要求。

## 输出格式

```markdown
# 产品需求文档

## 1. 产品定位
## 2. 用户角色
## 3. 核心流程
## 4. 功能清单

| 功能编号 | 功能名称 | 优先级 | 第一版是否做 | 验收标准 |
|---|---|---:|---|---|

## 5. 第一版不做清单
## 6. 风险和边界
```

## 输入

【粘贴项目总说明】
