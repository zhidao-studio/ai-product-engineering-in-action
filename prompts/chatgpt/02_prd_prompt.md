# 产品需求文档提示词

请根据《项目总说明》，生成一份轻量级《产品需求文档》。

这份 PRD 的作用是控制第一版功能范围，防止后续 Agent 自行扩展需求。

## 要求

1. 只面向第一版最小闭环。
2. 功能不要超过 8 个。
3. 每个功能必须有功能编号。
4. 每个功能必须有优先级。
5. 每个功能必须说明第一版是否做。
6. 每个功能必须有可人工检查的验收标准。
7. 明确第一版不做哪些功能。
8. 不要加入项目总说明之外的能力。
9. 标出哪些功能后续会进入页面清单、用户流程和数据规范。

## 输出格式

```markdown
# 产品需求文档

## 1. 产品定位
## 2. 用户角色
## 3. 核心流程
## 4. 功能清单

| 功能编号 | 功能名称 | 优先级 | 第一版是否做 | 验收标准 | 后续控制文档 |
|---|---|---:|---|---|---|

## 5. 第一版不做清单
## 6. Agent 不得自行扩展的功能
## 7. 风险和边界
```

## 输入

【粘贴项目总说明】
