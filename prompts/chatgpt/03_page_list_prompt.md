# 页面清单 / 用户流程 / 数据规范 / 设计原型提示词

请根据 PRD，生成《页面清单》《用户流程》《数据规范》和《设计原型说明》。

这四份文档的作用是继续控制需求实现：页面清单控制页面范围，用户流程控制动作闭环，数据规范控制字段和存储方式，设计原型控制布局、状态和高保真方向。

## 要求

1. 第一版页面不超过 5 个。
2. 每个页面必须有明确入口。
3. 每个页面必须有页面目标和用户动作。
4. 每个核心页面必须考虑 Empty / Error / Success 状态。
5. 必须给出页面跳转关系。
6. 用户每个动作后必须有结果承接。
7. 数据字段必须来自页面和流程，不允许自行发明无用字段。
8. 设计原型必须说明页面布局、模块顺序、关键按钮、状态展示和高保真风格。
9. 如果页面过多，请主动建议砍掉。
