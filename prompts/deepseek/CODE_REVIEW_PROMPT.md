# 代码审查提示词

请作为审查 Agent，审查这次代码变更。

审查目标不是只看代码能不能跑，而是判断：

```text
代码是否符合文档、任务边界和验收标准。
```

## 输入

当前任务：
【粘贴 TASK】

相关文档：
【粘贴项目总说明 / PRD / 页面清单 / 用户流程 / 数据规范 / 技术结构 / 文档到代码映射】

Agent 输出：
【粘贴修改说明】

代码变更：
【粘贴 diff 或修改文件内容】

## 审查重点

1. 是否符合当前任务。
2. 是否修改了无关文件。
3. 是否引入了不必要依赖。
4. 是否有硬编码敏感信息。
5. 是否新增了文档没有定义的功能、页面或字段。
6. 是否符合 PRD。
7. 是否符合页面清单。
8. 是否符合用户流程。
9. 是否符合数据规范。
10. 是否符合文档到代码映射。
11. 是否存在明显 bug。
12. 是否存在难以维护的写法。
13. 是否适合非技术学员继续跟做。
14. 是否可以进入模拟用户验收。

## 输出格式

```markdown
# 代码审查报告

## 1. 总体结论
通过 / 有条件通过 / 不建议通过

## 2. 文档一致性结论

## 3. 任务边界结论

## 4. 必须修改项

## 5. 建议优化项

## 6. 风险点

## 7. 测试建议

## 8. 是否可以进入模拟用户验收
```
