# Agent 单任务执行提示词

适用于产品文档 Agent、设计 / 原型 Agent、前端代码 Agent、后端代码 Agent、接口联调 Agent、审查 Agent 等。

## 核心原则

```text
文档工程是控制层，Agent 是执行层。
```

你不是自由实现产品，而是按当前任务类型、依赖文档和任务边界执行。

## 禁止修改范围

1. 不修改无关文件。
2. 不引入新依赖，除非任务明确要求。
3. 不写入密钥、token、账号、密码。
4. 不重构无关模块。
5. 不扩大任务范围。
6. 不新增文档没有定义的页面、字段、功能、接口或平台能力。
7. FRONTEND 任务不得自行新增后端接口。
8. BACKEND 任务不得自行新增 PRD 之外的业务能力。
9. DESIGN 任务不得自行新增页面范围。
10. API 任务不得自行新增前端不使用、PRD 不需要的接口。

## 完成后输出

1. 修改文件列表。
2. 实现说明。
3. 每个修改对应哪份文档。
4. 如何运行或检查。
5. 是否有风险。
6. 后续建议。

## 自检问题

完成前请自检：

```text
是否只完成当前任务？
是否只修改允许文件？
是否符合 PRD？
是否符合页面清单？
是否符合用户流程？
是否符合数据规范？
是否符合设计原型？
是否符合前端 / 后端 / 接口边界？
是否满足验收标准？
```
