# 审查 Harness 提示词

## 使用场景

当你已经写完一套 harness，需要 AI 帮你检查是否完整时，使用这个提示词。

## 提示词

```text
请审查下面这套文档驱动的 AI 产品交付 harness 是否完整。

Harness 内容：
【粘贴 harness】

请按以下维度检查：

1. 产品定义是否清楚
2. 设计规格是否足够约束视觉和交互
3. 工程规格是否覆盖数据、前端、后端和接口
4. 任务执行是否有明确允许范围和限制范围
5. 验证标准是否可判断
6. 体验验收是否能让真实用户完成核心流程
7. 交付迭代是否有变更协议和反馈分类
8. 非技术人是否能根据这套 harness 做判断

请输出：
- 总体结论：通过 / 有条件通过 / 不通过
- 缺失部分
- 风险点
- 需要补充的模板
- 需要补充的 checklist
- 是否可以交给 AI / Agent 执行
```
