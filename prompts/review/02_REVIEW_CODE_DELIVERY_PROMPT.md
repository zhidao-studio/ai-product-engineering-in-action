# 审查代码交付提示词

## 1. 使用场景

当 Agent 完成代码交付任务后，用本提示词检查产物是否符合 harness、工程基线和代码任务包。

## 2. 提示词模板

```text
请审查下面这次代码交付是否符合约束。

输入材料：
1. 产品 / 设计 / 工程 Harness
2. 工程基线
3. 代码任务包
4. API 契约
5. 数据库规范
6. 基础设施规范
7. Agent 输出
8. 代码变更说明

请按以下维度检查：

1. 是否只完成当前任务
2. 是否只修改允许范围内的文件
3. 是否符合 Vue 前端工程规范
4. 是否符合 Spring Boot 后端工程规范
5. 是否没有使用 Spring Cloud
6. 是否符合 API 契约
7. 是否符合数据库规范
8. 是否符合基础设施规范
9. 是否能启动和验证
10. 是否需要补充 harness 或工程规范

请输出：
- 总体结论：通过 / 有条件通过 / 不通过
- 范围问题
- 工程规范问题
- API / 数据问题
- 基础设施问题
- 必须修复项
- 可选优化项
- 是否可以进入下一任务
```
