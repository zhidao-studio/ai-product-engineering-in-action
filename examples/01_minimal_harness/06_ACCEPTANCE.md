# 示例：体验验收 Harness

## 1. 用户角色

用户：想收藏周末出行地点的人。

## 2. 用户任务

```text
打开页面
↓
找到一个地点
↓
查看详情
↓
收藏地点
↓
查看收藏结果
```

## 3. 体验检查

- [ ] 第一次打开能看懂这是地点收藏工具
- [ ] 用户知道可以点击地点
- [ ] 用户知道如何收藏
- [ ] 收藏后能看到结果
- [ ] 收藏为空时有提示

## 4. 多设备检查

- [ ] 手机屏幕可用
- [ ] 桌面浏览器可用
- [ ] 主要按钮不被遮挡
- [ ] 文字可读

## 5. 结论

是否通过：通过 / 有条件通过 / 不通过

主要卡点：

是否可以进入交付：是 / 否
