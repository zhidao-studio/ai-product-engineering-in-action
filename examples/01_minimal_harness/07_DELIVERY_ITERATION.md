# 示例：交付迭代 Harness

## 1. 交付条件

- 产品定义已确认
- 设计规格已确认
- 工程规格已确认
- 验证 Harness 已通过
- 体验验收已通过

## 2. 部署检查

- [ ] 页面可打开
- [ ] 核心流程可完成
- [ ] 空状态可显示
- [ ] 没有引入不做清单里的能力

## 3. 反馈分类

| 类型 | 处理方式 |
|---|---|
| bug | 进入修正任务 |
| 体验问题 | 更新体验验收或设计规格 |
| 范围变更 | 先更新产品定义或 PRD |
| 新需求 | 放入下一版本评估 |

## 4. 变更协议

任何变更必须先判断：

```text
是否影响产品定义？
是否影响设计规格？
是否影响工程规格？
是否需要新增任务？
是否需要重新验证？
```
