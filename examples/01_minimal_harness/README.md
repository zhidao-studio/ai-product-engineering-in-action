# 最小 Harness 示例

## 1. 示例目标

这个示例不开发完整产品，只展示如何从一个产品想法构造最小可用 harness。

示例产品想法：

```text
做一个页面，帮助用户收藏自己喜欢的地点。
```

## 2. 为什么是最小示例

本示例只演示：

```text
产品定义
设计规格
工程规格
任务执行
验证
体验验收
交付迭代
```

它不直接写代码。

它的目的不是展示产品功能，而是展示 harness 如何约束 AI 产品交付。

## 3. 示例文件

| 文件 | 作用 |
|---|---|
| `01_PRODUCT_DEFINITION.md` | 示例产品定义 |
| `02_DESIGN_SPEC.md` | 示例设计规格 |
| `03_ENGINEERING_SPEC.md` | 示例工程规格 |
| `04_TASK_EXECUTION.md` | 示例任务执行 harness |
| `05_VALIDATION.md` | 示例验证 harness |
| `06_ACCEPTANCE.md` | 示例体验验收 harness |
| `07_DELIVERY_ITERATION.md` | 示例交付迭代 harness |

## 4. 使用方式

1. 先阅读本示例。
2. 对照 `templates/harness/` 看模板如何被填写。
3. 用 `checklists/harness/` 检查是否完整。
4. 再尝试替换成自己的产品想法。

## 5. 最重要的一句话

```text
示例不是为了证明 AI 能写代码，
而是为了证明 harness 能约束 AI 交付。
```
