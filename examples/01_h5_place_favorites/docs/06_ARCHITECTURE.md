# 技术结构说明

## 1. 第一版技术原则

第一版 H5 样例只使用原生能力：

- HTML
- CSS
- JavaScript
- Mock 数据
- localStorage

不使用：

- Vue
- React
- Next.js
- 构建工具
- 后端服务
- 登录体系
- 地图 SDK

原因：

```text
第一版要讲清楚“文档如何落到代码”，而不是让学员学习前端框架。
```

## 2. 文件职责

| 文件 | 职责 | 来源文档 |
|---|---|---|
| `README.md` | 项目说明、运行方式、验收方式 | 项目总说明 |
| `index.html` | 页面结构和容器 | 页面清单 |
| `styles.css` | 视觉样式和响应式布局 | 页面清单、界面要求 |
| `app.js` | 页面渲染、导航、收藏交互 | 用户流程、任务清单 |
| `mock/places.js` | 地点 mock 数据 | 数据规范 |

## 3. 页面实现方式

第一版可以采用单 HTML 页面实现多个“页面状态”：

```text
首页
详情页
收藏页
设置页
关于页
```

页面切换可以通过 JavaScript 控制显示内容。

这样做的原因：

- 文件少，学员容易理解。
- 不需要学习路由框架。
- 便于演示用户流程。
- 便于后续迁移到小程序或 Web App。

## 4. 数据流

```text
mock/places.js
↓
app.js 读取地点数据
↓
index.html 容器展示
↓
用户点击收藏
↓
app.js 写入 localStorage
↓
收藏页读取 localStorage 并展示结果
```

## 5. 禁止事项

AI 编码工具第一版不得：

- 新增框架。
- 新增构建工具。
- 接入后端。
- 接入地图。
- 新增登录。
- 新增支付。
- 改变第一版页面数量。

## 6. 代码完成标准

- 浏览器可以打开页面。
- 首页能展示 mock 地点。
- 详情页能显示对应地点。
- 收藏功能能写入 localStorage。
- 收藏页能显示已收藏地点。
- 清空收藏能生效。
- 页面文案说明当前是课程样例数据。
