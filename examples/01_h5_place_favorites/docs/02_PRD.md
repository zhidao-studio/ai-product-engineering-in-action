# 产品需求文档

## 1. 产品定位

这是一个用于教学的 H5 地点收藏样例，目标是让学员理解如何从文档工程落到最小代码结构。

## 2. 用户角色

| 用户 | 说明 | 目标 |
|---|---|---|
| 新用户 | 第一次打开 H5 页面 | 浏览地点并理解页面用途 |
| 收藏用户 | 找到感兴趣地点 | 收藏地点并能回看 |
| 回访用户 | 已经收藏过地点 | 快速查看收藏列表 |

## 3. 核心流程

```text
首页浏览地点
↓
点击地点卡片
↓
进入详情页
↓
点击收藏
↓
进入收藏页查看结果
```

## 4. 功能清单

| 功能编号 | 功能名称 | 优先级 | 第一版是否做 | 验收标准 |
|---|---|---:|---|---|
| F001 | 首页地点列表 | P0 | 是 | 用户能看到地点卡片列表 |
| F002 | 地点详情 | P0 | 是 | 用户能查看地点名称、标签、说明和注意事项 |
| F003 | 收藏地点 | P0 | 是 | 用户能收藏和取消收藏地点 |
| F004 | 收藏页 | P0 | 是 | 用户能看到已收藏地点 |
| F005 | 设置页 | P1 | 是 | 用户能清空收藏并看到基础说明 |
| F006 | 关于页 | P1 | 是 | 用户能看到课程样例说明 |

## 5. 第一版不做清单

- 登录
- 后端接口
- 地图定位
- 用户发布
- 评论
- 私信
- 支付
- 云同步

## 6. 验收方式

每个功能必须能在浏览器中人工验证。

验收时只看：

- 页面是否显示。
- 点击是否有结果。
- 收藏是否保存。
- 刷新后收藏是否仍然存在。
- 用户是否知道当前是课程样例数据。
