# 项目总说明

## 1. 项目名称

钓鱼露营地点收藏 H5

## 2. 一句话说明

一个帮助用户浏览、查看和收藏钓鱼露营地点的 H5 页面 / 轻量 Web App 样例。

## 3. 目标用户

喜欢周末去城市周边钓鱼、露营、短途户外的人。

## 4. 用户痛点

钓鱼和露营地点信息分散在小红书、短视频、微信群、地图评论和朋友推荐里，用户想收藏和回看地点时不方便。

## 5. 核心场景

用户准备周末出行时，打开 H5 页面浏览地点，进入详情页查看说明，觉得有用就收藏，后续在收藏页快速找到。

## 6. 第一版做什么

- 展示地点列表。
- 查看地点详情。
- 收藏地点。
- 查看已收藏地点。
- 清空收藏。
- 展示关于说明。

## 7. 第一版不做什么

- 不做登录。
- 不做后端。
- 不做地图。
- 不做发布。
- 不做评论。
- 不做私信。
- 不做支付。
- 不做云同步。

## 8. 成功标准

用户可以完成：

```text
首页浏览地点
↓
进入详情页
↓
点击收藏
↓
进入收藏页查看
```

## 9. 最大风险

1. 用户可能误以为地点是真实推荐，因此页面必须说明当前是课程样例数据。
2. 第一版如果加入地图和登录，会显著增加复杂度。
3. 如果不写清文档到代码的映射，学员会误以为代码是凭空生成的。
