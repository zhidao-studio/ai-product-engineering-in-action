# 用户流程

## 1. 核心流程

```text
用户打开 H5
↓
首页看到地点列表
↓
点击一个地点卡片
↓
进入详情页
↓
点击收藏
↓
看到收藏状态变化
↓
进入收藏页
↓
看到刚刚收藏的地点
```

## 2. 流程闭环判断

错误流程：

```text
首页 → 详情页 → 收藏
```

问题：

```text
用户收藏之后去哪看？
收藏是否成功？
结果在哪里？
```

正确流程：

```text
首页 → 详情页 → 收藏 → 收藏页查看
```

规则：

```text
用户做了一个动作，就必须看到对应结果。
```

## 3. 页面跳转关系

| 当前页面 | 用户动作 | 目标页面 / 结果 |
|---|---|---|
| 首页 | 点击地点卡片 | 详情页 |
| 详情页 | 点击收藏 | 收藏状态变化 |
| 详情页 | 点击返回 | 首页 |
| 顶部导航 | 点击收藏 | 收藏页 |
| 收藏页 | 点击地点卡片 | 详情页 |
| 顶部导航 | 点击设置 | 设置页 |
| 顶部导航 | 点击关于 | 关于页 |
| 设置页 | 点击清空收藏 | 收藏列表清空 |

## 4. 用户验收任务

让真实用户完成：

```text
请打开页面，找到一个你感兴趣的地点，收藏它，并在收藏页找到它。
```

通过标准：

- 用户不需要解释就能找到地点列表。
- 用户知道点击地点可以看详情。
- 用户知道哪个按钮是收藏。
- 用户收藏后能在收藏页看到结果。
