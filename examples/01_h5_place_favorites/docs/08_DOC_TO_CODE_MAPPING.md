# 文档到代码映射

## 1. 为什么需要这份文档

这份文档用于回答学员最核心的问题：

```text
代码为什么是这样写的？
它是从哪份文档推导出来的？
```

如果没有这份映射，学员会觉得代码是 AI 直接凭空生成的。

## 2. 总体映射关系

| 文档 | 约束代码 | 说明 |
|---|---|---|
| `01_PROJECT_BRIEF.md` | README、首页说明文案 | 决定产品定位、目标用户和边界 |
| `02_PRD.md` | 任务清单、功能实现 | 决定第一版做哪些功能 |
| `03_PAGE_LIST.md` | `index.html`、页面渲染结构 | 决定有哪些页面和模块 |
| `04_UX_FLOW.md` | `app.js` | 决定点击、跳转、收藏后的结果 |
| `05_DATA_SPEC.md` | `mock/places.js`、`app.js` | 决定数据字段和收藏存储方式 |
| `06_ARCHITECTURE.md` | 文件结构 | 决定为什么是 HTML / CSS / JS / mock |
| `07_TASKS.md` | 每次代码变更 | 决定 AI 每次只改什么 |

## 3. 页面到代码映射

| 页面 | 来源文档 | 代码位置 | 验收方式 |
|---|---|---|---|
| 首页 | `03_PAGE_LIST.md` P001 | `index.html`、`app.js` | 能看到地点列表 |
| 详情页 | `03_PAGE_LIST.md` P002 | `app.js` | 点击地点后显示详情 |
| 收藏页 | `03_PAGE_LIST.md` P003 | `app.js` | 能看到已收藏地点 |
| 设置页 | `03_PAGE_LIST.md` P004 | `app.js` | 能清空收藏 |
| 关于页 | `03_PAGE_LIST.md` P005 | `app.js` | 能看到课程样例说明 |

## 4. 功能到代码映射

| 功能 | 来源文档 | 代码位置 | 验收标准 |
|---|---|---|---|
| 首页地点列表 | PRD F001 | `app.js` 渲染列表 | 至少显示 2 个地点 |
| 地点详情 | PRD F002 | `app.js` 渲染详情 | 展示名称、说明、标签、注意事项 |
| 收藏地点 | PRD F003 | `app.js` + localStorage | 收藏状态能切换 |
| 收藏页 | PRD F004 | `app.js` 读取收藏 ID | 收藏页显示已收藏地点 |
| 设置页 | PRD F005 | `app.js` 清空收藏 | 点击后收藏为空 |
| 关于页 | PRD F006 | `index.html` 或 `app.js` | 能说明课程样例定位 |

## 5. 数据字段到页面映射

| 数据字段 | 页面使用位置 | 说明 |
|---|---|---|
| `id` | 详情跳转、收藏存储 | 不直接展示，但用于逻辑判断 |
| `name` | 首页、详情页、收藏页 | 地点名称 |
| `region` | 首页、详情页 | 地区信息 |
| `distanceText` | 首页 | 距离说明 |
| `summary` | 首页 | 简短摘要 |
| `description` | 详情页 | 详细说明 |
| `tags` | 首页、详情页 | 标签 |
| `tips` | 详情页 | 注意事项 |

## 6. 代码审查时怎么用

审查 AI 代码时，不只问“能不能跑”，还要问：

```text
这段代码是否对应某个任务？
这个页面是否来自页面清单？
这个字段是否来自数据规范？
这个交互是否来自用户流程？
这个功能是否满足 PRD 验收标准？
```

## 7. 学员必须记住的一句话

```text
文档不是写完放着看的，
文档是用来约束 AI 写代码和验收代码的。
```
