# H5 样例文档工程入口

这里不是普通说明文档目录，而是 H5 实战样例的“需求实现控制系统”。

## 1. 核心目标

通过一整套完整文档，控制需求如何被 Agent 实现。

也就是说：

```text
不是先写代码，
而是先用文档定义边界、流程、数据、任务和验收，
再让 Agent 按任务实现。
```

## 2. 推荐阅读顺序

| 顺序 | 文档 | 作用 |
|---:|---|---|
| 1 | `00_DOC_ENGINEERING_FLOW.md` | 先理解文档工程如何控制代码实现 |

| 3 | `02_PRD.md` | 明确第一版功能和验收标准 |

## 4. 实现前规则

实现前必须先确认：

```text
需求已经进入 PRD。
页面已经进入页面清单。
流程已经进入用户流程。
数据字段已经进入数据规范。
文件职责已经进入技术结构。
任务已经进入任务清单。
Agent 输入已经写清楚允许和禁止范围。
```

否则不能进入代码实现。

## 5. 实现后规则

实现后必须确认：

```text
代码没有超出当前任务。
代码没有修改无关文件。
代码没有新增未经文档批准的能力。
代码符合页面清单。
代码符合用户流程。
代码符合数据规范。
代码满足验收标准。
用户能完成核心流程。
```

## 6. 一句话原则

```text
用文档控制需求，
用任务约束 Agent，
用验收判断结果。
```
