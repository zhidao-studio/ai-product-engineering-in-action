# H5 实战样例：钓鱼露营地点收藏

这是《AI 产品工程实战》的第一版实战样例工程。

## 为什么第一版先做 H5

相比微信小程序，H5 的启动门槛更低：

- 不需要 AppID。
- 不需要微信开发者工具。
- 浏览器即可运行。
- 方便非技术学员理解页面、数据、交互和本地存储。
- 后续可以迁移到微信小程序、iOS App 或 Web App。

## 为什么第一版使用这 5 类文件

第一版使用：

```text
index.html
styles.css
app.js
mock/places.js
README.md
```

这不是随便选的文件结构，而是为了让非技术学员先理解一个产品最基本的 5 个组成部分。

| 文件 | 学员应该理解成什么 | 负责什么 | 类比 |
|---|---|---|---|
| `index.html` | 页面结构 | 页面上有什么内容 | 房子的骨架 |
| `styles.css` | 视觉样式 | 页面长什么样 | 装修和排版 |
| `app.js` | 交互逻辑 | 点击、切换、收藏、状态变化 | 开关和电路 |
| `mock/places.js` | 模拟数据 | 页面展示哪些地点 | 练习用的数据表 |
| `README.md` | 项目说明 | 这个项目是什么、怎么运行、怎么验收 | 使用说明书 |

## 为什么不一开始用 Vue / React / 小程序框架

第一版不引入框架，是为了降低理解成本。

非技术学员第一步要学的不是框架，而是：

```text
页面上有什么
↓
页面长什么样
↓
用户点了会发生什么
↓
数据从哪里来
↓
怎么判断做完了
```

如果一开始就用 Vue、React、Next.js 或微信小程序框架，学员会同时面对：

- 工程配置
- 构建工具
- 依赖安装
- 组件语法
- 路由机制
- 状态管理
- 平台规则

这些会把课程重点从“AI 产品工程流程”转移到“前端框架学习”。

所以第一版先用原生 H5，把产品工程的主线讲清楚。

## 这 5 类文件和课程流程的关系

| 课程产物 | 对应文件 |
|---|---|
| 页面清单 | `index.html` |
| 界面规范 | `styles.css` |
| 用户流程 / 交互逻辑 | `app.js` |
| 数据规范 | `mock/places.js` |
| 项目总说明 / 验收说明 | `README.md` |

也就是说，学员不是在学“几个文件名”，而是在学习：

```text
产品文档如何落到代码结构里
```

## 第一版页面

- 首页：展示地点列表
- 详情页：展示地点详情和收藏按钮
- 收藏页：展示已收藏地点
- 设置页：清空收藏和基础说明
- 关于页：说明课程样例

## 第一版不做

- 登录
- 后端
- 地图
- 发布
- 评论
- 私信
- 支付
- 云同步

## 如何运行

进入本目录后，用浏览器打开：

```text
index.html
```

如果浏览器限制本地文件访问，可以使用本地静态服务器：

```bash
python -m http.server 5173
```

然后访问：

```text
http://localhost:5173
```

## 核心流程

```text
首页浏览地点
↓
点击地点进入详情
↓
点击收藏
↓
进入收藏页查看
```
